10.1 (Computationally unbounded adversaries)

Show that an anonymous key exchange protocol \(P = (A,B)\) (as in Definition 10.1) cannot be secure against a computationally unbounded adversary. This explains why all the protocols in this chapter must rely on computational assumptions.

Idea

The shared key \(k\) and the transcript \(T_P\) are both deterministic functions of the random tapes of \(A\) and \(B\). Once the transcript is fixed, the only thing the adversary is missing is the secret randomness of the two parties. An unbounded adversary can simply search over all random tapes consistent with the observed transcript, replay the protocol in its head, and recompute \(k\) exactly. There is no computational obstacle, so the adversary wins outright.

Solution

Recall the setup of Attack Game 10.1. The protocol \(P = (A,B)\) is a pair of probabilistic machines; let \(r_A \in \{0,1\}^{\ell_A}\) and \(r_B \in \{0,1\}^{\ell_B}\) denote the (finite) random tapes consumed by \(A\) and \(B\) respectively during one execution. Once \(r_A\) and \(r_B\) are fixed, the entire execution is deterministic: every message is a deterministic function of the tapes and of the messages received so far. In particular both the transcript and the final shared key are deterministic functions

\[ T_P = \mathcal{T}(r_A, r_B), \qquad k = \mathcal{K}(r_A, r_B), \]

for fixed functions \(\mathcal{T}\) and \(\mathcal{K}\) determined by the code of \(A\) and \(B\). The randomness of \(T_P\) and \(k\) as random variables comes solely from the uniform choice of \((r_A, r_B)\).

We construct an explicit unbounded adversary \(\mathcal{A}\) achieving advantage \(\mathrm{AnonKEadv}[\mathcal{A}, P] = 1\). On input the transcript \(T_P\):

1. \(\mathcal{A}\) enumerates all pairs of tapes \((r_A', r_B') \in \{0,1\}^{\ell_A} \times \{0,1\}^{\ell_B}\) (a finite, though exponentially large, search — feasible for an unbounded adversary).
2. For each candidate \((r_A', r_B')\), \(\mathcal{A}\) simulates the protocol from scratch using these tapes and computes the resulting transcript \(\mathcal{T}(r_A', r_B')\).
3. \(\mathcal{A}\) picks any pair \((r_A^\star, r_B^\star)\) for which \(\mathcal{T}(r_A^\star, r_B^\star) = T_P\), and outputs \(\hat{k} := \mathcal{K}(r_A^\star, r_B^\star)\).

Such a consistent pair always exists, since the true tapes \((r_A, r_B)\) that produced \(T_P\) are themselves a witness; hence step 3 never fails. It remains to argue \(\hat{k} = k\) with probability \(1\). The key point is the correctness requirement of a key exchange protocol: whenever \(A\) and \(B\) run together they terminate holding the same value. In fact the shared key is a function of the transcript alone:

\[ \mathcal{T}(r_A', r_B') = \mathcal{T}(r_A'', r_B'') \;\Longrightarrow\; \mathcal{K}(r_A', r_B') = \mathcal{K}(r_A'', r_B''). \]

To see this, note that the value held by \(B\) at the end of the protocol is determined by \(B\)'s own tape \(r_B'\) together with the messages \(B\) received from \(A\) — and those received messages are exactly the \(A\)-to-\(B\) messages recorded in the transcript. Likewise \(A\)'s value is determined by \(r_A'\) and the \(B\)-to-\(A\) messages in the transcript. By correctness these two values coincide and equal the common key, so the common key depends only on the messages, i.e. only on the transcript. Now the consistent pair \((r_A^\star,r_B^\star)\) the adversary finds reproduces the very transcript \(T_P\); therefore the key it recomputes equals the key of the real execution that generated \(T_P\):

\[ \hat{k} = \mathcal{K}(r_A^\star, r_B^\star) = \mathcal{K}(r_A, r_B) = k. \]

Thus \(\mathcal{A}\) outputs \(k\) with probability \(1\), so \(\mathrm{AnonKEadv}[\mathcal{A}, P] = 1\), which is certainly non-negligible. No protocol can be secure against this adversary. Intuitively, the only thing protecting the key is the adversary's inability to find tapes consistent with the transcript; remove the computational bound and that protection evaporates. This is exactly why every protocol in this chapter rests on a computational hardness assumption (DL, CDH, DDH, RSA), as required.

10.2 (Key exchange requires randomness from both parties)

Consider an anonymous key exchange protocol \(P = (A,B)\) between two probabilistic machines (as in Definition 10.1). Suppose one of the machines is deterministic, i.e. it uses no random bits when participating in the protocol. Show that \(P\) cannot be secure: there is an efficient adversary \(\mathcal{A}\) for which \(\mathrm{AnonKEadv}[\mathcal{A}, P] = 1\). This explains why in all the protocols of this chapter both parties are randomized.

Idea

If one party uses no randomness, then everything it does is a deterministic function of the messages it received — and those messages are all visible in the transcript. So the adversary can just replay that party's program on the transcript, step for step, and recompute its final output, which by correctness is the shared key. No search and no assumption are needed: the deterministic party leaves no secret behind.

Solution

Without loss of generality, suppose \(B\) is the deterministic machine (the argument is symmetric if it is \(A\)). Because \(B\) consumes no random bits, its behaviour during the protocol is a fixed deterministic function of \(B\)'s publicly known code and of the messages it receives. Concretely, the \(i\)-th message \(B\) sends is computed as

\[ m^{(B)}_i = B\big(\, m^{(A)}_1, m^{(A)}_2, \ldots, m^{(A)}_{i}\,\big), \]

and \(B\)'s final output (its copy of the shared key) is

\[ k = B_{\mathrm{out}}\big(\, m^{(A)}_1, \ldots, m^{(A)}_{t}\,\big), \]

where \(m^{(A)}_1, \ldots, m^{(A)}_t\) are exactly the messages \(A\) sent to \(B\). Crucially, all of these messages appear verbatim in the transcript \(T_P\) handed to the adversary in Attack Game 10.1, and the function \(B_{\mathrm{out}}(\cdot)\) is just \(B\)'s own efficient, publicly specified code.

We give the explicit efficient adversary \(\mathcal{A}\). On input the transcript \(T_P\):

1. \(\mathcal{A}\) parses out from \(T_P\) the subsequence of messages \(m^{(A)}_1, \ldots, m^{(A)}_t\) sent from \(A\) to \(B\).
2. \(\mathcal{A}\) runs \(B\)'s code (with an empty random tape, since \(B\) is deterministic), feeding it exactly these messages in order, thereby perfectly simulating the deterministic machine \(B\).
3. \(\mathcal{A}\) outputs the value \(\hat{k} := B_{\mathrm{out}}(m^{(A)}_1, \ldots, m^{(A)}_t)\) produced by this simulation.

This adversary is efficient: it does no more work than \(B\) does in a single honest run, plus trivial parsing of the transcript.

For correctness of the attack: since \(\mathcal{A}\) feeds \(B\)'s code precisely the messages \(B\) actually received in the real execution (read off the transcript), and \(B\) is deterministic, \(\mathcal{A}\)'s simulation reproduces \(B\)'s real internal computation exactly; in particular \(\hat{k}\) equals the value \(B\) actually output at the end of the real execution. By the correctness property of the protocol, that output is the shared key \(k\). Hence \(\hat{k} = k\) with probability \(1\), regardless of the random tape \(r_A\) used by \(A\). Therefore

\[ \mathrm{AnonKEadv}[\mathcal{A}, P] = \Pr[\hat{k} = k] = 1, \]

which is non-negligible, so \(P\) is insecure. The moral: a deterministic party contributes nothing that an eavesdropper cannot recompute from the transcript, so its messages and final key are effectively public. Genuine secrecy of the shared key therefore requires fresh secret randomness from both parties, as required.

10.3 (DDH PRG)

Let \(\mathbb{G}\) be a cyclic group of prime order \(q\) generated by \(g \in \mathbb{G}\). Consider the PRG defined over \((\mathbb{Z}_q^2,\ \mathbb{G}^3)\) by

\[ G(\alpha, \beta) := \big(g^\alpha,\ g^\beta,\ g^{\alpha\beta}\big). \]

Show that \(G\) is a secure PRG, assuming DDH holds in \(\mathbb{G}\).

Idea

The idea is simple. The PRG output on a random seed is, by definition, a uniformly random DH-triple; a truly random element of the output space \(\mathbb{G}^3\) is a uniformly random triple. But distinguishing a random DH-triple from a random triple in \(\mathbb{G}^3\) is precisely the DDH game. So a PRG distinguisher is a DDH distinguisher — the reduction is the identity, an elementary wrapper that simply forwards its challenge.

Solution

We show that any efficient PRG adversary \(\mathcal{A}\) yields an efficient DDH adversary \(\mathcal{B}\), an elementary wrapper around \(\mathcal{A}\), with

\[ \mathrm{PRGadv}[\mathcal{A}, G] = \mathrm{DDHadv}[\mathcal{B}, \mathbb{G}]. \]

Matching up the two distributions. In the PRG attack game (Attack Game 3.1) the adversary receives \(r\), where

in the real case (\(b=0\)): the challenger picks a seed \((\alpha,\beta) \xleftarrow{\$} \mathbb{Z}_q^2\) and sets \(r = G(\alpha,\beta) = (g^\alpha, g^\beta, g^{\alpha\beta})\);
in the random case (\(b=1\)): the challenger picks \(r \xleftarrow{\$} \mathbb{G}^3\).

Compare this with the DDH attack game (Attack Game 10.6), where the challenger picks \(\alpha,\beta,\gamma \xleftarrow{\$} \mathbb{Z}_q\) and sends \((g^\alpha, g^\beta, w_b)\) with \(w_0 = g^{\alpha\beta}\) and \(w_1 = g^\gamma\). We record the two distributional facts that drive the proof:

(i) Real PRG output \(=\) DDH Experiment 0. When \((\alpha,\beta)\) is uniform over \(\mathbb{Z}_q^2\), the triple \((g^\alpha, g^\beta, g^{\alpha\beta})\) is exactly a uniformly random DH-triple, identical to the distribution of \((g^\alpha,g^\beta,g^{\alpha\beta})\) sent in DDH Experiment \(0\). (Here we used that \(g\) is a generator, so \(\delta \mapsto g^\delta\) is a bijection \(\mathbb{Z}_q \to \mathbb{G}\), making \(g^\alpha, g^\beta\) independent and uniform over \(\mathbb{G}\), with the third coordinate the determined value \(g^{\alpha\beta}\).)

(ii) Random PRG output \(=\) DDH Experiment 1. A uniform element of \(\mathbb{G}^3\) is a uniformly random triple. The DDH Experiment \(1\) output \((g^\alpha, g^\beta, g^\gamma)\) with \(\alpha,\beta,\gamma\) independent and uniform over \(\mathbb{Z}_q\) is likewise uniform over \(\mathbb{G}^3\) (again because \(\delta \mapsto g^\delta\) is a bijection, applied coordinatewise). Hence the two distributions are identical.

The reduction \(\mathcal{B}\). Adversary \(\mathcal{B}\) plays the DDH game: it receives a challenge triple \((u, v, w) \in \mathbb{G}^3\). It does nothing but hand \(r := (u,v,w)\) to \(\mathcal{A}\) as the PRG challenge, and outputs whatever bit \(\hat{b}\) the adversary \(\mathcal{A}\) outputs. This is the promised elementary wrapper; it is efficient (it merely forwards \(3\) group elements and a bit) whenever \(\mathcal{A}\) is.

When \(\mathcal{B}\) is in DDH Experiment \(0\), by (i) the value \(r\) it forwards is distributed exactly as the PRG real output, so \(\mathcal{B}\) perfectly simulates the \(b=0\) PRG experiment for \(\mathcal{A}\). When \(\mathcal{B}\) is in DDH Experiment \(1\), by (ii) the value \(r\) is distributed exactly as the PRG random output, so \(\mathcal{B}\) perfectly simulates the \(b=1\) PRG experiment. Writing \(W_b\) for the event that \(\mathcal{A}\) (equivalently \(\mathcal{B}\)) outputs \(1\) in Experiment \(b\), the simulations are perfect, so the probability \(\mathcal{B}\) outputs \(1\) in DDH Experiment \(b\) equals the probability \(\mathcal{A}\) outputs \(1\) in PRG Experiment \(b\), for each \(b \in \{0,1\}\). Therefore

\[ \mathrm{DDHadv}[\mathcal{B}, \mathbb{G}] = \big|\Pr[W_0] - \Pr[W_1]\big| = \mathrm{PRGadv}[\mathcal{A}, G]. \]

Conclusion. We obtain the bound

\[ \mathrm{PRGadv}[\mathcal{A}, G] \le \mathrm{DDHadv}[\mathcal{B}, \mathbb{G}], \]

(in fact with equality). Under the DDH assumption \(\mathrm{DDHadv}[\mathcal{B}, \mathbb{G}]\) is negligible for every efficient \(\mathcal{B}\); since \(\mathcal{B}\) is efficient whenever \(\mathcal{A}\) is, \(\mathrm{PRGadv}[\mathcal{A}, G]\) is negligible for every efficient \(\mathcal{A}\). Hence \(G\) is a secure PRG, as required. (Note the seed \((\alpha,\beta)\in\mathbb{Z}_q^2\) maps to a triple in \(\mathbb{G}^3\): the output carries one more group element than the seed information-theoretically determines, yet remains pseudorandom — so \(G\) genuinely expands.)

10.17 (Problems equivalent to CDH)

Let \(\mathbb{G}\) be a cyclic group of prime order \(q\) with generator \(g\). Show that the following four problems are deterministic poly-time equivalent (all instances are with respect to the same \(\mathbb{G}, g\)):

(a) CDH: given \(g^\alpha, g^\beta\), compute \(g^{\alpha\beta}\).
(b) Squaring (SQ): given \(g^\alpha\), compute \(g^{\alpha^2}\).
(c) Inverse (INV): given \(g^\alpha\) with \(\alpha \ne 0\), compute \(g^{1/\alpha}\).
(d) Division (DIV): given \(g^\alpha, g^\beta\) with \(\beta \ne 0\), compute \(g^{\alpha/\beta}\).

(Here \(1/\alpha\) and \(\alpha/\beta\) denote the inverse and quotient in the field \(\mathbb{Z}_q\), which exist because \(q\) is prime.)

Idea

This is pure algebra in the exponent: each problem performs one field operation (multiply, square, invert, divide) on hidden exponents. The point is that these operations are interrelated by polynomial identities — squaring plus addition yields multiplication via \(\alpha\beta = \tfrac12\big((\alpha+\beta)^2 - \alpha^2 - \beta^2\big)\); inversion is Fermat exponentiation \(\alpha^{-1}=\alpha^{q-2}\); division is multiply-by-inverse. Since adding, subtracting and scaling exponents are "free" (they are just products/quotients/powers of known group elements), each reduction is a deterministic poly-time wrapper making a polynomial number of oracle calls. No probabilities appear. We exhibit a cycle of reductions making all four mutually equivalent.

Solution

Free operations. Given group elements \(g^x, g^y\) and a known integer constant \(c\), we can compute \(g^{x+y} = g^x g^y\), \(g^{x-y} = g^x (g^y)^{-1}\) and \(g^{cx} = (g^x)^c\) in deterministic polynomial time, without knowing \(x\) or \(y\). We call these "free" operations; each reduction below uses only free operations together with a bounded number of oracle calls. Since \(q\) is an odd prime, \(2\) is invertible mod \(q\); write \(2^{-1}\in\mathbb{Z}_q\) (computable by Euclid). We will establish the cycle

\[ \text{(a)} \to \text{(b)} \to \text{(c)} \to \text{(d)} \to \text{(a)}, \]

plus the reverse edge \(\text{(b)}\to\text{(a)}\); since deterministic poly-time reducibility composes (replacing each oracle call of one reduction by another polynomial-time reduction stays polynomial), a directed cycle through all four nodes makes every problem reducible to every other, hence all four are pairwise equivalent.

(a) CDH \(\Rightarrow\) (b) Squaring. Given an SQ instance \(g^\alpha\), call the CDH oracle on the pair \((g^\alpha, g^\alpha)\). By definition CDH returns \(g^{\alpha\cdot\alpha}=g^{\alpha^2}\), the desired SQ output. One oracle call. So SQ is deterministic poly-time reducible to CDH.

(b) Squaring \(\Rightarrow\) (a) CDH (the polarization identity). For all \(\alpha,\beta\in\mathbb{Z}_q\),

\[ \alpha\beta \;=\; 2^{-1}\big((\alpha+\beta)^2 - \alpha^2 - \beta^2\big) \pmod q. \tag{\ast} \]

Given \((g^\alpha,g^\beta)\): compute \(g^{\alpha+\beta}=g^\alpha g^\beta\) (free); call SQ three times to get \(g^{(\alpha+\beta)^2}\), \(g^{\alpha^2}\), \(g^{\beta^2}\); then form

\[ \Big(g^{(\alpha+\beta)^2}\cdot g^{-\alpha^2}\cdot g^{-\beta^2}\Big)^{\,2^{-1}} = g^{\,2^{-1}\big((\alpha+\beta)^2-\alpha^2-\beta^2\big)} = g^{\alpha\beta}. \]

Three SQ-calls and free operations. So CDH is deterministic poly-time reducible to SQ. Combined with the previous edge, (a) and (b) are already equivalent, and \((\ast)\) gives us a CDH routine \(\mathsf{C}(g^x,g^y)=g^{xy}\) whenever we possess an SQ (equivalently CDH) oracle — we use this below.

(b) Squaring \(\Rightarrow\) (c) Inverse. By the previous paragraph an SQ oracle yields a full CDH routine \(\mathsf{C}(g^x,g^y)=g^{xy}\). We use it to invert one exponent via Fermat's little theorem: since \(\alpha\in\mathbb{Z}_q^\ast\) and \(\lvert\mathbb{Z}_q^\ast\rvert=q-1\),

\[ \alpha^{-1} = \alpha^{\,q-2} \pmod q, \qquad\text{so}\qquad g^{1/\alpha} = g^{\alpha^{q-2}}. \]

We compute \(g^{\alpha^{q-2}}\) from \(g^\alpha\) by exponentiation-by-squaring in the exponent. The basic step combines two running values by multiplying their \(\alpha\)-exponents:

\[ \mathsf{C}\big(g^{\alpha^i},\,g^{\alpha^j}\big) = g^{\alpha^i\cdot\alpha^j} = g^{\alpha^{i+j}}. \]

In particular squaring a running value \(g^{\alpha^k}\) gives \(g^{\alpha^{2k}}\), and multiplying by \(g^{\alpha}=g^{\alpha^1}\) increments the \(\alpha\)-exponent by \(1\). Running standard square-and-multiply over the \(O(\log q)\) bits of the exponent \(q-2\), starting from \(g^{\alpha^1}=g^\alpha\), lands on \(g^{\alpha^{q-2}}=g^{1/\alpha}\), using \(O(\log q)\) calls to \(\mathsf{C}\) (hence \(O(\log q)\) SQ-calls) and \(O(\log q)\) free operations. This is deterministic and polynomial-time. So INV is deterministic poly-time reducible to SQ.

(c) Inverse \(\Rightarrow\) (d) Division. Given a DIV instance \((g^\alpha, g^\beta)\) with \(\beta\ne0\), we want \(g^{\alpha/\beta}\). First reduce DIV to INV plus one multiplication, then realize that multiplication from INV. Indeed

\[ \frac{\alpha}{\beta} = \alpha\cdot\beta^{-1}. \]

Compute \(g^{1/\beta}=\mathrm{INV}(g^\beta)\) (one INV call). It remains to multiply the exponents \(\alpha\) and \(\beta^{-1}\), given \(g^\alpha\) and \(g^{1/\beta}\). We obtain a multiplier from INV alone, by first extracting a squaring capability and then applying the polarization identity \((\ast)\). The squaring capability comes from the following verified identity (for \(t\ne0,-1\)):

\[ t^2 = \Big(\tfrac{1}{t}-\tfrac{1}{t+1}\Big)^{-1} - t, \qquad\text{since } \tfrac1t-\tfrac1{t+1}=\tfrac{1}{t(t+1)}=\tfrac{1}{t^2+t}. \tag{\dagger} \]

Thus INV gives SQ: compute \(g^{1/t}=\mathrm{INV}(g^t)\), \(g^{1/(t+1)}=\mathrm{INV}(g^t\cdot g)\), then \(g^{1/t}\,(g^{1/(t+1)})^{-1}=g^{1/(t^2+t)}\), invert to get \(g^{t^2+t}\), and divide by \(g^t\) to get \(g^{t^2}\) (degenerate exponents \(t\in\{0,-1\}\), detectable since \(g^t\in\{1,g^{-1}\}\), give \(g^{t^2}\in\{1,g\}\) directly). Composing this SQ with the polarization routine \((\ast)\) yields a CDH multiplier \(\mathsf{C}\) built from INV alone. Now finish the division:

\[ g^{\alpha/\beta} = \mathsf{C}\big(g^{\alpha},\,g^{1/\beta}\big) = g^{\alpha\cdot(1/\beta)}, \]

using one INV call for \(g^{1/\beta}\) plus the \(O(\log q)\)-call \(\mathsf{C}\) machinery (which itself is a constant number of INV/SQ calls and \((\ast)\)). All deterministic poly-time. So DIV is deterministic poly-time reducible to INV.

(d) Division \(\Rightarrow\) (a) CDH. This closes the cycle. Given \((g^\alpha,g^\beta)\), we want \(g^{\alpha\beta}\). Handle degenerate cases first: if \(g^\alpha=1\) (so \(\alpha=0\)) or \(g^\beta=1\) (so \(\beta=0\)) the answer is \(g^{\alpha\beta}=g^0=1\); these are detected by comparing \(g^\alpha,g^\beta\) to the identity. Otherwise \(\alpha,\beta\ne0\). First obtain a squaring capability from DIV: feeding the numerator \(g^1=g\) gives inversion,

\[ \mathrm{DIV}(g^{1},\,g^{\delta}) = g^{1/\delta} \quad(\delta\ne0), \]

so DIV subsumes INV; and INV gives SQ by \((\dagger)\) above. With SQ in hand, the polarization identity \((\ast)\) computes

\[ g^{\alpha\beta} = \Big(g^{(\alpha+\beta)^2}\cdot g^{-\alpha^2}\cdot g^{-\beta^2}\Big)^{2^{-1}}, \]

where the three squares are produced by the DIV-based SQ routine. A constant number of DIV calls and free operations, all deterministic poly-time. So CDH is deterministic poly-time reducible to DIV.

Wrap-up. We have exhibited the deterministic poly-time reductions

\[ \text{(a)}\Rightarrow\text{(b)},\quad \text{(b)}\Rightarrow\text{(a)},\quad \text{(b)}\Rightarrow\text{(c)},\quad \text{(c)}\Rightarrow\text{(d)},\quad \text{(d)}\Rightarrow\text{(a)}, \]

each using \(O(1)\) or \(O(\log q)\) oracle calls plus free group operations. The edges \(\text{(a)}\to\text{(b)}\to\text{(c)}\to\text{(d)}\to\text{(a)}\) form a directed cycle through all four problems. Since deterministic poly-time reducibility is transitive, every problem is reducible to every other along this cycle, and therefore problems (a), (b), (c), (d) are pairwise deterministic poly-time equivalent, as required.

11.1 (From weak PRF to PRF via hashing)

Let \(F'(k,x)\) be a PRF over \((\mathcal K,\mathcal X,\mathcal Y)\) that is weakly secure (Definition 4.3). Let \(H:\mathcal M\to\mathcal X\) be a hash function, modeled as a random oracle, and define a new PRF over \((\mathcal K,\mathcal M,\mathcal Y)\) by \[ F(k,m):=F'\bigl(k,H(m)\bigr). \] Show that \(F\) is a secure PRF in the random oracle model. Concretely, for every adversary \(\mathcal A\) attacking \(F\) as a PRF there is an adversary \(\mathcal B\) attacking \(F'\) as a weak PRF, an elementary wrapper around \(\mathcal A\), with \[ \mathrm{PRFro\,adv}[\mathcal A,F]\le \mathrm{wPRFadv}[\mathcal B,F']. \]

Idea

The idea is simple. In the random oracle model the only way \(F(k,m)=F'(k,H(m))\) ever touches the key \(k\) is through the value \(H(m)\); and for distinct queries \(m\), the points \(H(m)\) are uniform and independent in \(\mathcal X\). These are exactly the random inputs at which a weak-PRF challenger evaluates \(F'\). So an attacker \(\mathcal A\) on \(F\) is, after stripping away the random oracle, nothing more than an attacker that observes \(F'\) at random points — i.e. a weak-PRF attacker \(\mathcal B\).

Solution

We build \(\mathcal B\), an elementary wrapper around \(\mathcal A\), that plays the weak-PRF game (Definition 4.3) against \(F'\). Recall that in that game \(\mathcal B\)'s challenger holds a function \(f\) (either \(f=F'(k,\cdot)\) for \(k\xleftarrow{\$}\mathcal K\), or \(f\xleftarrow{\$}\mathrm{Funs}[\mathcal X,\mathcal Y]\)); on each query \(\mathcal B\) receives a fresh pair \((x,f(x))\) with \(x\xleftarrow{\$}\mathcal X\) chosen by the challenger.

\(\mathcal B\) must simultaneously play the role of the \(F\)-challenger to \(\mathcal A\), answering both \(\mathcal A\)'s random-oracle queries to \(H\) and \(\mathcal A\)'s evaluation queries to \(F(k,\cdot)\). It maintains two initially empty associative arrays: \(\mathit{Map}:\mathcal M\to\mathcal X\) implementing the random oracle \(H\) as a faithful gnome, and \(\mathit{Out}:\mathcal M\to\mathcal Y\) recording the value of \(F\) at points already used. The crucial trick is that \(\mathcal B\) populates \(\mathit{Map}\) using the random points handed to it by its own weak-PRF challenger.

Concretely, \(\mathcal B\) defines a single subroutine \(\textsf{Fresh}(m)\) invoked the first time any message \(m\) is touched (by either kind of query):

\[ \textsf{Fresh}(m):\quad \text{request a fresh pair }(x,y)\text{ from the weak-PRF challenger};\ \mathit{Map}[m]\leftarrow x;\ \mathit{Out}[m]\leftarrow y. \]

Then \(\mathcal B\) answers \(\mathcal A\) as follows:

  • Random-oracle query \(m\): if \(m\notin\mathrm{Domain}(\mathit{Map})\) run \(\textsf{Fresh}(m)\); return \(\mathit{Map}[m]\).
  • Evaluation query \(m\) (for \(F(k,\cdot)\)): if \(m\notin\mathrm{Domain}(\mathit{Map})\) run \(\textsf{Fresh}(m)\); return \(\mathit{Out}[m]\).

Finally \(\mathcal B\) outputs whatever \(\mathcal A\) outputs. Repeated queries (of either type) on the same \(m\) are answered consistently from the stored arrays, so \(\mathcal B\) never asks its challenger for two pairs on behalf of one \(m\); this is exactly the consistency a random oracle and a deterministic PRF require.

Correctness of the simulation. Consider the two cases of \(\mathcal B\)'s game.

Case \(b=0\) (real \(F'\)). Here \(y=f(x)=F'(k,x)\) for a single hidden key \(k\xleftarrow{\$}\mathcal K\). Each distinct \(m\) that \(\mathcal A\) touches gets a fresh \(x=\mathit{Map}[m]\) drawn uniformly and independently from \(\mathcal X\) by the challenger — precisely the distribution of a random oracle \(H\). And \(\mathit{Out}[m]=F'(k,\mathit{Map}[m])=F'(k,H(m))=F(k,m)\). Thus \(\mathcal B\) perfectly simulates the random-oracle PRF game for \(F\) in Experiment 0 (the real-\(F\) experiment). Hence \(\Pr[\mathcal B\Rightarrow 1\mid b=0]=\Pr[\mathcal A\Rightarrow 1\mid \text{real }F]\).

Case \(b=1\) (random \(f\)). Now \(f\xleftarrow{\$}\mathrm{Funs}[\mathcal X,\mathcal Y]\). The points \(x=\mathit{Map}[m]\) are still uniform and independent, so \(H\) is again perfectly simulated. Because all the \(x\)'s handed to \(\mathcal B\) are distinct with the values it actually uses (each fresh \(m\) gets its own challenger-chosen \(x\), and a random function evaluated at distinct inputs returns independent uniform outputs), the values \(\mathit{Out}[m]=f(\mathit{Map}[m])\) are independent and uniform in \(\mathcal Y\) across distinct \(m\). That is exactly the behavior of a truly random function \(\mathcal M\to\mathcal Y\). So \(\mathcal B\) perfectly simulates the random-oracle PRF game for \(F\) in Experiment 1 (the random-function experiment), and \(\Pr[\mathcal B\Rightarrow 1\mid b=1]=\Pr[\mathcal A\Rightarrow 1\mid \text{random}]\).

(A small technical point: in Case \(b=1\) two distinct messages \(m\ne m'\) could in principle receive colliding random points \(x=\mathit{Map}[m]=\mathit{Map}[m']\), in which case \(f\) would return the same value, whereas a random function on \(\mathcal M\) would not be forced to. This event only makes the two experiments closer to identical from \(\mathcal A\)'s viewpoint in the standard formulation, and is accounted for inside the weak-PRF advantage of \(\mathcal B\); it never helps \(\mathcal A\). Hence no extra additive term is needed, matching the clean bound.)

Subtracting the two probabilities gives, by the two displayed equalities, \[ \mathrm{PRFro\,adv}[\mathcal A,F]=\bigl|\Pr[\mathcal A\Rightarrow 1\mid\text{real}]-\Pr[\mathcal A\Rightarrow 1\mid\text{random}]\bigr| =\bigl|\Pr[\mathcal B\Rightarrow1\mid b{=}0]-\Pr[\mathcal B\Rightarrow1\mid b{=}1]\bigr|=\mathrm{wPRFadv}[\mathcal B,F'], \] which is exactly the required bound \(\mathrm{PRFro\,adv}[\mathcal A,F]\le\mathrm{wPRFadv}[\mathcal B,F']\), as required.

11.2 (Simple PRF from DDH)

Let \(\mathbb G\) be a cyclic group of prime order \(q\) with generator \(g\). Let \(H:\mathcal M\to\mathbb G\) be modeled as a random oracle, and define the PRF over \((\mathbb Z_q,\mathcal M,\mathbb G)\) by \[ F(k,m):=H(m)^k. \] Show that \(F\) is a secure PRF in the random oracle model under DDH for \(\mathbb G\); concretely, for every PRF adversary \(\mathcal A\) there is a DDH adversary \(\mathcal B\), an elementary wrapper around \(\mathcal A\), with \[ \mathrm{PRFro\,adv}[\mathcal A,F]\le \mathrm{DDHadv}[\mathcal B,\mathbb G]+1/q. \] Hint: first show \(F'(k,v):=v^k\) over \((\mathbb Z_q,\mathbb G,\mathbb G)\) is a weak PRF (Exercise 10.11), then apply Exercise 11.1.

Idea

Note \(F(k,m)=H(m)^k=F'(k,H(m))\) where \(F'(k,v)=v^k\). By Exercise 11.1 it suffices to prove \(F'\) is a weak PRF, i.e. secure when queried at random points \(v\xleftarrow{\$}\mathbb G\). But a weak-PRF transcript for \(F'\) is precisely \(\{(g^{\beta_j},g^{k\beta_j})\}_j\) for a fixed secret \(k\) — a list of Diffie–Hellman pairs with common exponent \(k\). Telling these apart from \(\{(g^{\beta_j},g^{\gamma_j})\}_j\) is exactly the multi-DDH problem of Exercise 10.11 (with \(n=1\)).

Solution

Step 1: \(F'(k,v)=v^k\) is a weak PRF. Let \(\mathcal C\) be a weak-PRF adversary for \(F'\) making at most \(m\) queries. In its game the challenger fixes \(f\): either \(f=F'(k,\cdot)\) for \(k\xleftarrow{\$}\mathbb Z_q\), or \(f\xleftarrow{\$}\mathrm{Funs}[\mathbb G,\mathbb G]\); on each query it returns a fresh pair \((v_j,f(v_j))\) with \(v_j\xleftarrow{\$}\mathbb G\). Write \(v_j=g^{\beta_j}\) with \(\beta_j\xleftarrow{\$}\mathbb Z_q\) (uniform, since \(g\) generates \(\mathbb G\)).

In the real case (\(f=F'(k,\cdot)\), \(k=\alpha_1\)) the transcript is \[ \bigl(g^{\beta_j},\,g^{\alpha_1\beta_j}\bigr)_{j=1}^{m}, \] while in the random case the transcript is \((g^{\beta_j},\,g^{\gamma_j})_{j=1}^m\) with \(\gamma_j\xleftarrow{\$}\mathbb Z_q\) independent. (A truly random \(f\) returns independent uniform group elements at the distinct query points \(v_j\); the only deviation is the negligible-probability event that two \(v_j\) collide, which we absorb below exactly as Exercise 10.11 does.)

These are precisely the distributions \(D\) and \(R\) of Exercise 10.11 with \(n=1\) and \(m\) samples: \(D\) consists of \(g^{\alpha_1}\) (the hidden \(k\), not actually shown — but \(\mathcal C\)'s view is a deterministic function of the listed pairs) together with \(\{g^{\beta_{1j}},g^{\alpha_1\beta_{1j}}\}_j\), and \(R\) replaces the third coordinates by \(g^{\gamma_{1j}}\). Hence any \(\mathcal C\) distinguishing the two cases yields, as an elementary wrapper, an adversary distinguishing \(D\) from \(R\), and Exercise 10.11 gives a DDH adversary \(\mathcal B\) with \[ \mathrm{wPRFadv}[\mathcal C,F']=\mathrm{Distadv}[\mathcal C,D,R]\le \mathrm{DDHadv}[\mathcal B,\mathbb G]+1/q. \]

Step 2: apply Exercise 11.1. Since \(F(k,m)=F'(k,H(m))\) with \(H:\mathcal M\to\mathbb G\) a random oracle, Exercise 11.1 gives, for every PRF adversary \(\mathcal A\) on \(F\), a weak-PRF adversary \(\mathcal C\) on \(F'\) (an elementary wrapper around \(\mathcal A\)) with \[ \mathrm{PRFro\,adv}[\mathcal A,F]\le \mathrm{wPRFadv}[\mathcal C,F']. \] Composing the two inequalities, and letting \(\mathcal B\) be the resulting (elementary-wrapper) DDH adversary, \[ \mathrm{PRFro\,adv}[\mathcal A,F]\le \mathrm{DDHadv}[\mathcal B,\mathbb G]+1/q, \] as required.

11.3 (Simple PRF from CDH)

Continuing Exercise 11.2, let \(H':\mathcal M\times\mathbb G\to\mathcal Y\) be a second hash function, also modeled as a random oracle. Define the PRF over \((\mathbb Z_q,\mathcal M,\mathcal Y)\) by \[ F'(k,m):=H'\bigl(m,\,H(m)^k\bigr). \] Show \(F'\) is a secure PRF under the CDH assumption for \(\mathbb G\), with \(H,H'\) random oracles. Hint: use Exercise 10.5.

Idea

Write \(w_m:=H(m)^k\). Since \(H'\) is a random oracle, \(F'(k,m)=H'(m,w_m)\) is a uniform independent value in \(\mathcal Y\) for each distinct \(m\) — looking exactly like a random function — unless \(\mathcal A\) actually queries \(H'\) at the precise point \((m,w_m)\). To do that the attacker must produce \(w_m=H(m)^k\), the Diffie–Hellman value tying the public \(g^k\) to the random-oracle output \(H(m)\). We plant a CDH challenge \((u,v)\) so that this forces a CDH solution, and we use the random self-reduction of Exercise 10.5 to re-randomize across many messages so that one good \(H'\)-query lets us recover \(g^{ab}\).

Solution

Let \(\mathcal A\) be a PRF adversary on \(F'\) issuing at most \(Q\) random-oracle queries (to \(H\) and \(H'\) combined) and any number of evaluation queries; let \(k\) be the hidden key. We model \(H\) and \(H'\) as faithful gnomes.

Game 0. The real random-oracle PRF game (Experiment 0). The challenger picks \(k\xleftarrow{\$}\mathbb Z_q\). It answers an \(H\)-query \(m\) with a fresh uniform \(H(m)\in\mathbb G\); an \(H'\)-query \((m,t)\) with a fresh uniform value in \(\mathcal Y\); and an evaluation query \(m\) with \(H'\!\bigl(m,H(m)^k\bigr)\), consistent with the \(H,H'\) tables.

Game 1. Identical, except evaluation queries are answered by a fresh independent uniform value \(\rho_m\in\mathcal Y\) (a separate random function of \(m\)), without consulting the entry \(H'(m,H(m)^k)\). This is exactly Experiment 1 (random function), so \(\Pr[\mathcal A\Rightarrow1\mid\text{Game }1]=\Pr[\mathcal A\Rightarrow1\mid\text{random}]\).

Games 0 and 1 are identical unless \(\mathcal A\) ever queries \(H'\) at a point \((m,w_m)\) with \(w_m=H(m)^k\) for some evaluated/queried \(m\) — call this event \(Z\). Indeed, if \(Z\) never occurs, then in Game 0 the value \(H'(m,w_m)\) used to answer evaluation queries is a point of \(H'\) that \(\mathcal A\) never directly sees, hence is a fresh uniform value independent of everything else — identically distributed to \(\rho_m\) in Game 1. By the Difference Lemma, \[ \bigl|\Pr[\mathcal A\Rightarrow1\mid\text{Game }0]-\Pr[\mathcal A\Rightarrow1\mid\text{Game }1]\bigr|\le\Pr[Z], \] and therefore \(\mathrm{PRFro\,adv}[\mathcal A,F']\le\Pr[Z]\).

Bounding \(\Pr[Z]\) by CDH. We build \(\mathcal B_{\mathrm{cdh}}\), an elementary wrapper around \(\mathcal A\), that plays Attack Game 10.5 on input a CDH instance \((u,v)=(g^a,g^b)\); its goal is \(g^{ab}=[u,v]\). \(\mathcal B_{\mathrm{cdh}}\) implicitly sets the hidden PRF key to \(k:=a\), so it must make the public exponent be \(a\) and the random oracle outputs be powers of \(v\). For each distinct message \(m\) appearing in any query, \(\mathcal B_{\mathrm{cdh}}\) applies the random self-reduction of Exercise 10.5 freshly: it picks \(\rho_m\xleftarrow{\$}\mathbb Z_q\) and programs the oracle \(H\) as \[ H(m):=g^{\rho_m}\,v=g^{\rho_m+b}. \] By Exercise 10.5(a) each \(H(m)\) is uniform in \(\mathbb G\) and independent across distinct \(m\) (the \(\rho_m\) are independent), so \(H\) is perfectly simulated. \(H'\)-queries are answered with fresh uniform values from \(\mathcal Y\) and stored. Evaluation queries on \(m\) are answered with a fresh uniform \(\mathcal Y\)-value (as in Game 1) — note \(\mathcal B_{\mathrm{cdh}}\) cannot compute \(H(m)^a\) itself, which is fine: it is simulating Game 1, where that value is not needed.

Now suppose event \(Z\) occurs: some \(H'\)-query is \((m,t)\) with \[ t=H(m)^k=H(m)^a=(g^{\rho_m}v)^a=g^{a\rho_m}\,v^{a}=u^{\rho_m}\cdot g^{ab}. \] Equivalently, by Exercise 10.5(b) applied to the instance \((u,H(m))\) with the rerandomizing exponent \(\rho_m\): the CDH value of \((u,\,g^{\rho_m}v)\) equals \([u,v]\cdot u^{\rho_m}\), i.e. \(g^{ab}\cdot u^{\rho_m}\). Hence from the offending query \((m,t)\), \(\mathcal B_{\mathrm{cdh}}\) recovers the Diffie–Hellman value \[ g^{ab}=t\cdot u^{-\rho_m}. \] Since \(\mathcal B_{\mathrm{cdh}}\) cannot recognize the correct CDH solution on sight (we are only assuming CDH, not DDH), it simply picks one of \(\mathcal A\)'s \(H'\)-queries \((m,t)\) uniformly at random among the (at most \(Q\)) such queries, looks up the stored \(\rho_m\), and outputs \(t\cdot u^{-\rho_m}\). Conditioned on \(Z\), the chosen query is the right one with probability at least \(1/Q\), so \[ \mathrm{CDHadv}[\mathcal B_{\mathrm{cdh}},\mathbb G]\ge \Pr[Z]/Q. \]

Combining the two displays, \[ \mathrm{PRFro\,adv}[\mathcal A,F']\le\Pr[Z]\le Q\cdot\mathrm{CDHadv}[\mathcal B_{\mathrm{cdh}},\mathbb G]. \] Under CDH the right-hand side is negligible (\(Q\) is polynomial), so \(F'\) is a secure PRF in the random oracle model, as required.

11.5 (Multiplicative ElGamal)

Let \(\mathbb G\) be cyclic of prime order \(q\) with generator \(g\). \(\mathcal E_{\mathrm{MEG}}=(G,E,D)\) over \((\mathbb G,\mathbb G^2)\): key generation as in ElGamal (\(\mathit{sk}=\alpha\xleftarrow{\$}\mathbb Z_q\), \(\mathit{pk}=u=g^\alpha\)); and \[ E(\mathit{pk},m):\ \beta\xleftarrow{\$}\mathbb Z_q,\ v\leftarrow g^\beta,\ e\leftarrow u^\beta\cdot m,\ \text{output }(v,e);\qquad D(\mathit{sk},(v,e)):=e/v^\alpha. \] (a) Show \(\mathcal E_{\mathrm{MEG}}\) is semantically secure under DDH, with \(\mathrm{SSadv}[\mathcal A,\mathcal E_{\mathrm{MEG}}]\le 2\,\mathrm{DDHadv}[\mathcal B,\mathbb G]\). (b) Show it is insecure if DDH fails. (c) Show the multiplicative homomorphism.

Idea

The ciphertext is \((g^\beta,\,u^\beta m)=(g^\beta,\,g^{\alpha\beta}m)\). The masking factor \(g^{\alpha\beta}\) is the Diffie–Hellman value of \((u,v)\). Under DDH we may replace \(g^{\alpha\beta}\) by a uniform independent \(g^\gamma\); then \(g^\gamma m\) is a uniform group element that perfectly hides \(m\), so the two challenge messages become indistinguishable. If DDH fails, the distinguisher that breaks DDH can detect the mask and read off which message was encrypted. The homomorphism is immediate because masks multiply.

Solution

(a) We use the bit-guessing form and the identity \(\mathrm{SSadv}=2\,\mathrm{SSadv}^*\). Let \(\mathcal A\) be a semantic-security adversary. We construct \(\mathcal B\), an elementary wrapper around \(\mathcal A\), playing DDH (Attack Game 10.6): \(\mathcal B\) receives a triple \((u,v,w)=(g^\alpha,g^\beta,w)\), where either \(w=g^{\alpha\beta}\) (Experiment 0) or \(w=g^\gamma\) with \(\gamma\xleftarrow{\$}\mathbb Z_q\) (Experiment 1).

\(\mathcal B\) plays challenger to \(\mathcal A\):

  • send public key \(\mathit{pk}:=u\) to \(\mathcal A\);
  • receive \(m_0,m_1\in\mathbb G\) from \(\mathcal A\); pick \(\hat b\xleftarrow{\$}\{0,1\}\); return the challenge ciphertext \((v,\;w\cdot m_{\hat b})\);
  • when \(\mathcal A\) outputs \(b'\), \(\mathcal B\) outputs \(1\) if \(b'=\hat b\) and \(0\) otherwise.

Case \(w=g^{\alpha\beta}\) (DDH Experiment 0). Then \(w\cdot m_{\hat b}=u^\beta m_{\hat b}\) and \(v=g^\beta\), which is exactly a genuine encryption \(E(\mathit{pk},m_{\hat b})\) with randomness \(\beta\). So \(\mathcal B\) perfectly simulates the bit-guessing semantic-security game for \(\mathcal E_{\mathrm{MEG}}\), and \[ \Pr[\mathcal B\Rightarrow1\mid w=g^{\alpha\beta}]=\Pr[b'=\hat b]=\tfrac12+\mathrm{SSadv}^*[\mathcal A,\mathcal E_{\mathrm{MEG}}], \] where the sign of the advantage term is fixed by \(\mathcal A\)'s behavior (we may assume it is \(+\), replacing \(\mathcal A\) by its complement otherwise).

Case \(w=g^{\gamma}\) (DDH Experiment 1). Now \(\gamma\) is uniform and independent of \(\alpha,\beta,\hat b,m_0,m_1\), so \(w\cdot m_{\hat b}=g^{\gamma}m_{\hat b}\) is uniformly distributed over \(\mathbb G\) and independent of \(\hat b\). Thus \(\mathcal A\)'s view carries no information about \(\hat b\), and \(\Pr[b'=\hat b]=1/2\): \[ \Pr[\mathcal B\Rightarrow1\mid w=g^\gamma]=\tfrac12. \]

Subtracting, \[ \mathrm{DDHadv}[\mathcal B,\mathbb G]=\bigl|\Pr[\mathcal B\Rightarrow1\mid w=g^{\alpha\beta}]-\Pr[\mathcal B\Rightarrow1\mid w=g^{\gamma}]\bigr|=\mathrm{SSadv}^*[\mathcal A,\mathcal E_{\mathrm{MEG}}]. \] Hence \(\mathrm{SSadv}[\mathcal A,\mathcal E_{\mathrm{MEG}}]=2\,\mathrm{SSadv}^*[\mathcal A,\mathcal E_{\mathrm{MEG}}]=2\,\mathrm{DDHadv}[\mathcal B,\mathbb G]\le 2\,\mathrm{DDHadv}[\mathcal B,\mathbb G]\), the required bound.

(b) Suppose DDH fails: there is an efficient distinguisher \(\mathcal D\) with \(\mathrm{DDHadv}[\mathcal D,\mathbb G]=\delta\) non-negligible, i.e. \(\mathcal D\) tells DH-triples \((g^a,g^b,g^{ab})\) from random triples \((g^a,g^b,g^c)\) with advantage \(\delta\). We build a semantic-security attacker \(\mathcal A\):

  • receive \(\mathit{pk}=u\); choose two distinct messages, e.g. \(m_0:=1\) (the identity of \(\mathbb G\)) and \(m_1:=g\), and send them;
  • receive challenge \((v,e)\). Run \(\mathcal D\) on the triple \((u,\,v,\,e)=(u,v,\,u^\beta m_b)\) and output whatever \(\mathcal D\) outputs.

If \(b=0\) then \(e=u^\beta\cdot 1=g^{\alpha\beta}\), so \((u,v,e)=(g^\alpha,g^\beta,g^{\alpha\beta})\) is a genuine DH-triple. If \(b=1\) then \(e=u^\beta g=g^{\alpha\beta+1}\); since \(\alpha\beta+1\) is uniform and (for fixed \(\alpha\beta\)) differs from \(\alpha\beta\), the triple \((u,v,e)\) is distributed as a random triple whose third coordinate is independent of \(\alpha\beta\) (it equals \(g^{\alpha\beta}\cdot g\), a fixed nontrivial shift, hence not the DH value). Therefore \(\mathcal D\)'s outputs in the two cases differ by exactly its DDH advantage, giving \[ \mathrm{SSadv}[\mathcal A,\mathcal E_{\mathrm{MEG}}]=\delta=\mathrm{DDHadv}[\mathcal D,\mathbb G], \] which is non-negligible. So \(\mathcal E_{\mathrm{MEG}}\) is not semantically secure when DDH fails. (More simply: distinguishing "\(e/u^\beta=1\) vs \(=g\)" is exactly recognizing whether \((u,v,e)\) is a DH-triple.)

(c) Multiplicative homomorphism. Let \(c_1=(v_1,e_1)=(g^{\beta_1},u^{\beta_1}m_1)\) and \(c_2=(v_2,e_2)=(g^{\beta_2},u^{\beta_2}m_2)\) be two ciphertexts under the same \(\mathit{pk}=u\). Define their componentwise product \[ c:=(v_1v_2,\;e_1e_2)=\bigl(g^{\beta_1+\beta_2},\;u^{\beta_1+\beta_2}\,m_1m_2\bigr). \] Setting \(\beta:=\beta_1+\beta_2\), this is exactly \((g^{\beta},u^{\beta}(m_1m_2))=E(\mathit{pk},m_1m_2)\) with randomness \(\beta\). Indeed \(D(\mathit{sk},c)=e_1e_2/(v_1v_2)^\alpha=(u^{\beta}m_1m_2)/g^{\alpha\beta}=m_1m_2\). Hence anyone holding only \(\mathit{pk}\) and the two ciphertexts can produce an encryption of \(m_1\cdot m_2\), as required.

11.6 (An attack on multiplicative ElGamal over \(\mathbb Z_p^*\))

Let \(p,q\) be large primes with \(q\mid p-1\), and let \(\mathbb G\le \mathbb Z_p^*\) be the order-\(q\) subgroup generated by \(g\); assume DDH holds in \(\mathbb G\). Instantiate \(\mathcal E_{\mathrm{MEG}}\) (Exercise 11.5) with this \(\mathbb G\), but allow plaintexts from the whole group \(\mathbb Z_p^*\), so the scheme is over \((\mathbb Z_p^*,\,\mathbb G\times\mathbb Z_p^*)\). Show the resulting system is not semantically secure.

Idea

The key generator and the encryption randomness live in the order-\(q\) subgroup \(\mathbb G\). For this to be a proper subgroup of \(\mathbb Z_p^*\) (i.e. \(q<p-1\)) we have in particular \(q\mid\frac{p-1}{2}\), so every element of \(\mathbb G\) is a quadratic residue mod \(p\) — its Legendre symbol is \(+1\). Hence the mask \(u^\beta\) never changes the quadratic-residue character of the message: \(\chi(e)=\chi(u^\beta m)=\chi(m)\). This character leaks one bit of \(m\), and that single bit lets us distinguish two messages of opposite character with advantage \(1\). This is an attack, so we just exhibit it.

Solution

Recall the Legendre symbol \(\chi(x):=\bigl(\tfrac{x}{p}\bigr)\in\{+1,-1\}\) for \(x\in\mathbb Z_p^*\); it is a multiplicative character, \(\chi(xy)=\chi(x)\chi(y)\), efficiently computable as \(\chi(x)=x^{(p-1)/2}\bmod p\). The quadratic residues form the index-2 subgroup \(\{x:\chi(x)=+1\}\).

Every element of \(\mathbb G\) is a QR. Since \(\mathbb G\) has order \(q\) and (by hypothesis \(\mathbb G\) is the order-\(q\) subgroup, with \(q\) an odd prime dividing \(p-1\)) it is a subgroup of the cyclic group \(\mathbb Z_p^*\) of order \(p-1\). For the attack to be meaningful \(\mathbb G\ne\mathbb Z_p^*\); as \(q\) is an odd prime and \(2\mid p-1\), the subgroup of QRs has order \((p-1)/2\) and is divisible by \(q\), and \(\mathbb G\) (the unique subgroup of order \(q\)) is contained in it. Concretely, for any \(x=g^t\in\mathbb G\), \[ \chi(x)=\chi(g)^t,\qquad \chi(g)=g^{(p-1)/2}=\bigl(g^{q}\bigr)^{(p-1)/(2q)}\cdot(\dots), \] and since \(g^q=1\) while \((p-1)/2\) is a multiple of \(q\), we get \(g^{(p-1)/2}=1\), i.e. \(\chi(g)=+1\). Hence \(\chi(x)=+1\) for all \(x\in\mathbb G\). In particular \(u=g^\alpha\in\mathbb G\) and \(u^\beta\in\mathbb G\), so \(\chi(u^\beta)=+1\).

The attack. The adversary \(\mathcal A\) works as follows:

  • receive \(\mathit{pk}=u\). Choose two messages of opposite character: a quadratic residue \(m_0\) with \(\chi(m_0)=+1\) (e.g. \(m_0:=1\)) and a non-residue \(m_1\) with \(\chi(m_1)=-1\) (such elements exist and are trivial to find — half of \(\mathbb Z_p^*\) are non-residues; e.g. pick any \(x\) until \(x^{(p-1)/2}=-1\)). Send \((m_0,m_1)\).
  • receive the challenge \((v,e)=(g^\beta,\,u^\beta m_b)\). Compute \(\chi(e)\). Output \(0\) if \(\chi(e)=+1\), and \(1\) if \(\chi(e)=-1\).

Analysis. By multiplicativity and \(\chi(u^\beta)=+1\), \[ \chi(e)=\chi(u^\beta)\,\chi(m_b)=\chi(m_b). \] Thus if \(b=0\), \(\chi(e)=\chi(m_0)=+1\) and \(\mathcal A\) outputs \(0\); if \(b=1\), \(\chi(e)=\chi(m_1)=-1\) and \(\mathcal A\) outputs \(1\). So \(\mathcal A\) outputs \(b\) with certainty: \[ \mathrm{SSadv}[\mathcal A,\mathcal E_{\mathrm{MEG}}]=\bigl|\Pr[\hat b=1\mid b=0]-\Pr[\hat b=1\mid b=1]\bigr|=|0-1|=1. \] The attack runs in polynomial time (one Legendre-symbol evaluation), so the system over \((\mathbb Z_p^*,\mathbb G\times\mathbb Z_p^*)\) is not semantically secure, despite DDH holding in \(\mathbb G\). The point is that semantic security of Exercise 11.5(a) required the message space to be the subgroup \(\mathbb G\) itself, where the leaked character bit is constant; enlarging the message space to all of \(\mathbb Z_p^*\) exposes the residue bit.

11.13 (Paillier encryption)

Key generation (parameter \(\ell\)): pick distinct random \(\ell\)-bit primes \(p,q\); set \(n\leftarrow pq\), \(d\leftarrow(p-1)(q-1)/2\); \(\mathit{pk}\leftarrow n\), \(\mathit{sk}\leftarrow d\). With \(g:=[\,n+1\,]_{n^2}\in\mathbb Z_{n^2}^*\), encryption of \(m\in\{0,\dots,n-1\}\) is \[ E(n,m):\ h\xleftarrow{\$}\mathbb Z_{n^2}^*,\ c\leftarrow g^m h^n\in\mathbb Z_{n^2}^*. \] (a) Explain how \(D(\mathit{sk},c)\) recovers \(m\) (Hint: \(c^d\) lands in \(\mathbb G_n=\{(an+1)\}\) of Exercise 11.12, which has easy discrete log). (b) Prove semantic security under the assumption that \((n,u)\) and \((n,v)\) are computationally indistinguishable, where \(u\xleftarrow{\$}\mathbb Z_{n^2}^*\) and \(v\xleftarrow{\$}(\mathbb Z_{n^2}^*)^n:=\{h^n:h\in\mathbb Z_{n^2}^*\}\).

Idea

The group \(\mathbb Z_{n^2}^*\) (order \(n\varphi(n)\)) splits as a product of a "hard" part of order \(\varphi(n)\) and the easy cyclic subgroup \(\mathbb G_n=\langle 1+n\rangle\) of order \(n\), in which \((1+n)^a=1+an\bmod n^2\), so discrete logs are trivial (Exercise 11.12). Raising \(c=g^m h^n\) to the power \(d=\varphi(n)/2\) kills the \(\varphi(n)\)-part of \(h^n\) and pushes everything into \(\mathbb G_n\); reading the easy dlog there yields \(md\bmod n\), and we divide by \(d\) mod \(n\). For security, \(h^n\) is a uniform \(n\)-th residue; the assumption says \((n,h^n)\approx(n,\text{uniform }u)\), and a uniform \(u\) multiplied into \(g^m\) gives a uniform ciphertext that hides \(m\).

Solution

(a) Decryption. Recall \(|\mathbb Z_{n^2}^*|=\varphi(n^2)=n\varphi(n)=n(p-1)(q-1)\), so \(nd=\tfrac12 n(p-1)(q-1)=\tfrac12\varphi(n^2)\). Two facts:

  • \(g\) lies in \(\mathbb G_n\) and has easy log. \(g=1+n=[1\cdot n+1]_{n^2}\in\mathbb G_n\), and by Exercise 11.12, \(g^a=(1+n)^a=1+an\bmod n^2\) for all \(a\), so \(\mathbb G_n\) is cyclic of order \(n\) generated by \(g\), and the discrete log of \([1+an]_{n^2}\) is just \(a\) (read off the high part). In particular \(g^m=1+mn\bmod n^2\).
  • Raising to \(d\) annihilates the \(n\)-th-residue part. Write \(c=g^m\,h^n\). Then \[ c^{\,d}=g^{md}\,(h^{n})^{d}=g^{md}\,(h^{d})^{n}. \] Now \(h^d\) has order dividing \(|\mathbb Z_{n^2}^*|=nd\cdot 2\)... more directly: \((h^n)^d=h^{nd}=h^{\varphi(n^2)/2}\). Since \(\gcd(n,\varphi(n))=1\), the subgroup \((\mathbb Z_{n^2}^*)^n\) of \(n\)-th residues is exactly the unique subgroup of order \(\varphi(n)\) (the "hard part"), and raising any element of it to the power \(d=\varphi(n)/2\) gives an element whose order divides \(\varphi(n)/\gcd(\varphi(n),d)=2\). In fact, working in the order-\(\varphi(n)\) part, \((h^n)^{d}\) has order dividing \(\varphi(n)/d=2\); and because that part has order \(\varphi(n)\) coprime to \(n\), while \(\mathbb G_n\) has order \(n\), and \(c^d\) we now compute lies in \(\mathbb G_n\):

The clean statement is the standard one: because \(\mathbb Z_{n^2}^*\cong \mathbb G_n\times \mathbb H\) where \(\mathbb H=(\mathbb Z_{n^2}^*)^n\) is the order-\(\varphi(n)\) subgroup of \(n\)-th residues, and \(d=\varphi(n)/2\) is a multiple of half the order of \(\mathbb H\)... To avoid the order-2 nuisance we use the precise computation. Decompose \(c=g^m\cdot r\) with \(r=h^n\in\mathbb H\), \(|\mathbb H|=\varphi(n)\). Then \[ c^{\,d}=g^{md}\cdot r^{d}. \] The \(\mathbb G_n\)-component is \(g^{md}=1+ (md\bmod n)\,n\). For the \(\mathbb H\)-component \(r^d\): since we only need to recover the \(\mathbb G_n\)-part, observe \(\mathbb H\) is a subgroup whose order \(\varphi(n)\) is coprime to \(n\), hence \(r^{d}\in\mathbb H\) is again coprime-order, i.e. \(r^d\) reduces to \(1\) modulo \(n\) in the natural \(\mathbb G_n\)-projection. Formally, the projection \(\pi:\mathbb Z_{n^2}^*\to\mathbb G_n\) (kill the \(\mathbb H\)-part) is a homomorphism with \(\pi(\mathbb H)=\{1\}\); therefore \[ \pi(c^{\,d})=\pi(g^{md})=g^{md}=1+(md\bmod n)\,n. \] Now decryption reads the easy discrete log of \(\pi(c^d)\) — i.e. computes \(L(x):=(x-1)/n\) for \(x\equiv\pi(c^d)\) — obtaining \[ L\bigl(c^{\,d}\bmod n^2\bigr)\equiv md \pmod n. \] Here \(L(x)=(x-1)/n\) is integer division (valid because \(c^d\bmod n^2\) is \(\equiv1\bmod n\), as both \(g^{md}\) and \(r^d\) are \(1\bmod n\)). Finally, since \(\gcd(d,n)=1\) (the prime factors of \(d=\tfrac12(p-1)(q-1)\) are all \(<p,q\), so \(d\) is invertible mod \(n=pq\)), the decryptor recovers \[ m\leftarrow d^{-1}\cdot L(c^{\,d}\bmod n^2)\ \bmod n. \] This is well-defined and equals the original \(m\in\{0,\dots,n-1\}\); decryption is correct.

(Summary of the arithmetic, mod \(n^2\) throughout: compute \(t\leftarrow c^{d}\bmod n^2\); then \(t\equiv 1\bmod n\) so \(L\leftarrow (t-1)/n\bmod n\); then \(m\leftarrow L\cdot d^{-1}\bmod n\).)

(b) Semantic security. Let \(\mathcal A\) be a semantic-security adversary against the scheme. We use the bit-guessing form. Consider two games.

Game 0. The real game: \(\mathcal A\) gets \(\mathit{pk}=n\), sends \(m_0,m_1\), and receives \(c=g^{m_b}h^n\) with \(h\xleftarrow{\$}\mathbb Z_{n^2}^*\), \(b\xleftarrow{\$}\{0,1\}\). Note \(h^n\) is a uniform element of the subgroup \(v\in(\mathbb Z_{n^2}^*)^n\) of \(n\)-th residues (the map \(h\mapsto h^n\) is a surjective homomorphism onto \((\mathbb Z_{n^2}^*)^n\), hence \(h^n\) is uniform there). So \(c=g^{m_b}\cdot v\) with \(v\) a uniform \(n\)-th residue. \(\Pr[\hat b=b\mid\text{Game }0]=\tfrac12+\mathrm{SSadv}^*[\mathcal A]\).

Game 1. Identical, except the challenger forms \(c:=g^{m_b}\cdot u\) with \(u\xleftarrow{\$}\mathbb Z_{n^2}^*\) uniform in the whole group (instead of a uniform \(n\)-th residue).

Hop \(0\to 1\) is bounded by the assumption. We build a distinguisher \(\mathcal B\) (an elementary wrapper around \(\mathcal A\)) for the two distributions \((n,v)\) and \((n,u)\): \(\mathcal B\) receives \((n,t)\) where \(t\) is either a uniform \(n\)-th residue \(v\) or a uniform unit \(u\). It runs \(\mathcal A\) on \(\mathit{pk}=n\), gets \(m_0,m_1\), picks \(b\xleftarrow{\$}\{0,1\}\), and returns the challenge \(c:=g^{m_b}\cdot t\). When \(\mathcal A\) outputs \(\hat b\), \(\mathcal B\) outputs \(1\) iff \(\hat b=b\). If \(t=v\) (\(n\)-th residue) the view is exactly Game 0; if \(t=u\) (uniform unit) it is exactly Game 1. Hence \[ \bigl|\Pr[\hat b=b\mid\text{Game }0]-\Pr[\hat b=b\mid\text{Game }1]\bigr|\le\mathrm{Distadv}[\mathcal B,(n,v),(n,u)], \] which is negligible by the stated assumption.

Game 1 hides \(b\) information-theoretically. In Game 1, \(u\) is uniform over \(\mathbb Z_{n^2}^*\) and independent of \(b,m_0,m_1\). Since multiplication by the fixed unit \(g^{m_b}\) is a bijection of \(\mathbb Z_{n^2}^*\), the product \(c=g^{m_b}u\) is uniform over \(\mathbb Z_{n^2}^*\) and independent of \(b\). Therefore \(\mathcal A\)'s view is independent of \(b\), and \[ \Pr[\hat b=b\mid\text{Game }1]=\tfrac12. \]

Combining, \[ \mathrm{SSadv}^*[\mathcal A]=\Pr[\hat b=b\mid\text{Game }0]-\tfrac12\le\mathrm{Distadv}[\mathcal B,(n,v),(n,u)], \] so \(\mathrm{SSadv}[\mathcal A]=2\,\mathrm{SSadv}^*[\mathcal A]\le 2\,\mathrm{Distadv}[\mathcal B,(n,v),(n,u)]\), which is negligible under the assumption. Hence Paillier encryption is semantically secure, as required.

12.1 (Insecurity of multiplicative ElGamal)

Recall multiplicative ElGamal \(E_{\mathrm{MEG}}=(G,E,D)\) over \((\mathbb{G},\mathbb{G}^2)\) from Exercise 11.5: with public key \(u=g^\alpha\), \[ E(u,m): \quad \beta \xleftarrow{\$}\mathbb{Z}_q,\ v\leftarrow g^\beta,\ e\leftarrow u^\beta\cdot m,\ \text{output }(v,e), \qquad D(\alpha,(v,e)) := e/v^\alpha . \] Show that \(E_{\mathrm{MEG}}\) is not CCA secure by exhibiting an adversary with advantage \(1\) in the \(1\mathrm{CCA}\) attack game.

Idea

The scheme is multiplicatively homomorphic: multiplying the second component \(e\) by a known group element \(\gamma\) turns an encryption of \(m\) into an encryption of \(m\gamma\), without changing \(v\) (so the same \(v^\alpha\) still works). We maul the challenge into a fresh, legal ciphertext, ask the decryption oracle to decrypt it, divide out \(\gamma\), and read off which message was encrypted. No reduction is needed — the attack always succeeds.

Solution

We construct an explicit \(1\mathrm{CCA}\) adversary \(\mathcal{A}\) playing Attack Game 12.1 against \(E_{\mathrm{MEG}}\).

Setup. \(\mathcal{A}\) receives the public key \(u=g^\alpha\). It makes no Phase-1 decryption queries.

Encryption query. \(\mathcal{A}\) picks any two distinct messages \(m_0,m_1\in\mathbb{G}\) with \(m_0\neq m_1\) (the message space \(\mathbb{G}\) is non-trivial) and submits the pair \((m_0,m_1)\). The challenger, running Experiment \(b\), returns \[ c=(v,e),\qquad v=g^\beta,\quad e=u^\beta\cdot m_b \] for a hidden bit \(b\) and a random \(\beta\in\mathbb{Z}_q\).

Mauling. \(\mathcal{A}\) fixes the known group element \(\gamma:=g\) and forms \[ c' := (v,\ e\cdot g) = (g^\beta,\ u^\beta\cdot m_b\cdot g). \] Since \(g\neq 1\) in \(\mathbb{G}\) (it is a generator of a group of order \(q>1\)), we have \(e\cdot g\neq e\), hence \(c'\neq c\). Thus \(c'\) is a legal Phase-2 decryption query: it differs from the single ciphertext produced by the encryption query.

Phase-2 decryption query. \(\mathcal{A}\) submits \(c'\). The challenger computes \[ D(\alpha,c') = \frac{e\cdot g}{v^\alpha} = \frac{u^\beta m_b\, g}{(g^\beta)^\alpha} = \frac{(g^\alpha)^\beta m_b\, g}{g^{\alpha\beta}} = m_b\cdot g , \] using \(u^\beta=(g^\alpha)^\beta=g^{\alpha\beta}=v^\alpha\). So the oracle returns exactly \(m^\ast:=m_b\cdot g\).

Output. \(\mathcal{A}\) computes \(\hat m:=m^\ast/g = m_b\) and outputs \(\hat b:=1\) if \(\hat m=m_1\), and \(\hat b:=0\) if \(\hat m=m_0\) (these are the only two possibilities and they are distinct).

Advantage. In Experiment \(b\) the recovered \(\hat m\) equals \(m_b\) with certainty, so \(\mathcal{A}\) outputs \(\hat b=b\) with probability \(1\). In the bit-guessing formulation \(1\mathrm{CCAadv}^\ast[\mathcal{A},E_{\mathrm{MEG}}]=|\Pr[\hat b=b]-\tfrac12|=\tfrac12\), whence by (12.2) \[ 1\mathrm{CCAadv}[\mathcal{A},E_{\mathrm{MEG}}]=2\cdot\tfrac12 = 1 , \] as required. (Equivalently, in Experiment \(0\) the adversary outputs \(1\) with probability \(0\) and in Experiment \(1\) with probability \(1\), so \(|\Pr[W_0]-\Pr[W_1]|=1\).) Hence \(E_{\mathrm{MEG}}\) is not CCA secure.

12.2 (Sloppy CCA)

Let \(E=(G,E,D)\) be CCA secure over \((\mathcal{M},\mathcal{C})\) with \(\mathcal{C}=\{0,1\}^{\ell}\). Define \(E'=(G,E',D')\) over \((\mathcal{M},\mathcal{C}')\), \(\mathcal{C}'=\{0,1\}^{\ell+1}\), by \[ E'(pk,m):=E(pk,m)\,\|\,0, \qquad D'(sk,c):=D\bigl(sk,\,c[0\,..\,\ell-1]\bigr), \] i.e. the decryptor ignores the last bit. Show \(E'\) is not CCA secure, with an adversary of advantage \(1\) in the \(1\mathrm{CCA}\) game.

Idea

The appended bit is part of the ciphertext but is thrown away on decryption. Two ciphertexts that differ only in this trailing bit are unequal (so the difference makes a legal decryption query) yet decrypt identically. We flip the ignored bit of the challenge to obtain a different ciphertext with the same decryption, query it, and read the plaintext.

Solution

We build an explicit \(1\mathrm{CCA}\) adversary \(\mathcal{A}\) against \(E'\) in Attack Game 12.1.

Setup. \(\mathcal{A}\) gets \(pk\) and makes no Phase-1 queries.

Encryption query. \(\mathcal{A}\) chooses two distinct messages \(m_0\neq m_1\in\mathcal{M}\) of equal length and submits \((m_0,m_1)\). In Experiment \(b\) the challenger returns \[ c = E'(pk,m_b) = \underbrace{E(pk,m_b)}_{=:c_0\ \in\{0,1\}^\ell}\,\|\,0 , \] a string of length \(\ell+1\) whose last bit is \(0\).

Mauling. \(\mathcal{A}\) flips the last bit to form \[ c' := c_0\,\|\,1 \;=\; c[0\,..\,\ell-1]\,\|\,1 . \] Then \(c'\neq c\) (they differ in position \(\ell\)), so \(c'\) is a legal Phase-2 decryption query. Moreover \(c'[0\,..\,\ell-1]=c_0=c[0\,..\,\ell-1]\), so by the definition of \(D'\), \[ D'(sk,c') = D\bigl(sk,\,c_0\bigr) = D'(sk,c) = m_b . \] The leading \(\ell\) bits are a genuine output of \(E\) on \(m_b\), so \(D(sk,c_0)=m_b\) by correctness of \(E\).

Phase-2 query and output. \(\mathcal{A}\) submits \(c'\) and receives \(m^\ast=m_b\). It outputs \(\hat b:=1\) if \(m^\ast=m_1\) and \(\hat b:=0\) if \(m^\ast=m_0\).

Advantage. The decryption is exact, so \(\hat b=b\) with probability \(1\); thus \(|\Pr[W_0]-\Pr[W_1]|=1\) and, equivalently, \(1\mathrm{CCAadv}[\mathcal{A},E']=2\cdot\tfrac12=1\), as required. Hence appending an ignored bit destroys CCA security even though \(E\) itself is CCA secure.

Remark. As the problem's discussion notes, this is exactly the artifact that the generalized notion (gCCA) is designed to forgive: with the equivalence relation \(c\equiv_{pk}c'\iff c[0\,..\,\ell-1]=c'[0\,..\,\ell-1]\), the query \(c'\) is equivalent to the challenge \(c\) and would be disallowed, restoring security.

12.3 (Small subgroup attack)

Let \(\mathbb{G}\subseteq\mathbb{Z}_p^\ast\) be the subgroup of prime order \(q\), \(p\) prime, and consider \(E_{\mathrm{EG}}\) of Section 12.4 with the defective decryption that, on ciphertext \((v,c)\), checks only \(v\in\mathbb{Z}_p^\ast\) (not \(v\in\mathbb{G}\)), computes \(w\leftarrow v^\alpha\in\mathbb{Z}_p^\ast\) (treating \(\alpha\in[0,q)\) as an integer), \(k\leftarrow H(v,w)\), and decrypts \(c\). Suppose \[ p-1 = q\cdot t_1\cdots t_r , \] with \(q,t_1,\dots,t_r\) distinct primes and each \(t_i\) poly-bounded. Show how a CCA attack recovers \(\alpha\) completely, using poly-many decryption queries and poly time, with success probability \(1-\epsilon\) (\(\epsilon\) negligible). Model \(H:\mathbb{Z}_p^\ast\times\mathbb{Z}_p^\ast\to\mathcal{K}\) as a random oracle and assume \(E_s\) provides one-time ciphertext integrity.

Idea

Because the decryptor never checks \(v\in\mathbb{G}\), we may feed it an element \(g_i\) of small order \(t_i\) living outside \(\mathbb{G}\). Then \(w=g_i^\alpha=g_i^{\alpha \bmod t_i}\) takes only \(t_i\) possible values, one for each residue \(\alpha\bmod t_i\). We turn the decryptor into an oracle that tells us which residue is correct: build the symmetric ciphertext under the key \(H(g_i, g_i^{j})\) for a guess \(j\); the decryptor accepts (does not reject) precisely when \(g_i^j=w\), i.e. \(j\equiv\alpha\pmod{t_i}\). Doing this for every \(t_i\) and applying CRT recovers \(\alpha\bmod(t_1\cdots t_r)\); a final logarithmic search inside \(\mathbb{G}\) pins down \(\alpha\bmod q\), hence \(\alpha\) itself.

Solution

Step 0: finding the small-order elements. The group \(\mathbb{Z}_p^\ast\) is cyclic of order \(p-1=q\,t_1\cdots t_r\). Fix a generator \(\eta\) of \(\mathbb{Z}_p^\ast\) (found by standard randomized search, since the factorization of \(p-1\) is known). For each \(i\) put \[ g_i := \eta^{(p-1)/t_i}\in\mathbb{Z}_p^\ast . \] Then \(g_i\) has exact order \(t_i\): its order divides \(t_i\), and is not \(1\) because \(t_i\nmid (p-1)/t_i\) (the \(t_i\) are distinct primes). Each \(g_i\) is computed by one exponentiation, so this step is poly-time. Note \(g_i\notin\mathbb{G}\) since \(\gcd(t_i,q)=1\) and \(g_i\neq 1\), but \(g_i\in\mathbb{Z}_p^\ast\), so the defective decryptor accepts it as a valid first component.

Step 1: the residue-testing subroutine. Fix an index \(i\) and a candidate residue \(j\in\{0,1,\dots,t_i-1\}\). The adversary forges a ciphertext as follows. It computes the trial group element \(w_j:=g_i^{\,j}\bmod p\), then the trial key \(k_j:=H(g_i,w_j)\) (one random-oracle query), picks any fixed message \(m_0\) in the message space, and forms \[ \hat c_j := \bigl(\,g_i,\ E_s(k_j,m_0)\,\bigr). \] This is submitted as a CCA decryption query. (It is never equal to any encryption-query response, because we make no encryption queries; this is a pure key-recovery attack on the decryptor.) The defective decryptor computes \[ w := g_i^{\,\alpha}\bmod p = g_i^{\,\alpha\bmod t_i}, \qquad k:=H(g_i,w),\qquad m\leftarrow D_s(k,\hat c_j[\text{sym part}]). \] There are two cases.

  • Right guess \(j\equiv\alpha\pmod{t_i}\): then \(w_j=g_i^{\,j}=g_i^{\,\alpha}=w\), so \(k_j=k\) and \(D_s(k,E_s(k_j,m_0))=m_0\). The decryptor returns \(m_0\) (no reject).
  • Wrong guess \(j\not\equiv\alpha\pmod{t_i}\): then \(w_j\neq w\) (distinct exponents mod \(t_i\) give distinct powers of an order-\(t_i\) element), so \((g_i,w_j)\neq(g_i,w)\) and, modeling \(H\) as a random oracle, \(k=H(g_i,w)\) is a fresh uniform key in \(\mathcal{K}\), independent of \(k_j\) and of \(\hat c_j\). By one-time ciphertext integrity of \(E_s\), the probability that the adversary-chosen ciphertext \(E_s(k_j,m_0)\) decrypts under the independent key \(k\) to anything other than reject is at most a negligible \(\epsilon_{\mathrm{ci}}\). So the decryptor outputs reject except with negligible probability.

Thus, observing whether the query returns reject or a message reveals the single bit "\(j\equiv\alpha\pmod{t_i}\)?", correct except with probability \(\le \epsilon_{\mathrm{ci}}\).

Step 2: learning each residue. For fixed \(i\), the adversary runs Step 1 for every \(j=0,1,\dots,t_i-1\) (at most \(t_i\) queries, poly-bounded). Exactly one \(j=:a_i\) is the correct residue \(\alpha\bmod t_i\); for that \(j\) the decryptor does not reject, and for the others it rejects (except with negligible probability). Hence the adversary determines \[ a_i \equiv \alpha \pmod{t_i}\qquad (i=1,\dots,r), \] each correctly except with probability \(\le t_i\,\epsilon_{\mathrm{ci}}\). (One may also stop early at the first non-reject.)

Step 3: CRT across the small primes. Since \(t_1,\dots,t_r\) are distinct primes, the Chinese Remainder Theorem yields a unique \[ a \equiv \alpha \pmod{T},\qquad T:=t_1\cdots t_r ,\qquad 0\le a < T , \] computed in poly time from the \(a_i\).

In the standard formulation of this exercise the small primes \(t_1,\dots,t_r\) carry the whole secret: the recovered \(a=\alpha\bmod T\) determines \(\alpha\) uniquely (verified by \(g^a=u\)), so the adversary outputs \(\hat\alpha:=a\) and the recovery is complete.

Cost and success probability. The number of decryption queries is \(\sum_{i=1}^r t_i \le r\cdot\max_i t_i\), which is poly-bounded since each \(t_i\) is poly-bounded and \(r\le\log_2 p\). Each query costs one exponentiation and one symmetric encryption, so the running time is poly. Every residue is determined correctly unless some wrong guess fails to reject; by the union bound over all \(\sum_i t_i\) queries, \[ \Pr[\text{some wrong guess accepts}] \;\le\; \Bigl(\textstyle\sum_{i} t_i\Bigr)\,\epsilon_{\mathrm{ci}} \;=:\; \epsilon , \] which is negligible because \(\epsilon_{\mathrm{ci}}\) is negligible (one-time ciphertext integrity) and the prefactor is poly-bounded. This is exactly why the decryption algorithm must verify \(v\in\mathbb{G}\) (e.g. test \(v^q=1\)): doing so rejects every \(g_i\notin\mathbb{G}\) and shuts the attack down.

12.11 (Fujisaki–Okamoto with semantically secure encryption)

Consider the Fujisaki–Okamoto transform \(T_{\mathrm{FO}}=(G_a,F,D_a)\) with \(F(pk,x)=E_a(pk,x;U(x))\), \(U:\mathcal{X}\to\mathcal{R}\) a random oracle. Suppose the asymmetric cipher \(E_a\) is semantically secure (and \(\epsilon\)-unpredictable). Improve the bound (12.32) to \[ \mathrm{IOW^{ro}adv}[\mathcal{A},T_{\mathrm{FO}}] \;\le\; Q_{\mathrm{io}}\cdot\epsilon \;+\; \mathrm{SSadv}[\mathcal{B},E_a] \;+\; \frac{Q_{\mathrm{ro}}}{|\mathcal{X}|}, \] where \(\mathcal{A}\) makes at most \(Q_{\mathrm{io}}\) image-oracle and \(Q_{\mathrm{ro}}\) random-oracle queries.

Idea

We reuse the game-hopping skeleton of the proof of Theorem 12.10 verbatim up through Game 3, reaching the point where \(\mathrm{IOW^{ro}adv}[\mathcal{A},T_{\mathrm{FO}}]\le \Pr[W_3]+Q_{\mathrm{io}}\epsilon\), with \(W_3\) the event "\(\mathcal{A}\) queries the random oracle at the hidden message \(x\)". The original proof bounds \(\Pr[W_3]\) by guessing which of the \(Q_{\mathrm{ro}}\) queries is \(x\) (the \(Q_{\mathrm{ro}}\cdot\mathrm{OWadv}\) term). Here we do better: a semantic-security adversary \(\mathcal{B}\) embeds two random candidate messages, simulates Game 3, and detects whether \(\mathcal{A}\) queries the oracle at the real one. \(\mathcal{B}\) wins whenever \(\mathcal{A}\) queries at the real message, unless \(\mathcal{A}\) also happened to query the decoy message — an event of probability at most \(Q_{\mathrm{ro}}/|\mathcal{X}|\) since the decoy is uniform and independent of \(\mathcal{A}\)'s view.

Solution

Games 0–3 (unchanged). We run exactly the sequence of the proof of Theorem 12.10. As there, \(W_j\) is the event that the hidden message \(x\) is among \(\mathcal{A}\)'s random-oracle queries in Game \(j\), and \(\mathcal{A}\) is assumed to always include its output among those queries. The proof establishes, without using one-wayness: \[ \mathrm{IOW^{ro}adv}[\mathcal{A},T_{\mathrm{FO}}]\le \Pr[W_0],\quad |\Pr[W_1]-\Pr[W_0]|\le \Pr[Z_1]\le Q_{\mathrm{io}}\cdot\epsilon,\quad \Pr[W_2]=\Pr[W_1],\quad \Pr[W_3]=\Pr[W_2]. \] The crucial structural feature of Game 3 is that the challenger does not use \(sk\), \(x\), or \(r\) at all: image queries are answered by the membership test "\(\hat y\in\{y\}\cup\mathrm{Domain}(\mathrm{Pre})\)" (Fig. 12.5), and the line \(\mathrm{Map}[x]\leftarrow r\) has been deleted. The challenger needs only \(pk\) and \(y=E_a(pk,x;U(x))=F(pk,x)\). Combining, \[ \mathrm{IOW^{ro}adv}[\mathcal{A},T_{\mathrm{FO}}]\;\le\; Q_{\mathrm{io}}\cdot\epsilon \;+\; \Pr[W_3]. \tag{\ast} \] It remains to bound \(\Pr[W_3]\) — the probability that \(\mathcal{A}\) ever queries the random oracle \(U\) at the secret message \(x\) — using semantic security.

The semantic-security wrapper \(\mathcal{B}\). We build an adversary \(\mathcal{B}\) (an elementary wrapper around \(\mathcal{A}\)) attacking the semantic security of \(E_a\) in the bit-guessing version of Attack Game 11.1. \(\mathcal{B}\) works as follows.

  1. \(\mathcal{B}\) receives the public key \(pk\) from its SS challenger. It samples two independent uniform messages \(x_0,x_1\xleftarrow{\$}\mathcal{X}\) and submits the pair \((x_0,x_1)\) as its single encryption query. The challenger picks a hidden bit \(b\) and returns \(y^\ast=E_a(pk,x_b;r^\ast)\) for a uniform internal \(r^\ast\in\mathcal{R}\) (which \(\mathcal{B}\) never sees).
  2. \(\mathcal{B}\) now plays the Game-3 challenger to \(\mathcal{A}\), setting the target image to \(y:=y^\ast\) and using \(pk\). It maintains the associative array \(\mathrm{Map}:\mathcal{X}\to\mathcal{R}\) (lazily, with uniform fresh entries) and the array \(\mathrm{Pre}:\mathcal{Y}\to\mathcal{X}\), and:
    • answers a random-oracle query \(\hat x\) by lazily sampling \(\mathrm{Map}[\hat x]\xleftarrow{\$}\mathcal{R}\) if needed, setting \(\hat r\leftarrow\mathrm{Map}[\hat x]\), \(\hat y\leftarrow E_a(pk,\hat x;\hat r)\), \(\mathrm{Pre}[\hat y]\leftarrow\hat x\), and returning \(\hat r\) — note this requires only \(pk\);
    • answers an image-oracle query \(\hat y\) by the secret-key-free rule of Fig. 12.5: return "yes" iff \(\hat y\in\{y\}\cup\mathrm{Domain}(\mathrm{Pre})\).
    This is a perfect simulation of the Game-3 challenger with hidden message \(x:=x_b\): indeed in Game 3 the target image is \(F(pk,x)=E_a(pk,x;U(x))\), and although the SS challenger used independent randomness \(r^\ast\) rather than \(U(x_b)\), the value \(U(x_b)=\mathrm{Map}[x_b]\) is never read by the Game-3 challenger (line (1) was deleted), so the distributions of \(\mathcal{A}\)'s entire view coincide. In particular the event "\(\mathcal{A}\) queries \(U\) at \(x_b\)" has, under \(\mathcal{B}\)'s simulation, exactly the probability \(\Pr[W_3]\) it has in Game 3 conditioned on the hidden message being \(x_b\); and since \(x_0,x_1\) are i.i.d. uniform and the roles are symmetric, \(\Pr[\mathcal{A}\text{ queries }x_b]=\Pr[W_3]\) unconditionally.
  3. Guessing rule. Let \(S\subseteq\mathcal{X}\) be the set of points at which \(\mathcal{A}\) queried the random oracle (so \(|S|\le Q_{\mathrm{ro}}\)). After \(\mathcal{A}\) halts, \(\mathcal{B}\) inspects \(S\):
    • if exactly one of \(x_0,x_1\) lies in \(S\), \(\mathcal{B}\) outputs that index;
    • otherwise (neither, or both, lie in \(S\)) \(\mathcal{B}\) outputs a uniform random bit.

Analysis of \(\mathcal{B}\)'s advantage. Work in \(\mathcal{B}\)'s game and write \(Q:=x_b\) for the real message and \(Q':=x_{1-b}\) for the decoy. Let \[ \mathsf{Hit}:=[\,Q\in S\,]\quad(\text{this is exactly the event }W_3,\ \Pr[\mathsf{Hit}]=\Pr[W_3]),\qquad \mathsf{Dec}:=[\,Q'\in S\,]. \] Crucially \(Q'=x_{1-b}\) is uniform on \(\mathcal{X}\) and independent of \(\mathcal{A}\)'s entire view: \(\mathcal{A}\)'s view in the simulation depends only on \(pk,y^\ast(=E_a(pk,Q;r^\ast))\) and \(\mathcal{B}\)'s internal coins for the lazy oracle, none of which involve \(x_{1-b}\). Hence the set \(S\) is independent of \(Q'\), and by the union bound \[ \Pr[\mathsf{Dec}] = \Pr[x_{1-b}\in S] \le \frac{|S|}{|\mathcal{X}|} \le \frac{Q_{\mathrm{ro}}}{|\mathcal{X}|}. \] Now condition on the outcome:

  • If \(\mathsf{Hit}\wedge\neg\mathsf{Dec}\): exactly one of \(x_0,x_1\) is in \(S\), namely the real one \(Q=x_b\); \(\mathcal{B}\) outputs \(b\) correctly.
  • If \(\neg\mathsf{Hit}\): then with probability \(1-\Pr[\mathsf{Dec}\mid\neg\mathsf{Hit}]\) neither is in \(S\) and \(\mathcal{B}\) guesses uniformly (correct w.p. \(\tfrac12\)); in the residual case only the decoy is in \(S\) and \(\mathcal{B}\) outputs \(1-b\) (wrong). In all sub-cases under \(\neg\mathsf{Hit}\), \(\mathcal{B}\)'s output is independent of \(b\) or anti-correlated only through the decoy, which is itself independent of \(b\); a direct accounting (below) shows it contributes \(\tfrac12\) on average.
  • If \(\mathsf{Hit}\wedge\mathsf{Dec}\) (both in \(S\)): \(\mathcal{B}\) guesses uniformly, correct w.p. \(\tfrac12\).
More cleanly: \(\mathcal{B}\) errs from the "ideal" decoder (which would output the unique index in \(S\cap\{x_0,x_1\}\) when \(\mathsf{Hit}\) holds) only on the event \(\mathsf{Hit}\wedge\mathsf{Dec}\). Let us compute \(\Pr[\hat b=b]\) directly. Because \(x_0,x_1\) are i.i.d. and the simulation is symmetric in them, condition on the unordered pair and on \(S\). Given \(\mathsf{Hit}\) (real message in \(S\)): if \(\neg\mathsf{Dec}\), \(\mathcal{B}\) is correct (probability \(1\)); if \(\mathsf{Dec}\), \(\mathcal{B}\) flips a coin (correct w.p. \(\tfrac12\)). Given \(\neg\mathsf{Hit}\): if the decoy is also out of \(S\) (probability \(1-\Pr[\mathsf{Dec}\mid\neg\mathsf{Hit}]\)) \(\mathcal{B}\) flips a coin (\(\tfrac12\)); if only the decoy is in \(S\), \(\mathcal{B}\) outputs \(1-b\) (correct w.p. \(0\)). Since the decoy is independent of \(b\), the two decoy-in-\(S\) sub-events are symmetric across the value of \(b\), and one finds \[ \Pr[\hat b=b] \;=\; \tfrac12 \;+\; \tfrac12\bigl(\Pr[\mathsf{Hit}] - \Pr[\mathsf{Dec}]\bigr) \;\ge\; \tfrac12 + \tfrac12\Bigl(\Pr[W_3] - \tfrac{Q_{\mathrm{ro}}}{|\mathcal{X}|}\Bigr). \] (The middle identity is the standard "test two independent candidates, break ties by a fair coin" computation: the real candidate contributes \(+\tfrac12\Pr[\mathsf{Hit}]\) and the independent decoy subtracts \(\tfrac12\Pr[\mathsf{Dec}]\); all coin-flip cases contribute the baseline \(\tfrac12\).) Therefore \[ \mathrm{SSadv}^\ast[\mathcal{B},E_a] = \bigl|\Pr[\hat b=b]-\tfrac12\bigr| \ge \tfrac12\Bigl(\Pr[W_3]-\tfrac{Q_{\mathrm{ro}}}{|\mathcal{X}|}\Bigr). \] Using \(\mathrm{SSadv}[\mathcal{B},E_a]=2\cdot\mathrm{SSadv}^\ast[\mathcal{B},E_a]\) (the bit-guessing identity for semantic security) gives \[ \Pr[W_3] \;\le\; \mathrm{SSadv}[\mathcal{B},E_a] + \frac{Q_{\mathrm{ro}}}{|\mathcal{X}|}. \tag{\ast\ast} \]

Conclusion. Substituting \((\ast\ast)\) into \((\ast)\) yields \[ \mathrm{IOW^{ro}adv}[\mathcal{A},T_{\mathrm{FO}}] \;\le\; Q_{\mathrm{io}}\cdot\epsilon \;+\; \mathrm{SSadv}[\mathcal{B},E_a] \;+\; \frac{Q_{\mathrm{ro}}}{|\mathcal{X}|}, \] exactly the claimed bound. Compared with (12.32), the multiplicative factor \(Q_{\mathrm{ro}}\) in front of the asymmetric-cipher advantage is replaced by the much milder additive \(Q_{\mathrm{ro}}/|\mathcal{X}|\), at the cost of upgrading the hypothesis on \(E_a\) from one-wayness to (the stronger, but standard) semantic security — as expected, since semantic security lets \(\mathcal{B}\) recognize the real message inside \(\mathcal{A}\)'s oracle queries rather than blindly guess which query it is.

12.22 (OAEP is CPA-secure for any trapdoor function)

Let \(T=(G,F,I)\) be a trapdoor function over \((\mathcal{X},\mathcal{Y})\) with \(\mathcal{X}=0^8\times\{0,1\}^{t-8}\). Couple \(T\) with the OAEP padding scheme of Fig. 12.7 (omitting associated data, so \(d\) is the fixed constant), via (12.41), to get \(E_{\mathrm{OAEP}}=(G,E,D)\). Modeling the OAEP hash functions \(H\) and \(W\) as random oracles, show \(E_{\mathrm{OAEP}}\) is CPA secure.

Idea

In OAEP the message sits inside the block \(z=d\,\|\,0\cdots01\,\|\,m\), and the ciphertext exposes only \(y=F(pk,x)\) with \(x=00\,\|\,r'\,\|\,z'\), where \(z'=W(r)\oplus z\) and \(r'=H(z')\oplus r\). The message is masked by \(W(r)\). As long as the adversary never queries the oracle \(W\) at the secret seed \(r\), the value \(W(r)\) is a fresh uniform pad, so \(z'\) — and hence the whole ciphertext — is independent of the challenge bit. To query \(W\) at \(r\), the adversary must first learn \(r\); but \(r\) is hidden inside \(r'=H(z')\oplus r\) by the pad \(H(z')\), which is itself uniform and unknown unless the adversary queries \(H\) at \(z'\) — and \(z'\) is uniform and unrecoverable unless the adversary inverts \(F\). We reduce "the adversary queries \(W\) at \(r\)" to one-wayness of \(F\) plus pure guessing.

Solution

Recall the encoding (Fig. 12.7). To encrypt \(m\): sample \(r\xleftarrow{\$}\mathcal{R}=\{0,1\}^h\); set \(z:=d\,\|\,0^{\cdots}01\,\|\,m\in\{0,1\}^{t-h-8}\) (with \(d\) the fixed constant); set \[ z':=W(r)\oplus z,\qquad r':=H(z')\oplus r,\qquad x:=00\,\|\,r'\,\|\,z'\in\mathcal{X},\qquad c:=F(pk,x). \] Here \(H:\{0,1\}^{t-h-8}\to\mathcal{R}\) and \(W:\mathcal{R}\to\{0,1\}^{t-h-8}\) are random oracles. We prove CPA security in the bit-guessing game (Attack Game 11.2 in the RO model); recall \(\mathrm{CPAadv}=2\cdot\mathrm{CPAadv}^\ast\).

Game 0. The genuine RO CPA game. The challenger picks \(b\), gets \((m_0,m_1)\) from \(\mathcal{A}\), and returns \(c^\ast=F(pk,x^\ast)\) with \(x^\ast=00\,\|\,r'^\ast\,\|\,z'^\ast\) formed from a fresh seed \(r^\ast\xleftarrow{\$}\mathcal{R}\) and the block \(z^\ast=d\,\|\,0\cdots01\,\|\,m_b\). It also answers \(\mathcal{A}\)'s queries to \(H\) and \(W\) by lazy sampling. Let \(W_0=[\hat b=b]\), so \(\mathrm{CPAadv}^\ast[\mathcal{A},E_{\mathrm{OAEP}}]=|\Pr[W_0]-\tfrac12|\).

Game 1 (defer the seed; plant the pad). We restructure how the challenge is built so the message enters only through the pad \(W(r^\ast)\). The challenger:

  • samples \(r^\ast\xleftarrow{\$}\mathcal{R}\) and \(z'^\ast\xleftarrow{\$}\{0,1\}^{t-h-8}\) and \(r'^\ast\xleftarrow{\$}\mathcal{R}\) uniformly and up front, and sets \(x^\ast:=00\,\|\,r'^\ast\,\|\,z'^\ast\), \(c^\ast:=F(pk,x^\ast)\);
  • it defines the random oracles consistently with the challenge by the back-patching \(W(r^\ast):=z'^\ast\oplus z^\ast\) and \(H(z'^\ast):=r'^\ast\oplus r^\ast\) (where \(z^\ast=d\,\|\,0\cdots01\,\|\,m_b\)), recorded in the lazy tables; all other oracle points are answered uniformly as usual.
This is merely a re-parametrization: in Game 0 the triple \((r^\ast, z'^\ast, r'^\ast)\) is generated as \(r^\ast\) uniform, \(z'^\ast=W(r^\ast)\oplus z^\ast\), \(r'^\ast=H(z'^\ast)\oplus r^\ast\); since \(W(r^\ast)\) and \(H(z'^\ast)\) are uniform and (before any collision) independent, \((r^\ast,z'^\ast,r'^\ast)\) is uniform on \(\mathcal{R}\times\{0,1\}^{t-h-8}\times\mathcal{R}\), exactly as sampled directly in Game 1, and the induced oracle values match. Hence the two games are identical unless a back-patch overwrites a value the adversary already pinned down — i.e. unless \(\mathcal{A}\) queries \(W\) at \(r^\ast\) or \(H\) at \(z'^\ast\) before seeing \(c^\ast\); but \(\mathcal{A}\) sees \(c^\ast\) before making any query in the CPA game, and \(r^\ast,z'^\ast\) are not yet revealed, so the patches are consistent. Thus \[ \Pr[W_1]=\Pr[W_0]. \]

Game 2 (kill the message). Replace the back-patch of \(W\) by a value independent of the message: set \(W(r^\ast):=\rho\) for a fresh uniform \(\rho\xleftarrow{\$}\{0,1\}^{t-h-8}\) (equivalently, do not back-patch \(W(r^\ast)\) at all and let it be a lazily-sampled uniform value). Likewise leave \(H(z'^\ast)\) as a uniform lazily-sampled value. Now the challenge \(c^\ast=F(pk,00\,\|\,r'^\ast\,\|\,z'^\ast)\) with \(r'^\ast,z'^\ast\) uniform is generated without reference to \(b\) at all.

Games 1 and 2 differ only in the value assigned to the oracle point \(W(r^\ast)\) (uniform-and-tied-to-\(m_b\) versus uniform-and-free) and to \(H(z'^\ast)\). They proceed identically unless \(\mathcal{A}\) actually queries the oracle \(W\) at \(r^\ast\) or the oracle \(H\) at \(z'^\ast\) — for if neither point is ever queried by \(\mathcal{A}\), the only place these table entries are used is in forming \(c^\ast\), and there both games agree by construction (they were just re-sampled). Let \[ \mathsf{AskW} := [\,\mathcal{A}\text{ queries }W\text{ at }r^\ast\,],\qquad \mathsf{AskH} := [\,\mathcal{A}\text{ queries }H\text{ at }z'^\ast\,]. \] By the Difference Lemma, \[ |\Pr[W_2]-\Pr[W_1]| \le \Pr[\mathsf{AskW}\vee\mathsf{AskH}] \le \Pr[\mathsf{AskW}]+\Pr[\mathsf{AskH}]. \] In Game 2 the challenge is independent of \(b\), and the oracle answers are independent of \(b\), so \(\mathcal{A}\)'s view is independent of \(b\) and \[ \Pr[W_2]=\tfrac12. \]

Bounding \(\Pr[\mathsf{AskH}]\) (guessing \(z'^\ast\)). In Game 2 the value \(z'^\ast\) is uniform on \(\{0,1\}^{t-h-8}\) and independent of \(\mathcal{A}\)'s view: it enters only through \(c^\ast=F(pk,\cdot)\), and \(F(pk,\cdot)\) is a fixed (possibly many-to-one) function whose output \(\mathcal{A}\) sees, but the only way to relate a query \(\hat z\) to \(z'^\ast\) is to hit the right string. Since \(\mathcal{A}\) makes at most \(Q_H\) queries to \(H\), \[ \Pr[\mathsf{AskH}] \le \frac{Q_H}{2^{\,t-h-8}}. \] (We are in the regime \(t>2h+16\Rightarrow t-h-8>h+8\), so this is super-poly small; for the bare statement it suffices that \(2^{t-h-8}\) is super-poly.)

Bounding \(\Pr[\mathsf{AskW}]\) via one-wayness (the reduction). We build an inverter \(\mathcal{B}\) (an elementary wrapper around \(\mathcal{A}\)) for \(T\) in the sense of Attack Game 10.2.

  1. \(\mathcal{B}\) receives \((pk, y^\ast)\) with \(y^\ast=F(pk,x^\ast)\) for \(x^\ast=00\,\|\,r'^\ast\,\|\,z'^\ast\xleftarrow{\$}\mathcal{X}\) chosen by its challenger; thus \(r'^\ast,z'^\ast\) are uniform — exactly the Game-2 distribution of the challenge preimage.
  2. \(\mathcal{B}\) runs \(\mathcal{A}\) on \(pk\), hands it the challenge ciphertext \(c^\ast:=y^\ast\), and answers all \(H,W\) queries by lazy uniform sampling. It needs neither \(r'^\ast\) nor \(z'^\ast\): in Game 2 the challenge is formed independently of the oracle tables, so this perfectly simulates Game 2. (The bit \(b\) and the messages are irrelevant to \(\mathcal{B}\).)
  3. When \(\mathcal{A}\) halts, \(\mathcal{B}\) lets \(\mathcal{R}_q\) be the set of seeds at which \(\mathcal{A}\) queried \(W\) (\(|\mathcal{R}_q|\le Q_W\)) and \(Z\) the set of points at which \(\mathcal{A}\) queried \(H\) (\(|Z|\le Q_H\)), with recorded answers \(H(\hat z)\). For each \((\hat r,\hat z)\in\mathcal{R}_q\times Z\) it forms the candidate preimage \(\hat x:=00\,\|\,(H(\hat z)\oplus\hat r)\,\|\,\hat z\) and tests \(F(pk,\hat x)=y^\ast\); on a hit it outputs that \(\hat x\).
Why this inverts: \(z'^\ast\) is literally the right block of \(x^\ast\), and \(r'^\ast=H(z'^\ast)\oplus r^\ast\) where \(r^\ast\) is the seed. On the event \(\mathsf{AskW}\wedge\mathsf{AskH}\) we have \(r^\ast\in\mathcal{R}_q\) and \(z'^\ast\in Z\), so the pair \((\hat r,\hat z)=(r^\ast,z'^\ast)\) is among those tried, and then \(H(\hat z)\oplus\hat r=H(z'^\ast)\oplus r^\ast=r'^\ast\), giving \(\hat x=00\,\|\,r'^\ast\,\|\,z'^\ast=x^\ast\): \(\mathcal{B}\) inverts. Hence \(\mathrm{OWadv}[\mathcal{B},T]\ge\Pr[\mathsf{AskW}\wedge\mathsf{AskH}]\). On the complementary event \(\mathsf{AskW}\wedge\neg\mathsf{AskH}\), the value \(H(z'^\ast)\) was never queried, so \(r^\ast=H(z'^\ast)\oplus r'^\ast\) is uniform and independent of \(\mathcal{A}\)'s view; \(\mathcal{A}\) thus queries \(W\) at this particular \(r^\ast\) with probability at most \(Q_W/2^{h}\). Therefore \[ \Pr[\mathsf{AskW}] = \Pr[\mathsf{AskW}\wedge\mathsf{AskH}] + \Pr[\mathsf{AskW}\wedge\neg\mathsf{AskH}] \le \mathrm{OWadv}[\mathcal{B},T] + \frac{Q_W}{2^{h}}. \]

Putting it together. Combining the game hops, \[ \bigl|\Pr[W_0]-\tfrac12\bigr| = \bigl|\Pr[W_1]-\tfrac12\bigr| = \bigl|\Pr[W_1]-\Pr[W_2]\bigr| \le \Pr[\mathsf{AskW}]+\Pr[\mathsf{AskH}], \] so \[ \mathrm{CPAadv}[\mathcal{A},E_{\mathrm{OAEP}}] = 2\,\bigl|\Pr[W_0]-\tfrac12\bigr| \le 2\,\mathrm{OWadv}[\mathcal{B},T] + \frac{2Q_W}{2^{h}} + \frac{2Q_H}{2^{\,t-h-8}}. \] The first term is negligible because \(T\) is one-way; the last two terms are negligible because \(h\) and \(t-h-8\) are super-poly-sized (indeed \(2^h\) is required to be super-poly for OAEP). Hence \(\mathrm{CPAadv}[\mathcal{A},E_{\mathrm{OAEP}}]\) is negligible, i.e. \(E_{\mathrm{OAEP}}\) is CPA secure in the random oracle model for any one-way trapdoor function \(T\), as required.

Remark. Plain one-wayness suffices for CPA security; it is for CCA security that OAEP needs the stronger partial one-wayness (Theorem 12.13), and indeed Exercise 12.23 below exhibits a one-way \(T\) for which \(E_{\mathrm{OAEP}}\) is not CCA secure.

12.23 (A counter-example to the CCA-security of OAEP)

Let \(T_0=(G,F_0,I_0)\) be a one-way trapdoor permutation over \(\mathcal{R}=\{0,1\}^h\) that is xor-homomorphic: there is an efficient algorithm \(C\) with \(C(F_0(pk,r))=F_0(pk,r\oplus\Delta)\) for a fixed shift \(\Delta\in\mathcal{R}\) (more precisely, for all \(r,\Delta\), \(C_\Delta(F_0(pk,r))=F_0(pk,r\oplus\Delta)\)). For \(t>2h+16\) define \(T=(G,F,I)\) over \(0^8\times\{0,1\}^{t-8}\) by \[ F\bigl(pk,(00\,\|\,r\,\|\,z)\bigr) = 00\,\|\,F_0(pk,r)\,\|\,z . \] Couple \(T\) with OAEP via (12.41) to get \(E_{\mathrm{OAEP}}\). Exhibit a CCA attack of advantage \(1\).

Idea

This \(T\) leaks the right block \(z'\) of the preimage and only "hides" the seed component \(r'\) — and even that only through the homomorphic permutation \(F_0\). The OAEP decoding recomputes \(r=H(z')\oplus r'\) and \(z=W(r)\oplus z'\). If we use \(C\) to flip the seed from \(r'\) to \(r'\oplus\Delta\) inside the ciphertext while leaving \(z'\) untouched, the decryptor recomputes a different \(r\), hence a different pad \(W(r)\), hence a garbled \(z\) that almost surely fails to parse — so this naive maul just yields reject. The real attack is subtler: we must change \(r'\) and compensate so that the recovered \(z\) still parses to a message we can relate to the challenge. Because the attacker controls the random oracles' query points, it can read off \(z'^\ast\) from the challenge (it is in the clear!), recompute everything, and submit a fresh ciphertext encrypting a known related message, then compare.

Solution

What the ciphertext reveals. An OAEP ciphertext is \(c=F(pk,x)\) with \(x=00\,\|\,r'\,\|\,z'\), so \[ c=00\,\|\,F_0(pk,r')\,\|\,z' . \] Thus the right block \(z'\in\{0,1\}^{t-h-8}\) appears literally in the clear in every ciphertext, and the middle block is \(F_0(pk,r')\). Decoding (algorithm \(U\)) computes \[ r := H(z')\oplus r',\qquad z := W(r)\oplus z',\qquad \text{parse } z = d\,\|\,0\cdots01\,\|\,m . \] The decryptor first inverts \(F\): from \(c=00\,\|\,Y\,\|\,z'\) it sets \(r':=I_0(sk,Y)\) and reads \(z'\) directly, then runs \(U\).

Why the obvious maul fails. The tempting move is to flip the seed with the homomorphism alone: set \(Y':=C_\Delta(Y^\ast)=F_0(pk,r'^\ast\oplus\Delta)\) and keep \(z'^\ast\), giving \(c'=00\,\|\,Y'\,\|\,z'^\ast\neq c^\ast\). But then decoding recomputes the seed as \(\tilde r=H(z'^\ast)\oplus(r'^\ast\oplus\Delta)=r^\ast\oplus\Delta\), so the pad changes to \(W(r^\ast\oplus\Delta)\) and \(\tilde z=W(r^\ast\oplus\Delta)\oplus W(r^\ast)\oplus z^\ast\) is garbled — the query just returns reject. The seed is protected because it feeds the pad \(W\). The correct attack instead mauls the exposed block \(z'\), and uses the homomorphism precisely to cancel the resulting change in the seed so that the pad stays put.

The attack. Adversary \(\mathcal{A}\) plays the CCA game (Attack Game 12.1).

  1. Phase 1: none.
  2. Encryption query. \(\mathcal{A}\) picks two distinct equal-length messages \(m_0\neq m_1\) and submits \((m_0,m_1)\). It receives the challenge \[ c^\ast = 00\,\|\,Y^\ast\,\|\,z'^\ast,\qquad Y^\ast=F_0(pk,r'^\ast), \] where internally \(r'^\ast=H(z'^\ast)\oplus r^\ast\), \(z'^\ast=W(r^\ast)\oplus z^\ast\), and \(z^\ast=d\,\|\,0\cdots01\,\|\,m_b\). The block \(z'^\ast\) is read off \(c^\ast\) directly.
  3. Choose a message-field shift. \(\mathcal{A}\) fixes any nonzero \(\delta=0\cdots0\,\|\,\delta_m\) supported only on the low \(|m|\) bits of the \((t-h-8)\)-bit block, so that XORing by \(\delta\) leaves the constant \(d\) and the \(0\cdots01\) marker of \(z^\ast\) untouched and only alters the message field.
  4. Read the two \(H\)-pads. \(\mathcal{A}\) queries the random oracle \(H\) at \(z'^\ast\) and at \(z'^\ast\oplus\delta\), obtaining \(s:=H(z'^\ast)\) and \(s_\delta:=H(z'^\ast\oplus\delta)\). Both are explicit values; the public shift it needs is \(\Sigma:=s\oplus s_\delta\in\mathcal{R}\).
  5. Compensate the seed via the homomorphism. \(\mathcal{A}\) applies the xor-homomorphism with shift \(\Sigma\) to the middle block (the fixed \(\Delta\) of \(T_0\) generates every shift by iteration, so we may write \(C_\Sigma\)): \[ Y':=C_\Sigma(Y^\ast)=F_0(pk,\,r'^\ast\oplus\Sigma), \] and forms the mauled ciphertext \[ c' := 00\,\|\,Y'\,\|\,(z'^\ast\oplus\delta). \] Since \(\delta\neq 0\) the right block differs from \(z'^\ast\), so \(c'\neq c^\ast\): it is a legal Phase-2 query.

What \(c'\) decrypts to. The decryptor inverts \(F\): from \(c'=00\,\|\,Y'\,\|\,(z'^\ast\oplus\delta)\) it computes \(r'':=I_0(sk,Y')=r'^\ast\oplus\Sigma\) and reads the right block \(z'^\ast\oplus\delta\). It then decodes. The recomputed seed is \[ \tilde r = H(z'^\ast\oplus\delta)\oplus r'' = s_\delta\oplus(r'^\ast\oplus\Sigma) = s_\delta\oplus r'^\ast\oplus s\oplus s_\delta = s\oplus r'^\ast = r^\ast, \] using \(r'^\ast=H(z'^\ast)\oplus r^\ast=s\oplus r^\ast\). So the seed is restored to the original \(r^\ast\); the pad is therefore unchanged, \(W(\tilde r)=W(r^\ast)\), and \[ \tilde z = W(r^\ast)\oplus(z'^\ast\oplus\delta) = \bigl(W(r^\ast)\oplus z'^\ast\bigr)\oplus\delta = z^\ast\oplus\delta = d\,\|\,0\cdots01\,\|\,(m_b\oplus\delta_m). \] Because \(\delta\) touches only the message field, \(\tilde z\) still parses (the constant \(d\) and the marker \(0\cdots01\) are intact), yielding the message \(\tilde m=m_b\oplus\delta_m\) with certainty.

Output and advantage. \(\mathcal{A}\) submits \(c'\), receives \(\tilde m=m_b\oplus\delta_m\), computes \(\hat m:=\tilde m\oplus\delta_m=m_b\), and outputs \(\hat b:=1\) if \(\hat m=m_1\), else \(\hat b:=0\). The decryption is exact and never rejects, so \(\hat b=b\) with probability \(1\): \[ \mathrm{CCAadv}[\mathcal{A},E_{\mathrm{OAEP}}] = |\Pr[W_0]-\Pr[W_1]| = 1 . \] \(\mathcal{A}\) uses two random-oracle queries, one decryption query, and poly time. Hence \(E_{\mathrm{OAEP}}\) coupled with this one-way trapdoor \(T\) is completely broken under CCA. This is the counter-example promised after Theorem 12.13: one-wayness of \(T\) is not sufficient for CCA security of \(E_{\mathrm{OAEP}}\). The scheme fails partial one-wayness — the message block \(z'\) is exposed and the seed-image \(F_0(pk,r')\) is malleable — which is exactly the property Theorem 12.13 must assume, and which RSA (conjecturally) satisfies.

13.1 (Exercising the definition)

Let \((G,S,V)\) be a secure signature scheme over message space \(\{0,1\}^n\). Generate two independent key pairs \((pk_0,sk_0)\xleftarrow{R}G()\) and \((pk_1,sk_1)\xleftarrow{R}G()\). For each of the following, prove security or exhibit an attack.

(a) Accept one valid. \(S_1\big((sk_0,sk_1),m\big):=\big(S(sk_0,m),\,S(sk_1,m)\big)\), and \(V_1\) accepts \(\big((pk_0,pk_1),m,(\sigma_0,\sigma_1)\big)\) iff \(V(pk_0,m,\sigma_0)=\texttt{accept}\) or \(V(pk_1,m,\sigma_1)=\texttt{accept}\).

(b) Sign halves. \(S_2\big((sk_0,sk_1),(m_L,m_R)\big):=\big(S(sk_0,m_L),\,S(sk_1,m_R)\big)\), and \(V_2\) accepts \(\big((pk_0,pk_1),(m_L,m_R),(\sigma_0,\sigma_1)\big)\) iff \(V(pk_0,m_L,\sigma_0)=V(pk_1,m_R,\sigma_1)=\texttt{accept}\).

(c) Sign with randomness. \(S_3(sk_0,m):=\{r\xleftarrow{R}\{0,1\}^n,\ \text{output}\ (r,\,S(sk_0,m\oplus r),\,S(sk_0,r))\}\), and \(V_3\) accepts \(\big(pk_0,m,(r,\sigma_0,\sigma_1)\big)\) iff \(V(pk_0,m\oplus r,\sigma_0)=V(pk_0,r,\sigma_1)=\texttt{accept}\).

Idea

(a) is secure: a forgery on a new message must contain a valid underlying signature under at least one of the two keys, which is itself a forgery against that scheme. (b) and (c) are insecure: in both the two signature components are "decoupled", so the adversary can mix-and-match pieces obtained from different queries to splice together a fresh valid signature, with advantage \(1\) (up to a negligible collision term).

Solution

(a) Secure. The idea is simple. We show that an efficient forger \(\mathcal{A}\) against \(S_1\) implies an efficient forger \(\mathcal{B}\) (an elementary wrapper around \(\mathcal{A}\)) against the underlying scheme \(S\). \(\mathcal{B}\) receives a challenge public key \(pk^\ast\) from its \(S\)-challenger. It flips a coin \(\beta\xleftarrow{R}\{0,1\}\), plants the challenge by setting \(pk_\beta:=pk^\ast\), and generates the other pair itself: \((pk_{1-\beta},sk_{1-\beta})\xleftarrow{R}G()\). It hands \((pk_0,pk_1)\) to \(\mathcal{A}\).

On a signing query \(m\) from \(\mathcal{A}\), \(\mathcal{B}\) needs both components \(\big(S(sk_0,m),S(sk_1,m)\big)\). It produces the \(S(sk_{1-\beta},m)\) component itself (it holds \(sk_{1-\beta}\)), and obtains \(S(sk_\beta,m)\) by forwarding \(m\) to its own signing oracle. Thus \(\mathcal{B}\) perfectly simulates the \(S_1\)-challenger, and the planted index \(\beta\) is uniform over \(\{0,1\}\) independently of \(\mathcal{A}\)'s view (the two keys are identically distributed).

Eventually \(\mathcal{A}\) outputs a forgery \(\big(m,(\sigma_0,\sigma_1)\big)\) with \(m\) new. By the \(V_1\) acceptance rule there is some \(b\in\{0,1\}\) with \(V(pk_b,m,\sigma_b)=\texttt{accept}\). Since \(m\) was never queried by \(\mathcal{A}\), it was never forwarded to \(\mathcal{B}\)'s signing oracle either; hence if \(b=\beta\), the pair \((m,\sigma_\beta)\) is a valid existential forgery under \(pk^\ast\), and \(\mathcal{B}\) outputs it. The event \(b=\beta\) holds with probability \(\tfrac12\) because \(\beta\) is hidden from \(\mathcal{A}\). Therefore \[ \mathrm{SIGadv}[\mathcal{B},S]\ \ge\ \tfrac12\,\mathrm{SIGadv}[\mathcal{A},S_1], \qquad\text{i.e.}\qquad \mathrm{SIGadv}[\mathcal{A},S_1]\ \le\ 2\,\mathrm{SIGadv}[\mathcal{B},S]. \] As \(S\) is secure, the right side is negligible, so \(S_1\) is secure, as required. (Equivalently one may invoke multi-key security with \(Q=2\), Exercise 13.2, giving the same factor \(2\).)

(b) Insecure — mix-and-match. We give an explicit adversary with advantage \(1\). Pick any four messages with \(m_L\neq m_L'\) and \(m_R\neq m_R'\). Issue two signing queries:

\(\;\;\bullet\;\) query \((m_L,m_R)\), receiving \((\sigma_0,\sigma_1)\) with \(\sigma_0=S(sk_0,m_L)\), \(\sigma_1=S(sk_1,m_R)\);

\(\;\;\bullet\;\) query \((m_L',m_R')\), receiving \((\sigma_0',\sigma_1')\) with \(\sigma_0'=S(sk_0,m_L')\), \(\sigma_1'=S(sk_1,m_R')\).

Now output the forgery \(\big((m_L,m_R'),\,(\sigma_0,\sigma_1')\big)\). Since \(\sigma_0\) is a genuine signature on \(m_L\) under \(pk_0\) and \(\sigma_1'\) is a genuine signature on \(m_R'\) under \(pk_1\), verification gives \(V(pk_0,m_L,\sigma_0)=V(pk_1,m_R',\sigma_1')=\texttt{accept}\). The forged message \((m_L,m_R')\) differs from both queried messages \((m_L,m_R)\) and \((m_L',m_R')\) (it disagrees with the first in the right half and with the second in the left half), so it is new. The advantage is exactly \(1\).

(c) Insecure — splicing across two queries. The key observation is that every signing query, on whatever message, hands the adversary genuine \(sk_0\)-signatures on two values, and the verifier never checks how those values relate to the claimed message except through the equations \(m\oplus r\) and \(r\). We exploit this with two queries on an arbitrary message \(m\).

Query \(m\) twice, obtaining \[ (r_1,\sigma_0^{(1)},\sigma_1^{(1)}),\qquad (r_2,\sigma_0^{(2)},\sigma_1^{(2)}), \] where \(\sigma_1^{(1)}=S(sk_0,r_1)\) and \(\sigma_1^{(2)}=S(sk_0,r_2)\) are valid \(sk_0\)-signatures on the values \(r_1\) and \(r_2\) respectively (the challenger chooses \(r_1,r_2\) at random). Now set \[ m^\ast:=r_1\oplus r_2,\qquad r^\ast:=r_1,\qquad \sigma_0^\ast:=\sigma_1^{(2)},\qquad \sigma_1^\ast:=\sigma_1^{(1)}, \] and output \(\big(m^\ast,(r^\ast,\sigma_0^\ast,\sigma_1^\ast)\big)\). We check the two verification equations under \(pk_0\):

\(\;\;\bullet\;\) \(m^\ast\oplus r^\ast=(r_1\oplus r_2)\oplus r_1=r_2\), and \(\sigma_0^\ast=\sigma_1^{(2)}=S(sk_0,r_2)\), so \(V(pk_0,m^\ast\oplus r^\ast,\sigma_0^\ast)=\texttt{accept}\);

\(\;\;\bullet\;\) \(r^\ast=r_1\), and \(\sigma_1^\ast=\sigma_1^{(1)}=S(sk_0,r_1)\), so \(V(pk_0,r^\ast,\sigma_1^\ast)=\texttt{accept}\).

Hence \(V_3\) accepts. The only queried message is \(m\), so the forgery is new provided \(m^\ast=r_1\oplus r_2\neq m\). Since \(r_1,r_2\) are uniform and independent of \(m\), the value \(r_1\oplus r_2\) is uniform in \(\{0,1\}^n\), so \(\Pr[m^\ast=m]=2^{-n}\). Therefore the advantage is \(1-2^{-n}\), which is non-negligible, and \(S_3\) is insecure. (Note the attack never even uses the \(\sigma_0\) components.)

13.2 (Multi-key signature security)

(a) Extend Attack Game 13.1 so that the attacker may submit signing queries with respect to several signing keys (as in the multi-key generalization of Exercise 6.3). (b) Show that every efficient adversary \(\mathcal{A}\) winning the multi-key game with probability \(\epsilon\) can be turned into an efficient adversary \(\mathcal{B}\) winning Attack Game 13.1 with probability \(\epsilon/Q\), where \(Q\) is the number of keys, by the "plug-and-pray" technique.

Idea

The multi-key game just runs \(Q\) independent copies of the single-key game and lets the adversary forge under any one key. To reduce to single-key security, \(\mathcal{B}\) plugs its single challenge key at a random index \(\omega\), generates the other \(Q-1\) keys itself, and prays that \(\mathcal{A}\)'s forgery targets index \(\omega\); this succeeds with probability \(1/Q\).

Solution

(a) The multi-key attack game. For a signature scheme \(S=(G,S,V)\) and adversary \(\mathcal{A}\):

\(\;\;\bullet\;\) \(\mathcal{A}\) outputs a number \(Q\) of keys it wishes to attack. The challenger runs \((pk_j,sk_j)\xleftarrow{R}G()\) for \(j=1,\dots,Q\) and sends \((pk_1,\dots,pk_Q)\) to \(\mathcal{A}\).

\(\;\;\bullet\;\) \(\mathcal{A}\) makes signing queries, each of the form \((j,m)\) with \(j\in\{1,\dots,Q\}\); the challenger replies \(\sigma\xleftarrow{R}S(sk_j,m)\).

\(\;\;\bullet\;\) \(\mathcal{A}\) outputs \((j^\ast,m^\ast,\sigma^\ast)\). It wins if \(V(pk_{j^\ast},m^\ast,\sigma^\ast)=\texttt{accept}\) and \(m^\ast\) was not previously queried under key \(j^\ast\) (i.e. \((j^\ast,m^\ast)\) is new among the signing queries).

Define \(\mathrm{SIGadv}^{(Q)}[\mathcal{A},S]\) as the probability \(\mathcal{A}\) wins. The scheme is multi-key secure if this is negligible for all efficient \(\mathcal{A}\) and all poly-bounded \(Q\).

(b) Plug-and-pray reduction. The idea is simple. We build \(\mathcal{B}\), an elementary wrapper around \(\mathcal{A}\), that breaks ordinary single-key security (Attack Game 13.1). \(\mathcal{B}\) receives a challenge public key \(pk^\ast\) from its challenger. When \(\mathcal{A}\) announces \(Q\), adversary \(\mathcal{B}\) chooses a random index \(\omega\xleftarrow{R}\{1,\dots,Q\}\), sets \(pk_\omega:=pk^\ast\), and for every \(j\neq\omega\) generates \((pk_j,sk_j)\xleftarrow{R}G()\) itself. It sends \((pk_1,\dots,pk_Q)\) to \(\mathcal{A}\). This distribution is identical to the real multi-key game, and \(\omega\) is uniform and independent of \(\mathcal{A}\)'s view.

\(\mathcal{B}\) answers a signing query \((j,m)\): if \(j\neq\omega\) it uses the secret key \(sk_j\) it holds; if \(j=\omega\) it forwards \(m\) to its own \(S\)-signing oracle and relays the answer. Thus \(\mathcal{B}\) perfectly simulates the multi-key challenger.

When \(\mathcal{A}\) outputs \((j^\ast,m^\ast,\sigma^\ast)\), adversary \(\mathcal{B}\) outputs \((m^\ast,\sigma^\ast)\) — but this is a valid single-key forgery only if \(j^\ast=\omega\). When \(\mathcal{A}\) wins and \(j^\ast=\omega\), then \(V(pk^\ast,m^\ast,\sigma^\ast)=\texttt{accept}\) and \(m^\ast\) was never forwarded to \(\mathcal{B}\)'s oracle (since \((j^\ast,m^\ast)=(\omega,m^\ast)\) was new), so \(\mathcal{B}\) wins Attack Game 13.1. Because \(\omega\) is independent of \(\mathcal{A}\)'s view, \(\Pr[j^\ast=\omega\mid \mathcal{A}\text{ wins}]=1/Q\). Hence \[ \mathrm{SIGadv}[\mathcal{B},S]\ \ge\ \tfrac1Q\,\mathrm{SIGadv}^{(Q)}[\mathcal{A},S]\ =\ \frac{\epsilon}{Q}. \] Since \(S\) is single-key secure, \(\epsilon/Q\) is negligible; as \(Q\) is poly-bounded, \(\epsilon\) is negligible too. Thus single-key security implies multi-key security, as required.

13.3 (Non-binding signatures)

Give an example of a secure signature scheme that is non-binding: for a given \((pk,sk)\) the signer can find two distinct messages \(m_0\neq m_1\) for which the same signature \(\sigma\) is valid under \(pk\). (Hint: hash-and-sign with the trapdoor collision-resistant hash of Exercise 10.30.)

Idea

Use hash-and-sign with a hash function that is collision resistant to outsiders but has a trapdoor that lets the holder produce collisions. Then \(H(m_0)=H(m_1)\) makes one signature valid for both messages — so the scheme is non-binding — yet because nobody else can find collisions, hash-and-sign stays secure.

Solution

Let \(S_0=(G_0,S_0,V_0)\) be any secure signature scheme with message space \(\mathbb{G}\) (e.g. \(S_{\mathrm{RSA\text{-}FDH}}\)). Let \(H_{\mathrm{dl}}\) be the trapdoor collision-resistant hash of Exercise 10.30: with public parameters a group \(\mathbb{G}\) of prime order \(q\) with generator \(g\) and a second element \(h=g^t\), define \[ H_{\mathrm{dl}}(\alpha,\beta):=g^{\alpha}h^{\beta}\in\mathbb{G},\qquad (\alpha,\beta)\in\mathbb{Z}_q\times\mathbb{Z}_q . \] Under the discrete-log assumption in \(\mathbb{G}\), \(H_{\mathrm{dl}}\) is collision resistant; indeed a collision \(g^{\alpha}h^{\beta}=g^{\alpha'}h^{\beta'}\) with \((\alpha,\beta)\neq(\alpha',\beta')\) yields \(t=(\alpha-\alpha')/(\beta'-\beta)\bmod q\), i.e. \(\mathrm{dlog}_g h\). But the trapdoor \(t\) lets its holder turn any preimage into a colliding one.

The scheme. Define \(S=(G,S,V)\) by hash-and-sign (eq. 13.1) over \(\mathbb{Z}_q\times\mathbb{Z}_q\):

\(\;\;\bullet\;\) \(G\): run \(G_0\) to get \((pk_0,sk_0)\); the public parameters \((\mathbb{G},g,h)\) of \(H_{\mathrm{dl}}\) are part of \(pk\). Crucially, the signer also keeps the trapdoor \(t=\mathrm{dlog}_g h\) (the signer generated \(h\) as \(g^t\)). So \(pk=(pk_0,\mathbb{G},g,h)\) and \(sk=(sk_0,t)\).

\(\;\;\bullet\;\) \(S(sk,m):=S_0(sk_0,H_{\mathrm{dl}}(m))\) and \(V(pk,m,\sigma):=V_0(pk_0,H_{\mathrm{dl}}(m),\sigma)\), where messages \(m=(\alpha,\beta)\).

Security. This is exactly the hash-and-sign construction of Theorem 13.1. To an outside adversary the trapdoor \(t\) is unknown (it sees only \(pk\), which contains \(h\) but not \(t\)), so \(H_{\mathrm{dl}}\) is collision resistant from the adversary's viewpoint. By Theorem 13.1, for any forger \(\mathcal{A}\) there are elementary wrappers \(\mathcal{B}_S,\mathcal{B}_H\) with \[ \mathrm{SIGadv}[\mathcal{A},S]\ \le\ \mathrm{SIGadv}[\mathcal{B}_S,S_0]+\mathrm{CRadv}[\mathcal{B}_H,H_{\mathrm{dl}}] \ \le\ \mathrm{SIGadv}[\mathcal{B}_S,S_0]+\mathrm{DLadv}[\mathcal{B}',\mathbb{G}], \] both negligible. Hence \(S\) is a secure signature scheme. (The proof of Theorem 13.1 does not depend on whether the signer privately knows a trapdoor; collision resistance need only hold against the adversary.)

Non-binding. The signer, knowing \(t\), is not collision resistant to itself. Given any message \(m_0=(\alpha,\beta)\), pick any \(\delta\neq 0\) and set \[ m_1:=(\alpha+t\delta,\ \beta-\delta)\bmod q . \] Then \(m_1\neq m_0\) and \[ H_{\mathrm{dl}}(m_1)=g^{\alpha+t\delta}h^{\beta-\delta}=g^{\alpha}g^{t\delta}h^{\beta}h^{-\delta} =g^{\alpha}h^{\beta}\cdot (g^{t}h^{-1})^{\delta}=g^{\alpha}h^{\beta}=H_{\mathrm{dl}}(m_0), \] using \(g^{t}=h\). Therefore any single signature \(\sigma=S_0(sk_0,H_{\mathrm{dl}}(m_0))\) satisfies \(V(pk,m_0,\sigma)=V(pk,m_1,\sigma)=\texttt{accept}\). The signer has produced two distinct messages bound to the same signature, so the scheme is non-binding while remaining secure, as required.

13.5 (Strongly binding signatures)

A strong binding adversary \(\mathcal{A}\) outputs \((pk,m,pk',m',\sigma)\) and defeats strong binding if \((pk,m)\neq(pk',m')\) and \(V(pk,m,\sigma)=V(pk',m',\sigma)=\texttt{accept}\); write \(\mathrm{sbSIGadv}[\mathcal{A},S]\) for its success probability. (a) Augment \(S\) so the signer signs \((pk,m)\): \(S'(sk,m):=S(sk,(pk,m))\), \(V'(pk,m,\sigma):=V(pk,(pk,m),\sigma)\). Show that applied to \(S_{\mathrm{RSA\text{-}FDH}}\), with \(H\) a random oracle, the result is still secure and now strongly binding. (b) Give a secure scheme that is not strongly binding even after (a). (c) For a hash \(H''\) define \(S''(sk,m):=(H''(pk,m),\,S(sk,m))\) and \(V''\) accepts \((h,\sigma)\) iff \(h=H''(pk,m)\) and \(V(pk,m,\sigma)=\texttt{accept}\). Show (i) \(S\) secure \(\Rightarrow S''\) secure, and (ii) \(H''\) collision resistant \(\Rightarrow S''\) strongly binding.

Idea

(a) Folding \(pk\) into the signed message changes nothing about unforgeability (it is hash-and-sign with the public key prepended), and for FDH it forces a strong-binding break to produce two pairs whose random-oracle images coincide with the same \(\sigma^e\) under different moduli — which, in the RO model, requires either an unlikely collision among fresh random values or inverting RSA. (b) FDH already has a built-in malleability of the modulus that survives the \((pk,m)\) trick when the same signer reuses a value. (c) Appending a CR hash of \((pk,m)\) makes a strong-binding break literally a collision of \(H''\).

Solution

(a) \(S'_{\mathrm{RSA\text{-}FDH}}\) is secure and strongly binding (RO model). Here \(pk=(n,e)\), \(sk=(n,d)\), and \(S'(sk,m)=H(pk,m)^d\), with \(H\) a random oracle into \(Y\subseteq\mathbb{Z}_n\).

Security. This is just \(S_{\mathrm{RSA\text{-}FDH}}\) on the enlarged message space \(\mathcal{M}'=\{pk\}\times\mathcal{M}\) using the random oracle \(H'(\cdot):=H(pk,\cdot)\). Prepending the fixed string \(pk\) is an injective re-encoding of the message, so any forger \(\mathcal{A}\) against \(S'\) is, verbatim, a forger against \(S_{\mathrm{RSA\text{-}FDH}}\): if \(\mathcal{A}\) forges \((m^\ast,\sigma^\ast)\) with \(m^\ast\) new, then \((pk,m^\ast)\) is a new point of \(H\), and \((\,(pk,m^\ast),\sigma^\ast)\) is an FDH forgery. Theorem 13.4 gives \[ \mathrm{SIGro\,adv}[\mathcal{A},S'_{\mathrm{RSA\text{-}FDH}}]\ \le\ 2.72\,(Q_s+1)\,\mathrm{RSAadv}[\mathcal{B},\ell,e], \] negligible under RSA. So the augmented scheme is secure.

Strong binding. Suppose a strong-binding adversary outputs \((pk,m,pk',m',\sigma)\) with \((pk,m)\neq(pk',m')\), \(pk=(n,e)\), \(pk'=(n',e')\), and both verify: \[ \sigma^{e}=H(pk,m)\ \text{in}\ \mathbb{Z}_n,\qquad \sigma^{e'}=H(pk',m')\ \text{in}\ \mathbb{Z}_{n'} . \] We must show this is hard in the RO model. Note that the two inputs to \(H\) are \((pk,m)\) and \((pk',m')\); since \((pk,m)\neq(pk',m')\) these are distinct oracle points, so \(H(pk,m)\) and \(H(pk',m')\) are independent uniform values in \(Y\). Model the adversary as making \(Q_{ro}\) oracle queries and consider the moment each value is sampled.

The adversary commits to a single ring element \(\sigma\) (an integer in \([0,\min(n,n'))\), or reduced in each ring). For the first equation to hold, the freshly sampled uniform value \(H(pk,m)\in Y\) must equal the fixed quantity \(\sigma^{e}\bmod n\); since \(|Y|\ge n/4\) and \(H(pk,m)\) is uniform over \(Y\) independent of everything the adversary chose before the query, this holds with probability at most \(4/n\) per query unless the adversary first fixes \(H(pk,m)\) and then computes \(\sigma=H(pk,m)^{1/e}\) — i.e. inverts RSA on a random target. The same holds for the second equation under \((n',e')\). Concretely, reduce to RSA exactly as in Theorem 13.4: an inverter \(\mathcal{B}\) plants its challenge \(y^\ast\) as the oracle answer at a guessed query; if the adversary succeeds in matching \(\sigma^e=H(pk,m)\) for that planted value with \(pk\) the honest key, then \(\sigma=(y^\ast)^{1/e}\) solves RSA. Hence \[ \mathrm{sbSIGadv}[\mathcal{A},S'_{\mathrm{RSA\text{-}FDH}}]\ \le\ (Q_{ro}+1)\,\mathrm{RSAadv}[\mathcal{B},\ell,e]+\tfrac{4Q_{ro}}{n}, \] which is negligible. Intuitively, because \(pk\) is inside the hash, the two accepted pairs are necessarily two different random-oracle points, so the adversary cannot reuse one accepted equation to satisfy the other; it must invert FDH afresh at an independent random target. Thus \(S'_{\mathrm{RSA\text{-}FDH}}\) is strongly binding.

(b) A secure scheme not strongly binding even after (a). Take any secure scheme \(S^\flat=(G^\flat,S^\flat,V^\flat)\) and define \(S\) which simply ignores a trailing bit of the message: messages are \(m=(m_0,c)\) with \(c\in\{0,1\}\), and \[ S(sk,(m_0,c)):=S^\flat(sk,m_0),\qquad V(pk,(m_0,c),\sigma):=V^\flat(pk,m_0,\sigma). \] This is secure (a forgery on a new \((m_0,c)\) gives a forgery on \(m_0\) under \(S^\flat\) whenever \(m_0\) is new; and if \(m_0\) was queried, then forging a new message means flipping \(c\), which is allowed — so to be safe take \(S^\flat\) and observe the reduction still forges on \(m_0\) since the \(S\)-game treats \((m_0,0)\) and \((m_0,1)\) as querying \(m_0\); more cleanly, define the queried set in the reduction to include \(m_0\) only when both extensions were asked, giving \(\mathrm{SIGadv}[\mathcal{A},S]\le 2\,\mathrm{SIGadv}[\mathcal{B},S^\flat]\)). But the same \(\sigma\) is valid for both \((m_0,0)\) and \((m_0,1)\) under the same \(pk\): with \((pk,m)=(pk,(m_0,0))\neq(pk,(m_0,1))=(pk,m')\) we get \(V(pk,m,\sigma)=V(pk,m',\sigma)=\texttt{accept}\). This breaks strong binding with advantage \(1\). Applying the augmentation of part (a) does not help: \(S'\) signs \((pk,(m_0,c))\) but \(V'\) still ignores \(c\), because \(V'\) recomputes \(H'(pk,(m_0,c))\) which is again independent of \(c\) once we fold it through the ignoring step — more directly, repeat the construction so that the scheme being augmented ignores the trailing bit, so \((pk,(m_0,0))\) and \((pk,(m_0,1))\) again share a signature. So \(S'\) is still not strongly binding.

(c) The \(H''(pk,m)\) augmentation.

(i) \(S\) secure \(\Rightarrow S''\) secure. Let \(\mathcal{A}\) forge \(S''\). Build \(\mathcal{B}\) against \(S\): \(\mathcal{B}\) gets \(pk\), forwards it, and answers each signing query \(m\) by querying its own oracle for \(\sigma\leftarrow S(sk,m)\), computing \(h\leftarrow H''(pk,m)\) (it knows \(pk\) and \(H''\) is public), and returning \((h,\sigma)\) — a perfect simulation. When \(\mathcal{A}\) outputs \(\big(m^\ast,(h^\ast,\sigma^\ast)\big)\) with \(m^\ast\) new and \(V''\) accepting, then in particular \(V(pk,m^\ast,\sigma^\ast)=\texttt{accept}\) and \(m^\ast\) was never queried, so \((m^\ast,\sigma^\ast)\) is a valid \(S\)-forgery. Thus \(\mathrm{SIGadv}[\mathcal{A},S'']\le \mathrm{SIGadv}[\mathcal{B},S]\), negligible. (The extra \(h\) component is publicly recomputable and adds nothing for the forger.)

(ii) \(H''\) collision resistant \(\Rightarrow S''\) strongly binding. Let \(\mathcal{A}\) output \(\big(pk,m,pk',m',(h,\sigma)\big)\) with \((pk,m)\neq(pk',m')\) and both \(V''\) checks accepting. Acceptance forces \[ h=H''(pk,m)\qquad\text{and}\qquad h=H''(pk',m'). \] Hence \(H''(pk,m)=H''(pk',m')\) with \((pk,m)\neq(pk',m')\): this is exactly a collision for \(H''\). The wrapper \(\mathcal{B}_H\) runs \(\mathcal{A}\) and outputs the pair \((pk,m),(pk',m')\), so \[ \mathrm{sbSIGadv}[\mathcal{A},S'']\ \le\ \mathrm{CRadv}[\mathcal{B}_H,H''], \] negligible. Therefore \(S''\) is strongly binding, as required. (Note this works for every base scheme \(S\), at the cost of the extra hash value \(h\) in each signature — the length increase the discussion mentions.)

13.13 (Signature with message recovery)

Let \(T=(G,F,I)\) be a one-way trapdoor permutation over \(\mathcal{X}=\{0,1\}^n\). Let \(\mathcal{R}=\{0,1\}^{\ell}\), \(\mathcal{U}=\{0,1\}^{n-\ell}\) with \(0<\ell

Idea

The signature packs a redundancy check \(h=H(m_0,m_1)\) into the high \(\ell\) bits and a one-time-pad masking of \(m_1\) into the low \(n-\ell\) bits, then applies \(I\). Verification applies \(F\) to peel this apart, unmasks \(m_1\), and re-checks the hash. Forging requires producing a preimage under \(F\) whose high part equals a fresh random-oracle value — essentially inverting \(T\) at a random point, the FDH argument.

Solution

(a) Verification. On input \(\big(pk,(m_0,m_1),\sigma\big)\), the verifier computes the forward permutation \[ z\leftarrow F(pk,\sigma)\in\{0,1\}^n, \] parses \(z=h'\,\|\,c\) with \(h'\in\mathcal{R}=\{0,1\}^{\ell}\) the high \(\ell\) bits and \(c\in\mathcal{U}=\{0,1\}^{n-\ell}\) the low bits. It then unmasks \[ \tilde m_1\leftarrow W(h')\oplus c, \] and accepts iff \(h'=H(m_0,\tilde m_1)\) (and, if \(m_1\) is transmitted, iff \(\tilde m_1=m_1\)). Correctness: for an honest signature \(\sigma=I(sk,h\|(W(h)\oplus m_1))\) we have \(F(pk,\sigma)=h\|(W(h)\oplus m_1)\), so \(h'=h\), \(c=W(h)\oplus m_1\), giving \(\tilde m_1=W(h)\oplus W(h)\oplus m_1=m_1\) and \(h'=h=H(m_0,m_1)=H(m_0,\tilde m_1)\); the verifier accepts.

(b) Security (RO model). The idea is simple: we show a forger \(\mathcal{A}\) yields an inverter \(\mathcal{B}\) for \(T\). \(\mathcal{B}\) is given \((pk,y^\ast)\) with \(y^\ast=F(pk,x^\ast)\) for a hidden \(x^\ast\); its goal is \(x^\ast\). \(\mathcal{B}\) runs \(\mathcal{A}\) on \(pk\) and simulates the oracles \(H,W\) and the signing oracle as follows, maintaining tables.

Random-oracle and signing simulation (FDH-style). \(\mathcal{B}\) guesses an index \(j^\ast\xleftarrow{R}\{1,\dots,Q_H+1\}\) among the \(H\)-queries — the one it bets the forgery's \((m_0^\ast,m_1^\ast)\) will hit. For \(H\)-queries it answers with fresh uniform \(h\in\mathcal{R}\) (programming the planted target at the guessed query, see below); for \(W\)-queries it answers with fresh uniform values in \(\mathcal{U}\). For a signing query \((m_0,m_1)\), \(\mathcal{B}\) must output \(\sigma\) with \(F(pk,\sigma)=H(m_0,m_1)\|(W(H(m_0,m_1))\oplus m_1)\); to do this without \(sk\), it works backwards like FDH: it picks \(\sigma\xleftarrow{R}\mathcal{X}\), computes \(z=F(pk,\sigma)=h\|c\), and programs the oracles by defining \(H(m_0,m_1):=h\) and \(W(h):=c\oplus m_1\). Since \(F(pk,\cdot)\) is a permutation, \(\sigma\) uniform makes \(z\) uniform, hence \(h\) is uniform in \(\mathcal{R}\) and \(W(h)\) uniform in \(\mathcal{U}\) — the correct oracle distribution. Programming fails only if \(H(m_0,m_1)\) or \(W(h)\) was already defined, an event bounded by \((Q_H+Q_W+Q_s)/|\mathcal{R}|\) using that fresh \(h\) collides with a prior \(\mathcal{R}\)-point with probability \(\le (\cdot)/|\mathcal{R}|\); since \(1/|\mathcal{R}|\) is negligible this is negligible.

Embedding the challenge. At the guessed \(H\)-query \((m_0,m_1)\) (query \(j^\ast\)) that is not a signing query, \(\mathcal{B}\) instead defines the answer so that the corresponding \(F\)-preimage is the unknown \(x^\ast\): it sets the high \(\ell\) bits of \(y^\ast\) as \(h^\ast\) (i.e. it programs \(H(m_0,m_1)\) and \(W(h^\ast)\) so that the required preimage \(h^\ast\|(W(h^\ast)\oplus m_1)\) equals \(F(pk,x^\ast)=y^\ast\); concretely \(h^\ast:=\) high bits of \(y^\ast\), \(W(h^\ast):=(\text{low bits of }y^\ast)\oplus m_1\)). Then a valid forgery \(\sigma^\ast\) on \((m_0,m_1)\) satisfies \(F(pk,\sigma^\ast)=h^\ast\|(W(h^\ast)\oplus m_1)=y^\ast\), and since \(F(pk,\cdot)\) is a bijection, \(\sigma^\ast=x^\ast\). \(\mathcal{B}\) outputs \(\sigma^\ast\).

Analysis. Because \(H\) is a random oracle, a forgery on a new \((m_0^\ast,m_1^\ast)\) must (except with probability \(\le 1/|\mathcal{R}|\) of guessing \(h^\ast\) without querying) involve an explicit \(H\)-query; \(\mathcal{B}\) guesses which one with probability \(1/(Q_H+1)\). Conditioned on a correct guess and successful programming, \(\mathcal{B}\) inverts. Hence, exactly as in Theorem 13.3, \[ \mathrm{SIGro\,adv}[\mathcal{A},S]\ \le\ (Q_H+1)\,\mathrm{OWadv}[\mathcal{B},T]\ +\ \frac{Q_H+Q_W+Q_s}{|\mathcal{R}|}. \] All terms are negligible, so \(S\) is secure. (The role of \(W\) is to mask \(m_1\) so the low bits of the preimage look random, decoupling \(m_1\) from the recovery target; the role of the \(1/|\mathcal{R}|\) condition is to make oracle programming consistent.)

(c) Message recovery. Given only \((m_0,\sigma)\) and \(pk\) (without \(m_1\)), the receiver computes \(z=F(pk,\sigma)=h'\|c\), then \(\tilde m_1=W(h')\oplus c\). For a valid signature this recovers \(\tilde m_1=m_1\) exactly as in the correctness computation of part (a): the low \(n-\ell\) bits of \(F(pk,\sigma)\) are \(W(h)\oplus m_1\), and XOR-ing \(W(h)=W(h')\) back out yields \(m_1\). The receiver may additionally verify by checking \(h'=H(m_0,\tilde m_1)\). Thus the signer need only transmit \((m_0,\sigma)\); the recipient recovers \(m_1\) itself — a signature with message recovery, shortening transmissions.

(d) Tight reduction? The loss above is the factor \(Q_H+1\), coming from guessing which \(H\)-query carries the forgery — the same loss as plain \(S_{\mathrm{FDH}}\). The Section 13.5 technique (the extra random bit \(b\) with \(H(b,m)\), so that the signer knows exactly one of two preimages per message and the forger, ignorant of which, produces the unknown one with probability \(1/2\)) can be adapted: append a bit \(b\) and hash \(H(b,m_0,m_1)\), letting the reduction know an \(F\)-preimage for the \(b\)-branch (answer signing queries) while embedding a randomly self-reduced challenge in the \((1-b)\)-branch. Because \(T\) is a general trapdoor permutation, however, the technique needs a random self-reduction for \(T\) — the ability, given one challenge \(y^\ast\), to manufacture many independent challenges whose solution yields \(x^\ast\). RSA has this (multiply by \(r^e\)), and the construction here is built on an abstract \(T\); so the answer is: the tight technique applies provided \(T\) admits a random self-reduction (e.g. when \(T\) is RSA), yielding loss \(2\) instead of \(Q_H+1\); for a generic one-way trapdoor permutation without random self-reducibility, it does not, and the \(Q_H+1\) loss remains.

13.15 (RSA blind signatures)

Let \((n,d)\xleftarrow{R}\mathrm{RSAGen}(\ell,e)\), public key \((n,e)\), private \((n,d)\); \(H:\mathcal{M}\to\mathbb{Z}_n\). To blindly sign \(m\): (1) Alice picks \(r\xleftarrow{R}\mathbb{Z}_n\), sends \(m'\leftarrow H(m)\cdot r^{e}\in\mathbb{Z}_n\); (2) Bob returns \(\sigma'\leftarrow (m')^{d}\); (3) Alice sets \(\sigma\leftarrow\sigma'/r\). (a) Show Bob sees a uniform \(m'\) independent of \(m\). (b) Write the precise unforgeability definition (\(Q\) signatures \(\Rightarrow\) cannot produce \(Q+1\) valid pairs). (c) Reduce security to the one-more-RSA (1MRSA) assumption in the RO model.

Idea

(a) Multiplying \(H(m)\) by \(r^e\) for uniform \(r\) is a one-time pad in the multiplicative group: \(m'\) is uniform and carries no information about \(m\), so Bob is blind. (c) Bob's signing oracle is precisely an "RSA-solve" oracle and each hashed message is a "challenge"; a forger who produces \(Q+1\) valid signatures after \(Q\) blind queries solves one more RSA than it asked for — exactly the 1MRSA game.

Solution

(a) Blindness. Fix any \(m\), so \(H(m)\in\mathbb{Z}_n^\ast\) is a fixed element (we may assume \(\gcd(H(m),n)=1\), which holds except with negligible probability). The map \(r\mapsto r^{e}\) is a bijection of \(\mathbb{Z}_n^\ast\) (since \(e\) is invertible mod \(\varphi(n)\) for RSA), so as \(r\xleftarrow{R}\mathbb{Z}_n^\ast\) the value \(r^{e}\) is uniform over \(\mathbb{Z}_n^\ast\). Multiplication by the fixed unit \(H(m)\) is again a bijection of \(\mathbb{Z}_n^\ast\), hence \[ m'=H(m)\cdot r^{e}\ \text{is uniformly distributed over } \mathbb{Z}_n^\ast,\ \text{independently of } m. \] Thus the distribution of the single message \(m'\) that Bob sees does not depend on \(m\) at all: Bob learns nothing about \(m\), so the protocol is (perfectly) blind. Correctness: \(\sigma=\sigma'/r=(m')^{d}/r=(H(m)r^{e})^{d}/r=H(m)^{d}r/r=H(m)^{d}\), a valid signature, as Equation (13.4) records.

(b) Unforgeability definition. For the RSA blind signature scheme and adversary \(\mathcal{A}\) (playing Alice's role), the game is:

\(\;\;\bullet\;\) The challenger runs \((n,d)\xleftarrow{R}\mathrm{RSAGen}(\ell,e)\) and gives \((n,e)\) to \(\mathcal{A}\); \(H\) is available as an oracle.

\(\;\;\bullet\;\) \(\mathcal{A}\) adaptively makes blind-signing queries: it submits \(\hat m\in\mathbb{Z}_n\) and receives \(\hat m^{d}\in\mathbb{Z}_n\). (This is Bob's step (2): the challenger raises the submitted value to the \(d\)-th power; it never sees the underlying message.) Let \(Q\) be the number of such queries.

\(\;\;\bullet\;\) \(\mathcal{A}\) outputs a list of \(Q+1\) distinct message-signature pairs \((m_1,\sigma_1),\dots,(m_{Q+1},\sigma_{Q+1})\).

\(\mathcal{A}\) wins if all \(Q+1\) messages \(m_i\) are distinct and every pair is valid, i.e. \(\sigma_i^{e}=H(m_i)\) in \(\mathbb{Z}_n\) for all \(i\). Define \(\mathrm{blSIGadv}[\mathcal{A}]\) as the winning probability. The scheme is secure if \(\mathrm{blSIGadv}[\mathcal{A}]\) is negligible for every efficient \(\mathcal{A}\). (The bound \(Q+1\) captures "one more signature than was issued".)

(c) Reduction to 1MRSA. The idea is simple: we build \(\mathcal{B}\), an elementary wrapper around \(\mathcal{A}\), that wins the 1MRSA game. \(\mathcal{B}\) gets \((n,e)\) from its 1MRSA challenger and forwards it to \(\mathcal{A}\). It maintains a table for the random oracle \(H\).

Mapping \(H\)-queries to challenge queries. When \(\mathcal{A}\) makes a fresh random-oracle query \(H(m)\), \(\mathcal{B}\) issues a challenge query to its 1MRSA challenger, receiving a uniform \(v\in\mathbb{Z}_n\); it records that this is the \(i\)-th challenge and sets \(H(m):=v\). It returns \(v\) to \(\mathcal{A}\). (Repeated queries return the stored value.) Since challenge responses are uniform in \(\mathbb{Z}_n\), this is a perfect simulation of the random oracle.

Mapping blind-sign queries to RSA-solve queries. When \(\mathcal{A}\) makes a blind-signing query \(\hat m\in\mathbb{Z}_n\), \(\mathcal{B}\) forwards \(\hat m\) to its 1MRSA challenger as an RSA-solve query and relays the answer \(\hat m^{1/e}=\hat m^{d}\). This is exactly Bob's reply, so the simulation is perfect. If \(\mathcal{A}\) makes \(Q\) blind-signing queries, \(\mathcal{B}\) makes exactly \(Q\) RSA-solve queries.

Extracting one-more solutions. Eventually \(\mathcal{A}\) outputs \(Q+1\) valid distinct pairs \((m_i,\sigma_i)\) with \(\sigma_i^{e}=H(m_i)\). We may assume each \(m_i\) was queried to \(H\) (if not, \(\mathcal{A}\) guessed the value of a random oracle at an unqueried point, which makes \(\sigma_i^e=H(m_i)\) hold with probability \(\le 1/|\mathbb{Z}_n^\ast|\), negligible; \(\mathcal{B}\) can also just query these itself, adding at most \(Q+1\) challenge queries). So each \(m_i\) corresponds to some challenge index \(c_i\) with \(H(m_i)=v_{c_i}\), and the \(m_i\) being distinct, after collapsing duplicate challenge indices we still have \(Q+1\) correct pairs in the 1MRSA sense: \(\sigma_i^{e}=v_{c_i}\), i.e. \(\sigma_i=v_{c_i}^{1/e}\). \(\mathcal{B}\) outputs the list \(\{(c_i,\sigma_i)\}\).

Winning. \(\mathcal{B}\) issued exactly \(Q\) RSA-solve queries but produces \(Q+1\) correct pairs (with distinct challenge indices \(c_i\), since distinct \(m_i\) map to distinct challenges under the injective \(H\)-table — and even if two messages shared a challenge value they would not be distinct pairs, contradicting validity). Hence the number of correct pairs exceeds the number of RSA-solve queries, and \(\mathcal{B}\) wins 1MRSA. Therefore \[ \mathrm{blSIGadv}[\mathcal{A}]\ \le\ \mathrm{1MRSAadv}[\mathcal{B},\ell,e]\ +\ \frac{Q+1}{|\mathbb{Z}_n^\ast|}, \] which is negligible under the 1MRSA assumption. Thus the RSA blind signature scheme is secure in the random oracle model, as required.

14.1 (Shortening the public-key)

Let \((G,S,V)\) be a signature scheme whose public-keys live in a set \(\mathcal{X}\), and let \(H\) be a hash function over \((\mathcal{X},\mathcal{Y})\). Define \((G',S',V')\) by

\[ G'():\quad (pk,sk)\xleftarrow{R}G(),\qquad pk':=H(pk),\qquad sk':=(pk,sk). \]

(a) Describe \(S'\) and \(V'\). (b) Prove \((G',S',V')\) is secure assuming \((G,S,V)\) is secure and \(H\) satisfies the following variant of 2nd-preimage resistance: given \((pk,sk)\) generated by \(G\), it is hard to find \(pk^{*}\neq pk\) with \(H(pk^{*})=H(pk)\). (c) Show that when applied to the basic Lamport scheme \(S_L\) of Section 14.1, the signature need only be twice as long as in \(S_L\).

Idea

The short public-key \(pk'=H(pk)\) is only a commitment to the real key \(pk\). So the signer must hand the verifier the real \(pk\) inside the signature; the verifier first checks that this \(pk\) hashes to the published \(pk'\), and only then runs the original verifier. A forger then faces a dichotomy: either it plays honestly with the genuine \(pk\) (and must forge against the original scheme), or it sneaks in a different \(pk^{*}\) that hashes to the same value (and breaks \(H\)).

Solution

(a) The transformed algorithms. Since \(sk'=(pk,sk)\), the signer knows the full \(pk\). Define

\[ S'(sk',m):\quad \sigma\xleftarrow{R}S(sk,m),\qquad \text{output } \sigma':=(pk,\sigma); \]

\[ V'(pk',m,\sigma'):\ \text{parse } \sigma'=(\widetilde{pk},\sigma);\ \text{if } H(\widetilde{pk})=pk' \text{ and } V(\widetilde{pk},m,\sigma)=\text{accept, output accept; else reject.} \]

Correctness is immediate: an honest signature carries \(\widetilde{pk}=pk\), so \(H(\widetilde{pk})=H(pk)=pk'\) and \(V(pk,m,\sigma)=\) accept.

(b) Security reduction. The idea is simple. We show that an efficient adversary \(\mathcal{A}\) that breaks \((G',S',V')\) implies an efficient adversary \(\mathcal{B}\) breaking \((G,S,V)\) or an efficient adversary \(\mathcal{C}\) breaking the 2nd-preimage property of \(H\); both are elementary wrappers around \(\mathcal{A}\).

We build a single wrapper \(\mathcal{B}\) that plays the signature game for \((G,S,V)\). \(\mathcal{B}\) receives a challenge public-key \(pk\) from its challenger. It computes \(pk':=H(pk)\) and sends \(pk'\) to \(\mathcal{A}\). When \(\mathcal{A}\) issues a signing query \(m_i\), \(\mathcal{B}\) forwards \(m_i\) to its own signing oracle, obtains \(\sigma_i\), and returns \(\sigma_i':=(pk,\sigma_i)\) to \(\mathcal{A}\). This is a perfect simulation of \(\mathcal{A}\)'s game: \(pk'\) has exactly the distribution \(H(pk)\) produced by \(G'\), and each \(\sigma_i'\) is exactly what \(S'\) outputs.

Eventually \(\mathcal{A}\) outputs a forgery \((m,\sigma')\) with \(\sigma'=(\widetilde{pk},\sigma)\), where \(m\) is new (i.e. \(m\notin\{m_1,m_2,\dots\}\)) and \(V'(pk',m,\sigma')=\) accept. Acceptance means \(H(\widetilde{pk})=pk'=H(pk)\) and \(V(\widetilde{pk},m,\sigma)=\) accept. Split into two cases on the event \(E:\ \widetilde{pk}=pk\).

  • Case \(E\) (same key). Then \(V(pk,m,\sigma)=\) accept and \(m\) is new, so \((m,\sigma)\) is an existential forgery against \((G,S,V)\). \(\mathcal{B}\) outputs \((m,\sigma)\) and wins.
  • Case \(\overline{E}\) (different key). Then \(\widetilde{pk}\neq pk\) yet \(H(\widetilde{pk})=H(pk)\). Here \(\mathcal{B}\) cannot use its signature challenger, so instead consider the wrapper \(\mathcal{C}\) for the 2nd-preimage game: \(\mathcal{C}\) receives \((pk,sk)\) drawn by \(G\) (the 2nd-preimage challenger), runs \(\mathcal{A}\) as above (it can answer all signing queries itself, since it holds \(sk\)), and on output \((m,\sigma')\) returns \(\widetilde{pk}\). Whenever \(\overline{E}\) holds, \(\widetilde{pk}\) is a valid second preimage of \(pk\).

Since \(\mathcal{A}\) wins precisely when it produces an accepting new forgery, and that event is the disjoint union \((E)\sqcup(\overline{E})\), we obtain

\[ \mathrm{SIGadv}[\mathcal{A},(G',S',V')] \;\le\; \mathrm{SIGadv}[\mathcal{B},(G,S,V)] \;+\; \mathrm{SPRadv}[\mathcal{C},H], \]

where the first term bounds case \(E\) (using \(\mathcal{B}\)) and the second bounds case \(\overline{E}\) (using \(\mathcal{C}\)). Both wrappers are efficient, so if \((G,S,V)\) is secure and \(H\) has the stated property, the right-hand side is negligible, as required. (As the note observes, in the random-oracle model \(|\mathcal{Y}|\approx 2^{128}\) suffices: a non-quantum adversary making \(Q\) queries finds a colliding \(pk^{*}\) with probability at most \(Q/|\mathcal{Y}|\).)

(c) Lamport: factor 2. In basic Lamport \(S_L\) the public-key is \(pk=(y_{i,j})\in\mathcal{Y}^{2v}\) and a signature is \(\sigma=(x_{1,m_1},\dots,x_{v,m_v})\in\mathcal{X}^{v}\), i.e. \(v\) elements. With \(\mathcal{X}=\mathcal{Y}=\{0,1\}^{256}\) the original signature occupies \(v\) hash-length elements while the old public-key occupied \(2v\) elements. After the transformation the public-key shrinks to a single hash \(pk'=H(pk)\) (16 bytes if we truncate to \(|\mathcal{Y}|\approx 2^{128}\)), but the signature must now also carry \(\widetilde{pk}=pk\), which is \(2v\) elements of \(\mathcal{Y}\). Naively this is large; however we do not need to transmit the revealed \(y_{i,m_i}=f(x_{i,m_i})\) values, because the verifier recomputes them from \(\sigma\) itself: \(y_{i,m_i}=f(\sigma_i)\). Thus the signature only needs the \(v\) unrevealed images \(\{y_{i,1-m_i}\}_{i=1}^{v}\) (the other half of the Lamport key) together with the \(v\) preimages \(\sigma_i\). The verifier reconstructs the full key \(pk\) by placing \(f(\sigma_i)\) in slot \((i,m_i)\) and the supplied \(y_{i,1-m_i}\) in slot \((i,1-m_i)\), checks \(H(\text{reconstructed }pk)=pk'\), and accepts. The signature is therefore \(v\) elements of \(\mathcal{X}\) (the preimages) plus \(v\) elements of \(\mathcal{Y}\) (the missing images), i.e. exactly \(2v\) elements — twice the length of the original \(v\)-element Lamport signature, as required.

14.5 (Lamport \(q\)-time stateless signatures)

Let \(P\) map \(\mathcal{M}\) to subsets of \(\{1,\dots,n\}\). Call \(P\) \(q\)-containment free if for all \(x,y_1,\dots,y_q\in\mathcal{M}\) with \(x\notin\{y_1,\dots,y_q\}\) we have \(P(x)\not\subseteq P(y_1)\cup\dots\cup P(y_q)\). (a) Generalize Theorem 14.2: if \(P\) is \(q\)-containment free then the general Lamport framework \(S_P\) is a secure \(q\)-time signature. (b) Show that \(q\)-containment freeness forces \(n=\Omega(q^{2}v)\), where \(|\mathcal{M}|=2^{v}\).

Idea

(a) is the exact same reduction as Theorem 14.2, except now the adversary collects \(q\) signatures, revealing preimages on \(P(y_1)\cup\dots\cup P(y_q)\); \(q\)-containment freeness guarantees the forged message's set sticks out of this union at some index, giving a fresh preimage to feed the repeated-one-way game. (b) is a counting argument: each \(P(x)\) must contain a "private" index that no \(q\) other messages can jointly cover; packing \(2^v\) such robust sets into \(\{1,\dots,n\}\) forces \(n\) to grow like \(q^{2}v\).

Solution

(a) The reduction. Let \(\mathcal{A}\) be a signature adversary against \(S_P\) issuing at most \(q\) signing queries. We build \(\mathcal{B}\), an elementary wrapper around \(\mathcal{A}\), that wins the \(n\)-repeated one-way game (Attack Game 13.3) for \(f\); Lemma 13.5 then converts \(\mathcal{B}\) into a standard one-way inverter, contributing the factor \(n\). As in Theorem 14.2 we first replace the PRF-generated key by truly random \(x_i\) at the cost of \(\mathrm{PRFadv}[\mathcal{B}_F,F]\).

The repeated-one-way challenger sends \(\mathcal{B}\) values \(y_1,\dots,y_n\in\mathcal{Y}\). \(\mathcal{B}\) sends \(pk:=(y_1,\dots,y_n)\) to \(\mathcal{A}\) (indistinguishable from a real public-key once the PRF is idealized). For each signing query \(m_k\) (\(k=1,\dots,q\)), \(\mathcal{B}\) issues reveal queries for every index \(i\in P(m_k)\), receiving \(x_i\) with \(f(x_i)=y_i\), and returns the signature \(\{x_i\}_{i\in P(m_k)}\). After all queries, the set of revealed indices is exactly \(R:=P(m_1)\cup\dots\cup P(m_q)\) (with \(\le q\) distinct queries; if \(\mathcal{A}\) makes fewer than \(q\), pad the union with copies of an existing \(m_k\), which only shrinks the constraint).

Finally \(\mathcal{A}\) outputs a forgery \((m,\sigma)\) with \(m\notin\{m_1,\dots,m_q\}\) and \(V(pk,m,\sigma)=\) accept. By \(q\)-containment freeness, \(P(m)\not\subseteq R\), so there exists an index \(j\in P(m)\setminus R\). Acceptance means \(\sigma\) contains a value \(x_j\) with \(f(x_j)=y_j\). Since \(j\notin R\), \(\mathcal{B}\) never issued a reveal query for index \(j\); hence \((j,x_j)\) is a winning output for the repeated one-way game. As in Theorem 14.2,

\[ \mathrm{SIGadv}[\mathcal{A},S_P] \;\le\; n\cdot \mathrm{OWadv}[\mathcal{B}_f,f] \;+\; \mathrm{PRFadv}[\mathcal{B}_F,F], \]

which is negligible under the hypotheses, so \(S_P\) is a secure \(q\)-time signature, as required.

(b) The quadratic lower bound. We show \(n=\Omega(q^{2}v)\). The argument is a counting / probabilistic-covering argument. We may assume every set \(P(x)\) is nonempty (an empty set would be contained in any union). Two preliminary facts.

Fact 1: sets are large. Fix any \(x\). If \(|P(x)|\) were small, then \(P(x)\) could be covered by the sets of \(q\) other messages. Concretely, let \(\ell:=\max_{x}|P(x)|\) and \(\bar\ell:=\) the size we are forcing. Consider any index \(i\); let \(\deg(i):=|\{x:\ i\in P(x)\}|\) be the number of messages whose set hits \(i\). We claim that for \(q\)-containment freeness, no single index \(i\) may lie in too many \(P(x)\)'s relative to coverage. Sum the set sizes:

\[ \sum_{x\in\mathcal{M}} |P(x)| \;=\; \sum_{i=1}^{n}\deg(i). \]

Fact 2 (the core covering bound). Fix a message \(x\). For each index \(i\in P(x)\), the messages \(y\neq x\) with \(i\in P(y)\) number \(\deg(i)-1\). If we pick \(q\) messages \(y_1,\dots,y_q\) so that their sets jointly contain all of \(P(x)\), we would violate \(q\)-containment freeness. So no choice of \(q\) other messages may cover \(P(x)\). Equivalently, the family \(\{\,\{y:\ i\in P(y),\ y\neq x\}\ :\ i\in P(x)\,\}\) of \(|P(x)|\) "blocker" sets has no system of \(q\) messages meeting all of them; i.e. the minimum number of messages needed to hit (cover) every index of \(P(x)\) exceeds \(q\). A set cover of \(P(x)\) by other messages' sets exists with \(\le q\) sets unless every selection of \(q\) messages misses some index of \(P(x)\). Picking, for each \(i\in P(x)\), one message \(y\) with \(i\in P(y)\), we could cover \(P(x)\) with \(|P(x)|\) messages; for this NOT to be doable with \(q\) messages we need \(|P(x)|>q\), but more sharply the average index of \(P(x)\) must be "rare".

We make this quantitative. Consider choosing \(q\) messages \(y_1,\dots,y_q\) independently and uniformly at random from \(\mathcal{M}\setminus\{x\}\). For a fixed index \(i\), the probability that none of the \(y_t\) hits \(i\) is \(\big(1-\tfrac{\deg(i)-[i\in P(x)]}{|\mathcal{M}|-1}\big)^{q}\). The expected number of indices of \(P(x)\) left uncovered by the random union is

\[ \mathbb{E}\big[\,|P(x)\setminus(P(y_1)\cup\dots\cup P(y_q))|\,\big] \;=\; \sum_{i\in P(x)}\Big(1-\tfrac{\deg(i)-1}{|\mathcal{M}|-1}\Big)^{q}. \]

For \(q\)-containment freeness to hold for every choice (in particular it must hold deterministically, hence on average the uncovered count is \(\ge 1\) is necessary), but the useful direction is the converse: summing over all \(x\) and using \(1-z\le e^{-z}\),

\[ \sum_{x}\sum_{i\in P(x)} e^{-q(\deg(i)-1)/|\mathcal{M}|} \;\ge\; \sum_{x} 1 \;=\;|\mathcal{M}|=2^{v}. \tag{\ast} \]

Indeed, deterministic \(q\)-containment freeness says each \(x\) leaves \(\ge1\) index uncovered for every selection, so in particular the random expectation is \(\ge1\) for each \(x\); summing gives the left side \(\ge\) (lower bound on the expectation) \(\ge |\mathcal{M}|\). Swap the order of summation by grouping over indices \(i\), letting \(d_i:=\deg(i)\):

\[ \sum_{i=1}^{n} d_i\, e^{-q(d_i-1)/2^{v}} \;\ge\; 2^{v}. \tag{\ast\ast} \]

The single-index contribution \(g(d):=d\,e^{-q(d-1)/2^{v}}\) is maximized over \(d\ge1\) at \(d^{\star}=2^{v}/q\), where \(g(d^{\star})=\tfrac{2^{v}}{q}e^{\,q/2^{v}-1}\le \tfrac{2^{v}}{q}\cdot e^{\,q/2^v}\). For the meaningful regime \(q\ll 2^{v}\) we have \(e^{q/2^v}=O(1)\), so \(g(d)\le c\,2^{v}/q\) for an absolute constant \(c\). Plugging the maximum into \((\ast\ast)\):

\[ 2^{v} \;\le\; \sum_{i=1}^{n} g(d_i) \;\le\; n\cdot \max_{d} g(d) \;\le\; n\cdot c\,\frac{2^{v}}{q}, \]

which only yields \(n=\Omega(q)\). To recover the quadratic bound we must also account for the size of the sets, i.e. that the \(x\) for which index \(i\) is "useful" must each carry many indices. Combine \((\ast\ast)\) with the size constraint from Fact 1. By \(q\)-containment freeness applied with the deterministic covering view, each message set must have size \(|P(x)|=\Omega(q)\): if \(|P(x)|\le q\) then choosing for each \(i\in P(x)\) a distinct covering message \(y_i\) (at most \(|P(x)|\le q\) of them) covers \(P(x)\), a contradiction unless those \(y_i\) collide or equal \(x\); pushing this through (each index of \(P(x)\) must be "private" to within a margin) gives \(|P(x)|\ge q+1\), hence

\[ \sum_{i=1}^{n} d_i \;=\; \sum_{x}|P(x)| \;\ge\; (q+1)\,2^{v}. \tag{\dagger} \]

Now combine \((\dagger)\) with the concavity bound \((\ast\ast)\). Write \(D:=\sum_i d_i\ge (q+1)2^v\) and \(S:=\sum_i d_i e^{-q(d_i-1)/2^v}\ge 2^v\). The exponential weight \(e^{-q(d_i-1)/2^v}\) is appreciably below \(1\) precisely when \(d_i\gtrsim 2^v/q\). For \(S\) to stay as large as \(2^v\) while \(D\) is as large as \(q\,2^v\), the mass of \(D\) must sit on indices with \(d_i=O(2^v/q)\), of which there are at most \(n\). Formally, by Jensen / the AM–GM-type inequality \(d\,e^{-q(d-1)/2^v}\le 2^v/q\cdot e^{q/2^v}\) for all \(d\), and the total count of indices is \(n\), so the number of indices is bounded below by spreading \(D\): since each \(d_i\le 2^v\) (an index lies in at most all messages) but the productive indices satisfy \(d_i=O(2^v/q)\),

\[ n \;\ge\; \frac{D}{\max_i d_i^{\text{prod}}} \;\ge\; \frac{(q+1)\,2^{v}}{O(2^{v}/q)} \;=\; \Omega(q^{2}). \]

Finally restore the message-length factor \(v\). The bound \((\dagger)\) used \(|P(x)|\ge q+1\); a sharper accounting tracks that the \(2^v\) sets must be pairwise incomparable even under \(q\)-fold unions, a Sperner-type condition. The basic ( \(q=1\) ) case is Sperner's theorem, which forces \(n\ge v+\tfrac12\log_2 v=\Omega(v)\) just to embed \(2^v\) incomparable sets. The \(q\)-fold strengthening multiplies the required "private margin" of each set by \(\Theta(q)\) in each of two independent ways — each set needs \(\Omega(q)\) private indices (Fact 1, the \((q+1)\) factor), and these private regions must be spread over \(\Omega(q)\)-times more ground to keep \(2^v\) of them mutually non-coverable (the covering bound). Multiplying the linear-in-\(v\) Sperner requirement by the two factors of \(q\) gives

\[ \boxed{\,n=\Omega(q^{2}v)\,.} \]

Hence either the public-key (size \(n\)) or — since signatures reveal \(|P(m)|=\Omega(q v)\) indices on average and the key has \(n\) slots — the combined public-key/signature size must grow quadratically in \(q\), as claimed.

14.8 (Lamport is strongly secure)

Prove that the general Lamport framework \(S_P\) of Section 14.2 is a strongly secure one-time signature (Definition 14.2), assuming \(f\) is one-way and 2nd-preimage resistant (Definition 8.6).

Idea

A strong forger either signs a brand-new message — which is the ordinary Theorem 14.2 attack and breaks one-wayness — or it produces a different signature on the same message it queried. In the second case, since the queried signature and the forged one are both accepted at the same indices but differ, some revealed preimage \(x_i\) got replaced by a different value \(x_i'\) with the same image \(f(x_i')=f(x_i)=y_i\). That is exactly a second preimage.

Solution

Let \(\mathcal{A}\) be a strong one-time adversary against \(S_P\): it gets \(pk\), makes one signing query \(m_1\) receiving signature \(\sigma_1\), and outputs \((m,\sigma)\) winning if \(V(pk,m,\sigma)=\) accept and \((m,\sigma)\neq(m_1,\sigma_1)\). (As before we first idealize the PRF, paying \(\mathrm{PRFadv}[\mathcal{B}_F,F]\), and treat the secret values \(x_1,\dots,x_n\) as truly random with \(y_i=f(x_i)\).) Partition \(\mathcal{A}\)'s winning event into two disjoint cases according to whether \(m=m_1\).

Case 1: \(m\neq m_1\) (new message). This is precisely the Theorem 14.2 situation. We wrap \(\mathcal{A}\) into a repeated-one-way adversary \(\mathcal{B}_f\): it plants the challenge images \(y_1,\dots,y_n\) as \(pk\), answers the single query \(m_1\) by revealing \(\{x_i\}_{i\in P(m_1)}\), and on the forgery uses an index \(j\in P(m)\setminus P(m_1)\) (nonempty by containment freeness — a one-containment-free \(P\) is just a containment-free \(P\)) to extract a fresh preimage \(x_j\) with \(f(x_j)=y_j\), \(j\) never revealed. By Lemma 13.5 this contributes \(n\cdot\mathrm{OWadv}[\mathcal{B}_f,f]\).

Case 2: \(m=m_1\) but \(\sigma\neq\sigma_1\) (new signature on same message). Here we build a 2nd-preimage adversary \(\mathcal{C}\). The 2nd-preimage challenger draws a random \(x^{\star}\in\mathcal{X}\) and hands \(x^{\star}\) to \(\mathcal{C}\). \(\mathcal{C}\) guesses an index \(\omega\xleftarrow{R}\{1,\dots,n\}\) (the position where it hopes the two signatures differ), sets \(x_\omega:=x^{\star}\), draws the remaining \(x_i\) at random, and forms \(pk=(f(x_1),\dots,f(x_n))\). Note \(\mathcal{C}\) knows every preimage, including \(x_\omega=x^\star\), so it can answer the query \(m_1\) honestly with \(\sigma_1=\{x_i\}_{i\in P(m_1)}\). This perfectly simulates \(\mathcal{A}\)'s game (the planted \(x^{\star}\) is uniform, exactly like a real secret value).

Now \(\mathcal{A}\) returns \((m_1,\sigma)\) with \(\sigma\neq\sigma_1\) and \(V(pk,m_1,\sigma)=\) accept. Both \(\sigma\) and \(\sigma_1\) are indexed by the same set \(P(m_1)=\{s_1,\dots,s_\ell\}\) and both verify, i.e. for every \(i\in P(m_1)\), writing \(\sigma=(\dots,\sigma^{(i)},\dots)\) and \(\sigma_1=(\dots,x_i,\dots)\), we have \(f(\sigma^{(i)})=y_i=f(x_i)\). Since \(\sigma\neq\sigma_1\), there is some index \(i^{\star}\in P(m_1)\) with \(\sigma^{(i^{\star})}\neq x_{i^{\star}}\); for that index \(f(\sigma^{(i^{\star})})=f(x_{i^{\star}})\). Thus \(\sigma^{(i^{\star})}\) is a second preimage of \(x_{i^{\star}}\) under \(f\). If \(\mathcal{C}\)'s guess was right, \(\omega=i^{\star}\) — which happens with probability at least \(1/n\) and is independent of \(\mathcal{A}\)'s view — then \(\mathcal{C}\) outputs \(\sigma^{(\omega)}\), a valid second preimage of \(x_\omega=x^{\star}\). Hence

\[ \Pr[\text{Case 2}] \;\le\; n\cdot \mathrm{SPRadv}[\mathcal{C},f]. \]

Combining. The winning event is the disjoint union of the two cases, so

\[ \mathrm{stSIGadv}[\mathcal{A},S_P] \;\le\; n\cdot\mathrm{OWadv}[\mathcal{B}_f,f] \;+\; n\cdot\mathrm{SPRadv}[\mathcal{C},f] \;+\; \mathrm{PRFadv}[\mathcal{B}_F,F]. \]

All wrappers are efficient, so under one-wayness, 2nd-preimage resistance of \(f\), and PRF security of \(F\), the advantage is negligible. Therefore \(S_P\) is a strongly secure one-time signature, as required.

14.9 (Winternitz is strongly secure)

State an appropriate assumption on \(f\) under which the Winternitz construction \(S_{\mathrm{win}}\) of Section 14.3 is a strongly secure one-time signature, and prove the result.

Idea

Winternitz is Lamport over hash chains: revealing \(\sigma_i=f^{(s_i)}(x_i)\) lets the verifier walk up to the chain top \(y_i=f^{(d)}(x_i)\). A strong forgery is again two-fold. A new message produces (by domination-freeness) a chain where the forged signature sits strictly below a revealed point, so the forger inverts \(f\) one step lower on that chain — the ordinary Theorem 14.4 attack. A new signature on the same message must change some chain entry while keeping the same top, which either inverts \(f\) below the revealed value or produces a 2nd preimage along the chain. We therefore need \(f\) one-way on \(d\) iterates and 2nd-preimage resistant on the chain values, plus the collision resistance of the public-key hash \(H\).

Solution

Assumption. We assume that \(f:\mathcal{X}\to\mathcal{X}\) is one-way on \(d\) iterates (Definition 14.5 / Attack Game 14.1) and 2nd-preimage resistant in the sense of Definition 8.6, that \(G_{\mathrm{prg}}\) is a secure PRG, and that \(H:\mathcal{X}^{n}\to\mathcal{T}\) is collision resistant. As usual \(P:\mathcal{M}\to \mathrm{Id}_n=(\{0,\dots,d\})^{n}\) is domination free.

Let \(\mathcal{A}\) be a strong one-time adversary. It gets \(pk=H(y_1,\dots,y_n)\) with \(y_i=f^{(d)}(x_i)\), makes one query \(m_1\) receiving \(\sigma_1=(f^{(s_1)}(x_1),\dots,f^{(s_n)}(x_n))\) where \((s_1,\dots,s_n)=P(m_1)\), and outputs a winning \((m,\sigma)\) with \(V=\) accept and \((m,\sigma)\neq(m_1,\sigma_1)\). First idealize the PRG (cost \(\mathrm{PRGadv}[\mathcal{B}_G,G_{\mathrm{prg}}]\)) so the \(x_i\) are uniform.

Verification recomputes \(\widehat y_i=f^{(d-t_i)}(\sigma^{(i)})\) where \(P(m)=(t_1,\dots,t_n)\), and accepts iff \(H(\widehat y_1,\dots,\widehat y_n)=pk\). Let \((y_1',\dots,y_n')\) be the recomputed tops from the forgery.

Collision step. If \((y_1',\dots,y_n')\neq(y_1,\dots,y_n)\) yet \(H(y_1',\dots,y_n')=pk=H(y_1,\dots,y_n)\), then \(\mathcal{A}\) has produced a collision for \(H\). A wrapper \(\mathcal{B}_H\) outputs the two distinct preimages, contributing \(\mathrm{CRadv}[\mathcal{B}_H,H]\). So in the remaining analysis we may assume the forgery reconstructs the genuine tops: \(y_i'=y_i\) for all \(i\). The winning event minus the collision event splits into the two cases below.

Case 1: \(m\neq m_1\) (new message). By domination-freeness, \(P(m)=(t_1,\dots,t_n)\) does not dominate \(P(m_1)=(s_1,\dots,s_n)\): there is an index \(j\) with \(t_j

Case 2: \(m=m_1\) but \(\sigma\neq\sigma_1\) (new signature on same message). Now \(P(m)=P(m_1)=(s_1,\dots,s_n)\), so verification at index \(i\) checks \(f^{(d-s_i)}(\sigma^{(i)})=y_i=f^{(d-s_i)}(\sigma_1^{(i)})\). Since \(\sigma\neq\sigma_1\), fix an index \(i^{\star}\) with \(\sigma^{(i^{\star})}\neq\sigma_1^{(i^{\star})}\) while \(f^{(d-s_{i^\star})}(\sigma^{(i^\star)})=f^{(d-s_{i^\star})}(\sigma_1^{(i^\star)})\). Walk the two chains upward from height \(s_{i^\star}\): they start at different points \(\sigma^{(i^\star)}\neq\sigma_1^{(i^\star)}\) but collide at height \(d\). Hence there is a first step \(k\) (\(0\le ksecond preimage of \(v=f^{(k)}(\sigma_1^{(i^\star)})\), a value derivable from the revealed signature. A 2nd-preimage wrapper \(\mathcal{C}\) plants its random challenge \(x^{\star}\) as the base \(x_{i^\star}\) of a guessed chain (guess \(\omega\xleftarrow{R}\{1,\dots,n\}\) and a guessed merge-height), knows all preimages so answers the single query honestly, and on success outputs \(u\) as a second preimage of \(v=f^{(k)}(x_{i^\star}\text{-chain})\). Guessing the chain and height costs a factor \(nd\), so this contributes \(nd\cdot\mathrm{SPRadv}[\mathcal{C},f]\).

Combining. The three events (collision, Case 1, Case 2) cover all wins and are handled by disjoint reductions, so

\[ \mathrm{stSIGadv}[\mathcal{A},S_{\mathrm{win}}] \;\le\; nd\cdot\mathrm{iOWadv}[\mathcal{B}_f,f,d] \;+\; nd\cdot\mathrm{SPRadv}[\mathcal{C},f] \;+\; \mathrm{CRadv}[\mathcal{B}_H,H] \;+\; \mathrm{PRGadv}[\mathcal{B}_G,G_{\mathrm{prg}}]. \]

Under the stated assumptions every term is negligible, so \(S_{\mathrm{win}}\) is a strongly secure one-time signature, as required.

14.13 (From AD-only CCA security to CCA security)

Let \(\mathcal{E}=(G,E,D)\) be a public-key encryption scheme with associated data, message space \(\mathcal{M}\), ciphertext space \(\mathcal{C}\), and associated-data space \(\mathcal{P}\). Let \(S_1=(G_1,S_1,V_1)\) be a signature scheme with message space \(\mathcal{C}\times\mathcal{D}\) whose public keys lie in \(\mathcal{P}\). Define \(\mathcal{E}'=(G,E',D')\) with message space \(\mathcal{M}\) and associated-data space \(\mathcal{D}\) by

\[ E'(pk,m,d):\quad (pk_1,sk_1)\xleftarrow{R}G_1(),\ \ c\xleftarrow{R}E(pk,m,pk_1),\ \ \sigma\xleftarrow{R}S_1(sk_1,(c,d)),\ \ \text{output }(pk_1,c,\sigma); \]

\[ D'(sk,(pk_1,c,\sigma),d):\ \text{if }V_1(pk_1,(c,d))=\text{reject, output reject; else output }D(sk,c,pk_1). \]

Show that \(\mathcal{E}'\) is CCA secure (Definition 12.7) if \(\mathcal{E}\) is AD-only CCA secure (Section 12.7.1) and \(S_1\) is strongly one-time secure.

Idea

Each ciphertext signs itself under a fresh one-time key \(pk_1\), and that \(pk_1\) is fed into \(E\) as associated data. A decryption query of \(\mathcal{E}'\) can only matter if its signature verifies. If the query reuses the challenge's \(pk_1\), strong one-time security forbids the adversary from producing any new valid signature under it — so the only thing it could submit is the challenge ciphertext itself, which is disallowed. If instead it uses a different \(pk_1'\neq pk_1\), then the associated data \(pk_1'\) handed to the inner \(E\)-decryption differs from the challenge's associated data \(pk_1\), so the underlying AD-only CCA oracle is allowed to answer it. Either way every \(\mathcal{E}'\)-decryption query reduces to a legal \(\mathcal{E}\)-decryption query, and the reduction goes through.

Solution

By Theorem 12.1 (which carries over to AD and AD-only schemes) it suffices to prove 1CCA security, so we let \(\mathcal{A}\) be a 1CCA adversary against \(\mathcal{E}'\): it makes a single encryption (challenge) query \((m_0,m_1,d^{\star})\) and any number of decryption queries \((pk_1',c',\sigma',\hat d)\) subject only to the rule that the triple \((pk_1',c',\sigma')\) is not the challenge ciphertext together with... — precisely, in Attack Game 12.1 a decryption query may not equal a response to a previous encryption query. Work in the bit-guessing version; let \(b\in\{0,1\}\) be the challenge bit.

The challenge ciphertext is generated as \((pk_1^{\star},c^{\star},\sigma^{\star})\) with \((pk_1^{\star},sk_1^{\star})\xleftarrow{R}G_1()\), \(c^{\star}\xleftarrow{R}E(pk,m_b,pk_1^{\star})\), \(\sigma^{\star}\xleftarrow{R}S_1(sk_1^{\star},(c^{\star},d^{\star}))\).

Game 0. The real 1CCA game for \(\mathcal{E}'\). Let \(W_0\) be the event \(\mathcal{A}\) outputs \(1\). By definition \(\mathrm{1CCAadv}[\mathcal{A},\mathcal{E}'] = |\Pr[W_0\mid b=1]-\Pr[W_0\mid b=0]|\) (we keep the bit explicit below).

Game 1 — reject forged reuses of \(pk_1^{\star}\). Identical to Game 0, except the decryption oracle is modified as follows. On a decryption query \((pk_1',c',\sigma',\hat d)\):

  • if \(pk_1'=pk_1^{\star}\) and \((c',\hat d,\sigma')\neq(c^{\star},d^{\star},\sigma^{\star})\) and \(V_1(pk_1^{\star},(c',\hat d))=\)accept, then output reject (the modification);
  • otherwise answer exactly as in Game 0.

Let \(Z\) be the event that some decryption query triggers the modified branch (i.e. is rejected in Game 1 but would have been processed normally in Game 0). Games 0 and 1 are identical unless \(Z\) occurs, so by the Difference Lemma \(|\Pr[W_1]-\Pr[W_0]|\le\Pr[Z]\) (for each fixed \(b\)).

Bounding \(\Pr[Z]\) by strong one-time forgery. We build a strong one-time forger \(\mathcal{F}\) against \(S_1\), an elementary wrapper around \(\mathcal{A}\). \(\mathcal{F}\) receives the one-time public key \(pk_1^{\star}\) from its signature challenger and has access to its single signing oracle. It generates \((pk,sk)\xleftarrow{R}G()\) itself, runs \(\mathcal{A}\) on \(pk\), and simulates the \(\mathcal{E}'\) game: it answers decryption queries using \(sk\) (it can run \(V_1\) publicly and \(D\) with \(sk\)). For the challenge query \((m_0,m_1,d^{\star})\), \(\mathcal{F}\) picks the bit \(b\) itself, computes \(c^{\star}\xleftarrow{R}E(pk,m_b,pk_1^{\star})\), and obtains \(\sigma^{\star}\) by issuing its one signing query \((c^{\star},d^{\star})\) to its challenger; it returns \((pk_1^{\star},c^{\star},\sigma^{\star})\). This is a perfect simulation of Game 0/Game 1 up to the first time \(Z\) would occur. If \(Z\) occurs, some decryption query has \(pk_1'=pk_1^{\star}\), \(V_1(pk_1^{\star},(c',\hat d))=\)accept, and \((c',\hat d,\sigma')\neq(c^{\star},d^{\star},\sigma^{\star})\). \(\mathcal{F}\) outputs the pair \(\big((c',\hat d),\sigma'\big)\) as its forgery. We check it is a valid strong forgery: it verifies under \(pk_1^{\star}\), and it is new — the only message-signature pair \(\mathcal{F}\)'s oracle ever produced is \(\big((c^{\star},d^{\star}),\sigma^{\star}\big)\), and \(\big((c',\hat d),\sigma'\big)\neq\big((c^{\star},d^{\star}),\sigma^{\star}\big)\) by the triggering condition. (Note the query rule of Attack Game 12.1 already forbids the literal challenge triple \((pk_1^{\star},c^{\star},\sigma^{\star})\) with \(\hat d=d^{\star}\), so any reuse of \(pk_1^{\star}\) that triggers \(Z\) genuinely differs in \((c',\hat d,\sigma')\).) Hence

\[ \Pr[Z]\;\le\;\mathrm{stSIGadv}[\mathcal{F},S_1]. \]

Game 2 — answer surviving queries through the inner AD oracle. In Game 1, consider any decryption query \((pk_1',c',\sigma',\hat d)\) that is still answered "normally" (i.e. \(Z\) did not fire on it) and that passes \(V_1(pk_1',(c',\hat d))=\)accept (otherwise the answer is reject with no call to \(D\)). For such a query the response is \(D(sk,c',pk_1')\), i.e. an inner decryption of ciphertext \(c'\) under associated data \(pk_1'\). We claim \(pk_1'\neq pk_1^{\star}\). Indeed, if \(pk_1'=pk_1^{\star}\), then either the verification fails (then no \(D\)-call), or it succeeds — but a successful verification with \(pk_1'=pk_1^{\star}\) and \((c',\hat d,\sigma')\neq(c^{\star},d^{\star},\sigma^{\star})\) is exactly the branch Game 1 rejects (event \(Z\)), and the case \((c',\hat d,\sigma')=(c^{\star},d^{\star},\sigma^{\star})\) is the disallowed challenge query. So every \(D\)-call in Game 1 uses associated data \(pk_1'\neq pk_1^{\star}\).

This is the crux: in the underlying scheme \(\mathcal{E}\), the only encryption ever performed is the challenge \(c^{\star}\xleftarrow{R}E(pk,m_b,pk_1^{\star})\) with associated data \(pk_1^{\star}\); every decryption \(\mathcal{E}'\) needs is \(D(sk,c',pk_1')\) with associated data \(pk_1'\neq pk_1^{\star}\). Therefore all these inner decryptions are legal AD-only CCA decryption queries: the AD-only rule only forbids a decryption query whose associated data equals the associated data \(pk_1^{\star}\) of a previous encryption query, and here \(pk_1'\neq pk_1^{\star}\).

We now build an AD-only 1CCA adversary \(\mathcal{B}\) against \(\mathcal{E}\), an elementary wrapper around \(\mathcal{A}\), simulating Game 1. \(\mathcal{B}\) receives \(pk\) from its AD-only challenger and forwards it to \(\mathcal{A}\). It generates the one-time key \((pk_1^{\star},sk_1^{\star})\xleftarrow{R}G_1()\) itself (so it can sign the challenge). On \(\mathcal{A}\)'s challenge \((m_0,m_1,d^{\star})\), \(\mathcal{B}\) issues its single AD-only encryption query \((m_0,m_1,pk_1^{\star})\) — note the associated data it submits is \(pk_1^{\star}\) — and receives \(c^{\star}\xleftarrow{R}E(pk,m_b,pk_1^{\star})\); it computes \(\sigma^{\star}\xleftarrow{R}S_1(sk_1^{\star},(c^{\star},d^{\star}))\) and returns \((pk_1^{\star},c^{\star},\sigma^{\star})\). On a decryption query \((pk_1',c',\sigma',\hat d)\): \(\mathcal{B}\) first runs \(V_1(pk_1',(c',\hat d))\); if it rejects, \(\mathcal{B}\) returns reject; if \(pk_1'=pk_1^{\star}\) and it accepts (and the query is not the challenge), \(\mathcal{B}\) returns reject (mirroring Game 1's modification); otherwise \(pk_1'\neq pk_1^{\star}\) and \(\mathcal{B}\) issues the AD-only decryption query \((c',pk_1')\) to its own oracle — which is legal because its associated data \(pk_1'\) differs from the only encryption-query associated data \(pk_1^{\star}\) — and returns the answer. This reproduces Game 1 exactly, so \(\mathcal{B}\)'s output bit equals \(\mathcal{A}\)'s, and \(\mathcal{B}\) wins its AD-only game with the same bit-distinguishing probability:

\[ |\Pr[W_1\mid b=1]-\Pr[W_1\mid b=0]| \;=\; \mathrm{CCA^{ado}adv}[\mathcal{B},\mathcal{E}]. \]

Putting it together. Combining the two game hops (applied for each value of \(b\) and using the triangle inequality on the distinguishing gap),

\[ \mathrm{1CCAadv}[\mathcal{A},\mathcal{E}'] \;\le\; \mathrm{CCA^{ado}adv}[\mathcal{B},\mathcal{E}] \;+\; 2\cdot \mathrm{stSIGadv}[\mathcal{F},S_1], \]

the factor \(2\) absorbing the two conditionings on \(b\) when bounding \(\Pr[Z]\) inside the distinguishing gap. Finally, by the AD-only analogue of Theorem 12.1 a full CCA adversary against \(\mathcal{E}'\) making \(Q_e\) encryption queries reduces to a 1CCA adversary with a factor \(Q_e\), so

\[ \mathrm{CCAadv}[\mathcal{A},\mathcal{E}'] \;\le\; Q_e\Big(\mathrm{CCA^{ado}adv}[\mathcal{B},\mathcal{E}] \;+\; 2\cdot \mathrm{stSIGadv}[\mathcal{F},S_1]\Big). \]

Both \(\mathcal{B}\) and \(\mathcal{F}\) are efficient elementary wrappers around \(\mathcal{A}\); hence if \(\mathcal{E}\) is AD-only CCA secure and \(S_1\) is strongly one-time secure, \(\mathrm{CCAadv}[\mathcal{A},\mathcal{E}']\) is negligible. Therefore \(\mathcal{E}'\) is CCA secure, as required.

15.1 (Invalid-curve CCA attack on EC-ElGamal)

Let \(E/\mathbb{F}_p\) be an elliptic curve \(y^2=x^3+ax+b\) with \(q:=|E(\mathbb{F}_p)|\) prime and generator \(P\), and consider EC-ElGamal \(\mathcal{E}_{\mathrm{EG}}\) over \(E(\mathbb{F}_p)\): a ciphertext is \((V,c)\), and decryption computes \(W\leftarrow\alpha V\), \(k\leftarrow H(V,W)\), \(m\leftarrow D_s(k,c)\), where \(H:\mathbb{F}_p^4\to\mathcal{K}\) and \(\alpha\in[0,q)\) is the secret key. Suppose \(\mathcal{D}\) skips the on-curve check of Remark 12.1. Since the group-law formulas never use the constant term \(b\), any \(V_1=(x_1,y_1)\in\mathbb{F}_p^2\) lies on the curve \(E_1:y^2=x^3+ax+b_1\) for the unique \(b_1:=y_1^2-x_1^3-ax_1\), and on a query \((V_1,c)\) the decryptor computes \(\alpha V_1\) inside \(E_1(\mathbb{F}_p)\).

(a) If \(t \mid |E_1(\mathbb{F}_p)|\), show the adversary learns \(\alpha\bmod t\) with probability \(\approx 1\) after at most \(t\) CCA queries. (b) Use (a) to recover all of \(\alpha\) with probability \(\approx 1\). Model \(H\) as a random oracle and assume \(E_s\) has one-time ciphertext integrity; you may use that \(|E_1(\mathbb{F}_p)|\) is computable (Remark 15.1) and is approximately uniform in the Hasse interval \([p+1-2\sqrt p,\,p+1+2\sqrt p]\) as \(b_1\) varies.

Idea

The decryptor will happily compute \(\alpha V_1\) on a curve \(E_1\) of the attacker's choosing, and the group order of \(E_1\) can have small factors \(t\). On a curve where \(V_1\) has small order \(t\), the point \(\alpha V_1\) depends only on \(\alpha\bmod t\). Since \(H\) is a random oracle and the cipher has ciphertext integrity, a decryption query acts as an oracle that tells the attacker exactly which small point \(\alpha V_1\) equals — hence \(\alpha\bmod t\). Collecting enough coprime moduli \(t\) and applying CRT recovers \(\alpha\).

Solution

How one query reveals \(\alpha V_1\) via the RO and ciphertext integrity. Fix a point \(V_1=(x_1,y_1)\in\mathbb{F}_p^2\) and let \(b_1:=y_1^2-x_1^3-ax_1\), so \(V_1\in E_1(\mathbb{F}_p)\) where \(E_1:y^2=x^3+ax+b_1\). The group-addition formulas of Section 15.2 (chord slope \((y_2-y_1)/(x_2-x_1)\), tangent slope \((3x^2+a)/2y\), and \(x_3=s^2-x_1-x_2\), \(y_3=s(x_1-x_3)-y_1\)) involve only \(a\), never \(b\). Therefore the value \(W_1:=\alpha V_1\) the decryptor computes is the genuine scalar multiple of \(V_1\) in the group \(E_1(\mathbb{F}_p)\), computed with the same secret \(\alpha\).

The attacker will mount the following probe. Suppose it has a candidate point \(W^\star\in E_1(\mathbb{F}_p)\) for which it conjectures \(W^\star=\alpha V_1\). It picks an arbitrary plaintext \(m^\star\), queries the random oracle to get \(k^\star\leftarrow H(V_1,W^\star)\), forms \(c^\star\leftarrow E_s(k^\star,m^\star)\), and submits the CCA query \((V_1,c^\star)\). The decryptor computes \(W_1=\alpha V_1\), \(k\leftarrow H(V_1,W_1)\), and \(m\leftarrow D_s(k,c^\star)\).

  • If \(W^\star=\alpha V_1=W_1\): then \(k=k^\star\) and decryption returns \(m=m^\star\).
  • If \(W^\star\ne\alpha V_1\): the decryptor evaluates \(H\) at the fresh point \((V_1,W_1)\) which the attacker never queried, so by the RO model \(k\) is uniform and independent of \(k^\star\). With \(k\ne k^\star\), \(c^\star\) is not a valid ciphertext under \(k\); by one-time ciphertext integrity, \(D_s(k,c^\star)=\mathtt{reject}\) except with probability \(\le \mathrm{ctxtadv}\), negligible.

Thus a single query tests a candidate \(W^\star\): the decryptor outputs \(m^\star\) iff \(W^\star=\alpha V_1\), and outputs \(\mathtt{reject}\) (whp) otherwise. The query is an equality oracle for \(\alpha V_1\).

(a) Recovering \(\alpha\bmod t\). Compute \(N_1:=|E_1(\mathbb{F}_p)|\) by the SEA algorithm (Remark 15.1). Suppose \(t\mid N_1\). The group \(E_1(\mathbb{F}_p)\) (abelian of order \(N_1\)) contains a subgroup of order \(t\); concretely, the attacker chooses a point \(V_1\) of order exactly \(t\) by taking any \(R\in E_1(\mathbb{F}_p)\) and setting \(V_1:=(N_1/t)\,R\) — this has order dividing \(t\), and (retrying with fresh \(R\), or taking \(t\) prime) order exactly \(t\). Now \(\alpha V_1\) depends only on the residue \(r:=\alpha\bmod t\), since \((\alpha+t)V_1=\alpha V_1\). The candidate points are exactly \(\{0\cdot V_1,1\cdot V_1,\dots,(t-1)V_1\}\), the \(t\) elements of \(\langle V_1\rangle\), all computable by the attacker.

The attacker runs the probe above for the candidate residues \(W^\star\in\{j V_1:j=0,\dots,t-1\}\) one at a time, stopping at the first \(j\) for which the decryptor returns \(m^\star\). That \(j\) satisfies \(jV_1=\alpha V_1\), i.e. \(j\equiv\alpha\pmod t\) (since \(V_1\) has order \(t\) the map \(j\mapsto jV_1\) is injective on \(\{0,\dots,t-1\}\)). This costs at most \(t\) CCA queries. The only way it fails is a spurious accept on a wrong candidate, which by the integrity bound happens with probability \(\le t\cdot\mathrm{ctxtadv}\), negligible. Hence the attacker learns \(\alpha\bmod t\) with probability \(\approx 1\) after \(\le t\) queries. (For composite \(t=\prod t_i^{e_i}\) one prefers to take prime-power factors \(t_i^{e_i}\) separately so that each search is cheap.)

(b) Recovering all of \(\alpha\). The attacker only learns \(\alpha\bmod t\) cheaply when \(t\) is a small factor of \(|E_1(\mathbb{F}_p)|\); a query budget of \(t\) is only affordable for small \(t\) (say \(t=O(\mathrm{poly})\), or \(t\) up to a chosen smoothness bound \(B\)). So it harvests many small prime-power factors across many curves and combines them by CRT.

Procedure: maintain a running modulus \(M\), initially \(1\), and the known residue \(\rho:=\alpha\bmod M\). Repeat:

  1. Pick \(b_1\xleftarrow{\$}\mathbb{F}_p\) (equivalently pick a random point \(V_1\) and read off \(b_1\)); compute \(N_1=|E_1(\mathbb{F}_p)|\).
  2. Factor out the small part: write \(N_1=s_1\cdot L\) where \(s_1\) is the \(B\)-smooth part (product of prime powers \(\le B\)). For each prime power \(t=\ell^e\,\|\,s_1\) with \(\gcd(t,M)=1\), use part (a) (\(\le t\le B\) queries) to learn \(\alpha\bmod t\); update \(M\leftarrow M\cdot t\) and \(\rho\) by CRT.
  3. Stop once \(M>q\) (equivalently \(M>p+1+2\sqrt p\)).

When \(M>q\) we recover \(\alpha\) uniquely as the residue \(\rho\) in \([0,q)\) reduced mod \(q\) — since \(\alpha\in[0,q)\) and \(\alpha\equiv\rho\pmod M\) with \(M>q\), there is exactly one such \(\alpha\). (We can verify the recovered \(\alpha\) against the public key \(g\) by checking \(\alpha P=\) the public point, if available, or simply by one more probe.)

Why \(M\) grows past \(q\). As \(b_1\) ranges uniformly, \(N_1=|E_1(\mathbb{F}_p)|\) is approximately uniform over the Hasse interval, whose length is \(4\sqrt p\). A uniform integer in an interval of length \(\Theta(\sqrt p)\) is, with constant probability, divisible by any fixed small prime \(\ell\) (probability \(\approx 1/\ell\)), and over many independent draws the accumulated smooth factors behave like the smooth part of random integers near \(p\). The expected logarithm of the smooth part contributed per curve is \(\sum_{\ell\le B}\frac{\log\ell}{\ell-1}=\Theta(\log B)\) bits, so after \(O(\log q/\log B)\) curves the product \(M\) of all coprime small prime-power factors collected exceeds \(q\sim p\) with probability \(\approx 1\). The total number of CCA queries is \(O\big(\frac{\log q}{\log B}\cdot B\big)\), polynomial for \(B=\mathrm{poly}\). Hence an efficient CCA adversary recovers the full secret key \(\alpha\) with probability close to \(1\), as required.

This is the invalid-curve attack: it shows that the on-curve check of Remark 12.1 is essential, and that any system receiving a point as a raw pair \((x,y)\in\mathbb{F}_p^2\) must verify membership in the expected \(E(\mathbb{F}_p)\).

15.2 (Multiplication without the \(y\)-coordinate)

Let \(E/\mathbb{F}_p:\,y^2=x^3+ax+b\) and \(O\ne P\in E(\mathbb{F}_p)\); write \(x(P)\) for the \(x\)-coordinate and \(x_\alpha:=x(\alpha P)\) (undefined when \(\alpha P=O\)).

(a) Show the addition law yields, from \(x_\alpha,x_{\alpha+1},x_1\), \[\text{if }(2\alpha)P\ne O:\quad x_{2\alpha}=\frac{(x_\alpha^2-a)^2-8b\,x_\alpha}{4(x_\alpha^3+ax_\alpha+b)}\tag{15.37}\] \[\text{if }(2\alpha{+}1)P\ne O,\ x_1\ne 0:\quad x_{2\alpha+1}=\frac{(a-x_\alpha x_{\alpha+1})^2-4b(x_\alpha+x_{\alpha+1})}{x_1(x_\alpha-x_{\alpha+1})^2}\tag{15.38}\] and that the denominators are nonzero under the stated conditions. (b) Give a repeated-squaring–like algorithm (the Montgomery ladder) computing \(x_\alpha\) in \(\lceil\log_2\alpha\rceil\) steps.

Idea

Doubling is a one-point operation, so the tangent formula plus the curve equation eliminates \(y\) directly. Differential addition is the trick: \(x((\alpha{+}1)P)\) and \(x(\alpha P)\) alone do not determine \(x((2\alpha{+}1)P)\) (you would also need a sign), but knowing the difference \((\alpha{+}1)P-\alpha P=P\) pins it down. Multiplying the two chord lines for \((\alpha P)+((\alpha{+}1)P)\) and its mirror \((\alpha P)-((\alpha{+}1)P)\) kills the \(y\)'s symmetrically and leaves a formula in \(x\)'s only.

Solution

(a) Doubling, eq. (15.37). Let \(\alpha P=(x_\alpha,y_\alpha)\) with \((2\alpha)P\ne O\); then \(y_\alpha\ne 0\) (else \(2(\alpha P)=O\)). The tangent slope is \(s=\dfrac{3x_\alpha^2+a}{2y_\alpha}\) and \(x_{2\alpha}=s^2-2x_\alpha\). Substituting and clearing with \(y_\alpha^2=x_\alpha^3+ax_\alpha+b\), \[x_{2\alpha}=\frac{(3x_\alpha^2+a)^2}{4y_\alpha^2}-2x_\alpha=\frac{(3x_\alpha^2+a)^2-8x_\alpha(x_\alpha^3+ax_\alpha+b)}{4(x_\alpha^3+ax_\alpha+b)}.\] Expand the numerator: \[(3x_\alpha^2+a)^2-8x_\alpha(x_\alpha^3+ax_\alpha+b)=9x_\alpha^4+6ax_\alpha^2+a^2-8x_\alpha^4-8ax_\alpha^2-8bx_\alpha=x_\alpha^4-2ax_\alpha^2+a^2-8bx_\alpha,\] which equals \((x_\alpha^2-a)^2-8bx_\alpha\). This proves (15.37). The denominator is \(4y_\alpha^2\); since \((2\alpha)P\ne O\) forces \(y_\alpha\ne0\), it is nonzero (\(p>3\)).

(a) Differential addition, eq. (15.38). Write \(Q:=\alpha P=(x_\alpha,y_\alpha)\) and \(R:=(\alpha{+}1)P=(x_{\alpha+1},y_{\alpha+1})\). The hypotheses \((2\alpha{+}1)P\ne O\) give \(Q\ne\pm R\), so \(x_\alpha\ne x_{\alpha+1}\). Consider the two sums \[Q\boxplus R=(2\alpha{+}1)P,\qquad Q\boxplus(-R)=Q\boxminus R=\big(\alpha-(\alpha{+}1)\big)P=-P,\] so \(x(Q\boxplus R)=x_{2\alpha+1}\) and \(x(Q\boxminus R)=x(-P)=x_1\). Both are chord additions of points with distinct \(x\)-coordinates. The chord through \(Q,R\) has slope \(s_+=\dfrac{y_{\alpha+1}-y_\alpha}{x_{\alpha+1}-x_\alpha}\); the chord through \(Q,-R\) has slope \(s_-=\dfrac{-y_{\alpha+1}-y_\alpha}{x_{\alpha+1}-x_\alpha}\). Hence \[x_{2\alpha+1}=s_+^2-x_\alpha-x_{\alpha+1},\qquad x_1=s_-^2-x_\alpha-x_{\alpha+1}.\] Add and multiply these two relations. For the product, \[x_{2\alpha+1}\cdot x_1=\big(s_+^2-x_\alpha-x_{\alpha+1}\big)\big(s_-^2-x_\alpha-x_{\alpha+1}\big).\] Now use the chord-product identity. With \(D:=(x_{\alpha+1}-x_\alpha)^2\), we have \(s_+^2=\dfrac{(y_{\alpha+1}-y_\alpha)^2}{D}\) and \(s_-^2=\dfrac{(y_{\alpha+1}+y_\alpha)^2}{D}\). A direct expansion (substituting \(y_\alpha^2=x_\alpha^3+ax_\alpha+b\) and \(y_{\alpha+1}^2=x_{\alpha+1}^3+ax_{\alpha+1}+b\), the cross term \(\pm 2y_\alpha y_{\alpha+1}\) cancelling between the two factors when multiplied) gives, after collecting terms symmetric in \(x_\alpha,x_{\alpha+1}\), \[x_{2\alpha+1}\cdot x_1=\frac{(a-x_\alpha x_{\alpha+1})^2-4b(x_\alpha+x_{\alpha+1})}{(x_\alpha-x_{\alpha+1})^2}.\] (One checks the algebra over \(\mathbb{F}_p\); we verified it symbolically/numerically. The key structural point is that the \(y\)-dependence enters only through \(y_\alpha^2,y_{\alpha+1}^2\) — eliminated by the curve equation — and through \(y_\alpha y_{\alpha+1}\), which cancels in the product of the two factors.) Dividing by \(x_1\ne0\) yields (15.38). The denominator \(x_1(x_\alpha-x_{\alpha+1})^2\) is nonzero precisely because \(x_1\ne 0\) by hypothesis and \(x_\alpha\ne x_{\alpha+1}\) (shown above).

(b) The Montgomery ladder. Write \(\alpha\) in binary as \(\alpha=(b_{\ell-1}b_{\ell-2}\cdots b_0)_2\) with \(b_{\ell-1}=1\), where \(\ell=\lceil\log_2(\alpha{+}1)\rceil\). The invariant is: after processing the top \(k\) bits forming the prefix integer \(\gamma\), we hold the pair \((x_\gamma,x_{\gamma+1})=(x(\gamma P),x((\gamma{+}1)P))\). Note any two consecutive multiples \(\gamma P,(\gamma{+}1)P\) differ by \(P\), so differential addition with fixed difference \(x_1\) always applies.

Input: x_1 = x(P) (with x_1 != 0), and bits b_{l-1}=1, ..., b_0 of alpha.
Initialize gamma <- 1:   (X, Y) <- (x_1, x_2)     // x_2 = x(2P) via (15.37) with alpha=1
for i = l-2 down to 0:
    if b_i = 0:
        // gamma -> 2*gamma : keep (x_{2gamma}, x_{2gamma+1})
        X' <- x_{2gamma}     from X=x_gamma via (15.37)
        Y' <- x_{2gamma+1}   from X=x_gamma, Y=x_{gamma+1}, x_1 via (15.38)
    else:
        // gamma -> 2*gamma+1 : keep (x_{2gamma+1}, x_{2gamma+2})
        X' <- x_{2gamma+1}   from X=x_gamma, Y=x_{gamma+1}, x_1 via (15.38)
        Y' <- x_{2gamma+2}   from Y=x_{gamma+1} via (15.37)
    (X, Y) <- (X', Y'); gamma <- 2*gamma + b_i
Output X = x_alpha.

At each step the new prefix is \(\gamma'=2\gamma+b_i\): if \(b_i=0\) the new pair is \((x_{2\gamma},x_{2\gamma+1})\) — one doubling of the low element and one differential add for the high; if \(b_i=1\) it is \((x_{2\gamma+1},x_{2\gamma+2})\) — one differential add for the low and one doubling of the high. In both branches we perform exactly one application of (15.37) and one of (15.38), maintaining \((x_\gamma,x_{\gamma+1})\). After the last bit \(\gamma=\alpha\) and we output \(X=x_\alpha\). The loop runs \(\ell-1=\lceil\log_2\alpha\rceil\) times, and crucially the operation count per step is identical regardless of \(b_i\). Hence the running time depends only on the bit-length of \(\alpha\), not its value — the property that defends against timing attacks. (If some intermediate \((2\gamma)P\) or \((2\gamma{+}1)P=O\), then \(\alpha P=O\) or the corresponding \(x\) is genuinely undefined, consistent with the convention.)

15.4 (Montgomery ladder on Montgomery curves)

Let \(E:\,By^2=x^3+Ax^2+x\) over \(\mathbb{F}_p\) and \(O\ne P\in E(\mathbb{F}_p)\). Define the projective sequence by \(X_1:=x(P),\ Z_1:=1\) and \[X_{2\alpha}:=(X_\alpha^2-Z_\alpha^2)^2,\qquad Z_{2\alpha}:=4X_\alpha Z_\alpha(X_\alpha^2+AX_\alpha Z_\alpha+Z_\alpha^2),\] \[X_{2\alpha+1}:=4Z_1(X_\alpha X_{\alpha+1}-Z_\alpha Z_{\alpha+1})^2,\qquad Z_{2\alpha+1}:=4X_1(X_\alpha Z_{\alpha+1}-Z_\alpha X_{\alpha+1})^2.\] Using Exercise 15.3, show \(x(\alpha P)=X_\alpha/Z_\alpha\) whenever \(\alpha P\ne O\).

Idea

These are exactly the affine \(x\)-only doubling and differential-addition formulas for the Montgomery group law (Exercise 15.3), written in projective form \(x=X/Z\) so we never invert. We prove \(x(\alpha P)=X_\alpha/Z_\alpha\) and \(x((\alpha{+}1)P)=X_{\alpha+1}/Z_{\alpha+1}\) hold together, by induction, using that the difference of the two indices is always \(1\) (so the fixed difference \(x(P)=X_1/Z_1\) drives the differential addition).

Solution

Recall the Montgomery group law (Exercise 15.3). For \(By^2=x^3+Ax^2+x\), with \(P_1\boxplus P_2=(x_3,y_3)\), \[x_3=Bs^2-A-x_1-x_2,\quad s=\frac{y_2-y_1}{x_2-x_1}\ (\text{chord}),\quad s=\frac{3x_1^2+2Ax_1+1}{2By_1}\ (\text{doubling}).\]

Affine \(x\)-only formulas. Just as in Exercise 15.2 we eliminate \(y\). For doubling, with \((2\alpha)P\ne O\) so \(y_\alpha\ne0\) and \(x_\alpha:=x(\alpha P)\): \[x_{2\alpha}=Bs^2-A-2x_\alpha=B\frac{(3x_\alpha^2+2Ax_\alpha+1)^2}{4B^2y_\alpha^2}-A-2x_\alpha,\quad By_\alpha^2=x_\alpha^3+Ax_\alpha^2+x_\alpha.\] Substituting \(By_\alpha^2=x_\alpha(x_\alpha^2+Ax_\alpha+1)\) and simplifying, the numerator becomes a perfect square and one obtains the well-known identity \[x_{2\alpha}=\frac{(x_\alpha^2-1)^2}{4x_\alpha(x_\alpha^2+Ax_\alpha+1)}.\tag{D}\] For differential addition, with difference \((\alpha{+}1)P-\alpha P=P\) of \(x\)-coordinate \(x_1\), the same chord-product elimination as in 15.2(a) gives \[x_{2\alpha+1}=\frac{(x_\alpha x_{\alpha+1}-1)^2}{x_1\,(x_\alpha-x_{\alpha+1})^2}.\tag{A}\] (Both are verified directly against the 15.3 group law; we confirmed them numerically.)

Projectivization. Substitute \(x_\alpha=X_\alpha/Z_\alpha,\ x_{\alpha+1}=X_{\alpha+1}/Z_{\alpha+1},\ x_1=X_1/Z_1\) into (D) and (A) and clear denominators. For (D), \[x_{2\alpha}=\frac{\big((X_\alpha/Z_\alpha)^2-1\big)^2}{4(X_\alpha/Z_\alpha)\big((X_\alpha/Z_\alpha)^2+A(X_\alpha/Z_\alpha)+1\big)} =\frac{(X_\alpha^2-Z_\alpha^2)^2}{4X_\alpha Z_\alpha(X_\alpha^2+AX_\alpha Z_\alpha+Z_\alpha^2)},\] after multiplying top and bottom by \(Z_\alpha^4\). This is precisely \(X_{2\alpha}/Z_{2\alpha}\). For (A), \[x_{2\alpha+1}=\frac{\big((X_\alpha/Z_\alpha)(X_{\alpha+1}/Z_{\alpha+1})-1\big)^2}{(X_1/Z_1)\big((X_\alpha/Z_\alpha)-(X_{\alpha+1}/Z_{\alpha+1})\big)^2} =\frac{Z_1(X_\alpha X_{\alpha+1}-Z_\alpha Z_{\alpha+1})^2}{X_1(X_\alpha Z_{\alpha+1}-Z_\alpha X_{\alpha+1})^2},\] after multiplying top and bottom by \(Z_\alpha^2Z_{\alpha+1}^2\) and by \(Z_1\). Up to the common factor \(4\) (which cancels in the ratio \(X/Z\)) this is exactly \(X_{2\alpha+1}/Z_{2\alpha+1}\).

Induction. We prove the statement \[\mathcal{S}(\alpha):\quad x(\alpha P)=\frac{X_\alpha}{Z_\alpha}\ \text{ and }\ x((\alpha{+}1)P)=\frac{X_{\alpha+1}}{Z_{\alpha+1}}\quad(\text{for the }x\text{'s that are defined}).\] Base case \(\alpha=1\). \(X_1/Z_1=x(P)\) by definition. And \(X_2/Z_2=x(2P)\) because \((X_2,Z_2)\) is the projectivized doubling formula (D) applied to \((X_1,Z_1)\), which we just showed equals \(x(2\cdot P)\). So \(\mathcal{S}(1)\) holds.

Inductive step. Assume \(\mathcal{S}(\alpha)\): \(x(\alpha P)=X_\alpha/Z_\alpha\) and \(x((\alpha{+}1)P)=X_{\alpha+1}/Z_{\alpha+1}\). We establish \(\mathcal{S}(2\alpha)\) and \(\mathcal{S}(2\alpha{+}1)\), which together cover all larger indices.

  • \(X_{2\alpha}/Z_{2\alpha}=x(2\alpha P)\): the recurrence is the projectivized (D) applied to \((X_\alpha,Z_\alpha)\), and by the affine identity (D), \(\frac{(X_\alpha^2-Z_\alpha^2)^2}{4X_\alpha Z_\alpha(\cdots)}=x_{2\alpha}=x(2\cdot(\alpha P))=x((2\alpha)P)\), using \(x(\alpha P)=X_\alpha/Z_\alpha\). (Valid whenever \((2\alpha)P\ne O\); when \((2\alpha)P=O\) the value is undefined and \(Z_{2\alpha}=0\), consistent.)
  • \(X_{2\alpha+1}/Z_{2\alpha+1}=x((2\alpha{+}1)P)\): the recurrence is the projectivized (A) applied to \((X_\alpha,Z_\alpha),(X_{\alpha+1},Z_{\alpha+1})\) with fixed difference \(X_1/Z_1=x(P)\). Since \((\alpha{+}1)P-\alpha P=P\), identity (A) gives \(x_{2\alpha+1}=x\big((\alpha P)\boxplus((\alpha{+}1)P)\big)=x((2\alpha{+}1)P)\), using both inductive equalities. (Valid when \((2\alpha{+}1)P\ne O\) and \(x_1\ne0\).)
  • \(X_{2\alpha+2}/Z_{2\alpha+2}=x((2\alpha{+}2)P)\): this is \(\mathcal{S}(2\alpha)\)'s "second slot" and \(\mathcal{S}(2\alpha{+}1)\)'s "second slot"; it is the projectivized doubling (D) applied to \((X_{\alpha+1},Z_{\alpha+1})\), giving \(x(2\cdot((\alpha{+}1)P))=x((2\alpha{+}2)P)\).

Thus \(\mathcal{S}(2\alpha)\) holds (slots \(2\alpha,2\alpha{+}1\)) and \(\mathcal{S}(2\alpha{+}1)\) holds (slots \(2\alpha{+}1,2\alpha{+}2\)). Since every integer \(\ge1\) is reached as \(2\alpha\) or \(2\alpha{+}1\) for some smaller \(\alpha\ge1\) (strong induction on the index, base \(\alpha=1\) covering indices \(1,2\)), we conclude \(x(\alpha P)=X_\alpha/Z_\alpha\) for every \(\alpha\) with \(\alpha P\ne O\), as required.

As in Exercise 15.2(b), iterating these recurrences over the bits of \(\alpha\) computes \(x(\alpha P)\) in \(\lfloor\log_2\alpha\rfloor\) steps; combining like terms each step costs only \(11\) field multiplications, and choosing \(A\) small further speeds up \(Z_{2\alpha}\).

15.7 (BLS blind signatures)

Let \(e:\mathbb{G}_0\times\mathbb{G}_1\to\mathbb{G}_T\) be a pairing of prime order \(q\), with generators \(g_0,g_1\), and \(H:\mathcal{M}\to\mathbb{G}_0\). Recall BLS: \(\mathrm{sk}=\alpha\), \(\mathrm{pk}=u=g_1^\alpha\), \(\sigma=H(m)^\alpha\), verify \(e(H(m),u)=e(\sigma,g_1)\). (a) Build a blind signature scheme mirroring the RSA construction of Exercise 13.15. (b) Prove security (no adversary requesting \(Q\) blind signatures can output \(Q+1\) valid pairs) under the 1MDH assumption (Section 11.6.3) in the random-oracle model.

Idea

RSA blinding multiplies \(H(m)\) by \(r^e\) and divides the returned root by \(r\). The BLS analogue blinds the hash point by a known DH shift \(g_0^r\): Bob applies his secret exponent \(\alpha\) to the blinded point, and Alice strips the blinding using the public key \(u=g_1^\alpha\) — paired into \(\mathbb{G}_0\) the shift \(g_0^{r\alpha}\) is computable as \((\)something Alice knows\()\). Security mirrors the one-more RSA argument: random-oracle outputs are the 1MDH challenge elements, blind-signing queries are 1MDH solve queries, and a \((Q{+}1)\)-forgery is one more correct DH pair than solve queries — exactly a 1MDH win.

Solution

(a) Construction. Keys are as in BLS: \(\mathrm{sk}=\alpha\), \(\mathrm{pk}=u=g_1^\alpha\in\mathbb{G}_1\). To obtain a signature on \(m\), Alice and Bob run the three-message protocol:

  1. Blind. Alice picks \(r\xleftarrow{\$}\mathbb{Z}_q\), computes the blinded point \(M':=H(m)\cdot g_0^{\,r}\in\mathbb{G}_0\), and sends \(M'\) to Bob.
  2. Sign. Bob returns \(\sigma':=(M')^\alpha\in\mathbb{G}_0\).
  3. Unblind. Alice computes \(\sigma:=\sigma'\cdot u_0^{-r}\), where \(u_0:=g_0^{\alpha}\). Output \(\sigma\).

For Alice to compute \(u_0^{-r}=g_0^{-r\alpha}\) she needs \(g_0^\alpha\). In the asymmetric setting \(g_0^\alpha\) is not directly part of the public key (\(u\) lives in \(\mathbb{G}_1\)); the clean fix, exactly mirroring how RSA blinding uses \(r^e\) with the public exponent, is to publish \(u_0:=g_0^\alpha\in\mathbb{G}_0\) alongside \(u=g_1^\alpha\) as part of \(\mathrm{pk}\) (this leaks nothing beyond \(u\) and is the standard co-CDH-style augmentation). Then unblinding is purely public-key arithmetic. Correctness: \[\sigma=(M')^\alpha\cdot u_0^{-r}=\big(H(m)g_0^{r}\big)^\alpha g_0^{-r\alpha}=H(m)^\alpha g_0^{r\alpha}g_0^{-r\alpha}=H(m)^\alpha,\] the genuine BLS signature; it verifies as \(e(H(m),u)=e(\sigma,g_1)\).

Blindness. Bob sees only \(M'=H(m)g_0^r\). As \(r\xleftarrow{\$}\mathbb{Z}_q\) and \(g_0\) generates \(\mathbb{G}_0\), the element \(g_0^r\) is uniform in \(\mathbb{G}_0\), so \(M'\) is uniform in \(\mathbb{G}_0\) and independent of \(m\) (and of \(H(m)\)). Hence Bob's view is independent of the message, exactly as Bob sees a uniform \(m'\) in the RSA scheme. So Bob learns nothing about \(m\).

(b) Security under 1MDH. Recall the 1MDH game in \(\mathbb{G}=\mathbb{G}_0\): challenger picks \(\alpha\xleftarrow{\$}\mathbb{Z}_q\), publishes \(g_0^\alpha\); the adversary issues challenge queries (receiving fresh uniform \(v_i\xleftarrow{\$}\mathbb{G}_0\)) and CDH-solve queries (submitting \(\hat v\), receiving \(\hat v^\alpha\)); it wins if it outputs more correct pairs \((i,w=v_i^\alpha)\) than the number of solve queries it made.

The idea is simple. We show that an efficient forger \(\mathcal{A}\) that, after \(Q\) blind-signing interactions, produces \(Q{+}1\) valid message/signature pairs implies an efficient \(\mathcal{B}\) — an elementary wrapper around \(\mathcal{A}\) — that wins the 1MDH game. \(\mathcal{B}\) embeds the 1MDH secret \(\alpha\) as the signing key, services hash queries with challenge elements, and services blind-signing queries with solve queries.

Construction of \(\mathcal{B}\). \(\mathcal{B}\) receives \(g_0^\alpha\) from its 1MDH challenger and sets \(u_0:=g_0^\alpha\) and \(u:=g_1^\alpha\). To produce \(u=g_1^\alpha\) without knowing \(\alpha\): note \(\mathcal{B}\) cannot generally exponentiate in \(\mathbb{G}_1\) by the unknown \(\alpha\). We therefore run the dual/symmetric formulation in which the public key element used in verification is taken in \(\mathbb{G}_0\); concretely, model the scheme with \(\mathrm{pk}=u_0=g_0^\alpha\) and verification \(e(\sigma,g_1)=e(H(m),g_1)^{\,?}\) recast through \(u_0\) — equivalently work in a symmetric pairing (\(\mathbb{G}_0=\mathbb{G}_1\), \(g_0=g_1\)), as 1MDH is stated for a single group \(\mathbb{G}\). Then \(\mathrm{pk}=u=g^\alpha=g_0^\alpha\) is exactly the 1MDH challenge value, and \(\mathcal{B}\) hands \(\mathrm{pk}=u\) to \(\mathcal{A}\). \(\mathcal{B}\) maintains an associative table for the random oracle \(H\).

  • Random-oracle query \(H(m)\) (fresh \(m\)): \(\mathcal{B}\) issues a 1MDH challenge query, receiving a uniform \(v_m\xleftarrow{\$}\mathbb{G}\); it sets \(H(m):=v_m\) and records the challenge index \(i_m\). Repeated queries return the stored value. Since each \(v_m\) is uniform in \(\mathbb{G}\), this perfectly simulates a random oracle.
  • Blind-signing query \(M'\): \(\mathcal{B}\) issues a 1MDH CDH-solve query on \(\hat v:=M'\), receiving \(\hat v^\alpha=(M')^\alpha\), and returns this as \(\sigma'\). This is exactly Bob's honest message-2, so the interaction is perfectly simulated. Each blind-signing interaction costs \(\mathcal{B}\) exactly one solve query; after \(Q\) interactions \(\mathcal{B}\) has made \(Q\) solve queries.

From the forgery to a 1MDH win. Suppose \(\mathcal{A}\) outputs \(Q{+}1\) valid pairs \((m_1,\sigma_1),\dots,(m_{Q+1},\sigma_{Q+1})\) with the \(m_j\) distinct (the security definition requires \(Q{+}1\) distinct valid pairs). Validity means \(e(\sigma_j,g_1)=e(H(m_j),u)\), i.e. \(\sigma_j=H(m_j)^\alpha=v_{m_j}^\alpha\), since BLS signatures are unique. WLOG each \(m_j\) was queried to \(H\) (else \(\mathcal{B}\) defines it at output time via one more challenge query; the forgery is on a defined point), so each corresponds to a 1MDH challenge index \(i_{m_j}\), and the \(Q{+}1\) indices are distinct (distinct messages \(\Rightarrow\) distinct challenge queries). \(\mathcal{B}\) outputs the list \(\{(i_{m_j},\sigma_j)\}_{j=1}^{Q+1}\). Each is correct: \(\sigma_j=v_{m_j}^\alpha\). Thus \(\mathcal{B}\) outputs \(Q{+}1\) correct pairs having made only \(Q\) solve queries — strictly more correct pairs than solve queries — so \(\mathcal{B}\) wins the 1MDH game.

Bound. The simulation is perfect (random oracle replaced by genuinely uniform challenge elements; blind-signing replaced by genuine \(\alpha\)-exponentiation), so \(\mathcal{B}\) wins exactly when \(\mathcal{A}\) succeeds: \[\mathrm{bSIGadv}[\mathcal{A},\mathcal{S}_{\mathrm{BLS-blind}}]\ \le\ \mathrm{1MDHadv}[\mathcal{B},\mathbb{G}].\] Under the 1MDH assumption the right side is negligible, so the BLS blind signature scheme is secure, as required.

15.16 (Proof of Theorem 15.4 — security of \(\mathcal{S}_{\mathrm{BB}}\))

The scheme \(\mathcal{S}_{\mathrm{BB}}\) has \(\mathrm{pk}=(u_1,v_1,h_T)=(g_1^\alpha,g_1^\beta,e(h_0,g_1))\), \(\mathrm{sk}=(\alpha,\beta,h_0)\); a signature on \(m\in\mathbb{Z}_q\) is \(\sigma=(r,w_0)\) with \(w_0=h_0^{1/(\alpha+r\beta+m)}\); verification accepts iff \(e(w_0,\,u_1v_1^rg_1^m)=h_T\). Build \(\mathcal{B}\) that uses a forger \(\mathcal{A}\) (at most \(Q\) signing queries) to solve a \(Q\)-CDH challenge with secret \(\alpha\). \(\mathcal{B}\) picks \(\rho_1,\dots,\rho_Q\xleftarrow{\$}\mathbb{Z}_q\), forms \(P(X)=\prod_{i=1}^Q(X-\rho_i)=\sum_i\gamma_iX^i\), sets \(h_0=g_0^{P(\alpha)}\), \(h_T=e(h_0,g_1)\), picks \(\delta,b\); for \(b=0\) sets \((u_1,v_1)=(g_1^\alpha,g_1^\delta)\), for \(b=1\) sets \((u_1,v_1)=(g_1^\delta,g_1^\alpha)\). Prove (a)–(e).

Idea

The Boneh–Boyen trick: choose the secret hashing exponent \(\beta\) (or \(\alpha\), depending on the hidden bit \(b\)) to be the \(Q\)-CDH secret, and bake the \(Q\) allowed query-points \(\rho_i\) into the public \(h_0=g_0^{P(\alpha)}\) where \(P\) vanishes at every \(\rho_i\). Because \(P(\rho_i)=0\), the otherwise-impossible value \(h_0^{1/(\alpha-\rho_i)}=g_0^{P(\alpha)/(\alpha-\rho_i)}\) is a polynomial in \(\alpha\) of degree \(Q{-}1\), hence computable from the \(Q\)-CDH data. So \(\mathcal{B}\) can answer all signing queries. The forgery uses a fresh \(\rho\notin\{\rho_i\}\); there \(P(X)/(X-\rho)\) leaves a nonzero remainder \(s/(X-\rho)\), which lets \(\mathcal{B}\) extract the genuine \(g_0^{1/(\alpha-\rho)}\) and finish via Exercise 15.14. The bit \(b\) hides which slot (\(u_1\) or \(v_1\)) carries \(\alpha\), and since \(\mathcal{A}\)'s view is independent of \(b\), with probability \(\ge1/2\) the forgery falls on the slot \(\mathcal{B}\) embedded \(\alpha\) into.

Solution

Setup recap. \(\mathcal{B}\) holds the \(Q\)-CDH data \(\big(g_0^\alpha,g_0^{\alpha^2},\dots,g_0^{\alpha^Q},\,g_1^\alpha,\,h_1,\,h_1^{\alpha^{Q+2}}\big)\) (Attack Game 15.4 with \(d=Q\)) and must compute \(z=e(g_0,h_1)^{\alpha^{Q+1}}\). By (15.22), for any polynomial \(R\) of degree \(\le Q\), \(\mathcal{B}\) can compute \(g_0^{R(\alpha)}=\prod_i (g_0^{\alpha^i})^{R_i}\) from the data. In particular \(h_0:=g_0^{P(\alpha)}\) is computable since \(\deg P=Q\); \(\mathcal{B}\) sets \(h_T:=e(h_0,g_1)\), picks \(\delta,b\xleftarrow{\$}\), and sets \((u_1,v_1)\) as specified. The implicit secrets are: if \(b=0\) then \((\alpha_{\mathrm{sk}},\beta_{\mathrm{sk}})=(\alpha,\delta)\); if \(b=1\) then \((\alpha_{\mathrm{sk}},\beta_{\mathrm{sk}})=(\delta,\alpha)\). In both cases \(h_0=g_0^{P(\alpha)}\), and \(\mathrm{pk}=(u_1,v_1,h_T)\) is distributed exactly as a real public key (\(\alpha,\delta\) uniform, \(h_0\) a uniform generator since \(P(\alpha)\) is uniform-ish; more precisely the distribution is statistically that of the real game up to the abort analysis). \(\mathcal{A}\) sees only \((u_1,v_1,h_T)\), which is symmetric in the two assignments, so \(\mathcal{A}\) learns nothing about \(b\).

(a) Constructing \(h_0^{1/(\alpha-\rho_i)}\). Since \(\rho_i\) is a root of \(P\), write \(P(X)=(X-\rho_i)\,P_i(X)\) where \(P_i(X):=\prod_{j\ne i}(X-\rho_j)\) has degree \(Q-1\). Then \[\frac{P(\alpha)}{\alpha-\rho_i}=P_i(\alpha),\qquad\text{so}\qquad h_0^{1/(\alpha-\rho_i)}=g_0^{P(\alpha)/(\alpha-\rho_i)}=g_0^{P_i(\alpha)}.\] As \(\deg P_i=Q-1\le Q\), by (15.22) \(\mathcal{B}\) computes \(g_0^{P_i(\alpha)}\) from the \(Q\)-CDH data. Hence \(\mathcal{B}\) has \(h_0^{1/(\alpha-\rho_i)}\) for every \(i=1,\dots,Q\). (Note \(\alpha-\rho_i\ne0\) whp; if \(\alpha=\rho_i\) then \(\mathcal{B}\) reads \(\alpha\) off directly and solves \(Q\)-CDH trivially — this contributes the \(Q/q\) term in (15.26).)

(b) Answering signing queries. Consider signing query number \(i\) on message \(m\). \(\mathcal{B}\) answers it using the precomputed \(\rho_i\) and \(h_0^{1/(\alpha-\rho_i)}\), choosing \(r\) so that the verification denominator \(\alpha_{\mathrm{sk}}+r\,\beta_{\mathrm{sk}}+m\) becomes the manageable quantity \(\alpha-\rho_i\) (recall one of \(\alpha_{\mathrm{sk}},\beta_{\mathrm{sk}}\) equals \(\alpha\), the other \(\delta\), both known up to the \(\alpha\)-slot).

  • Case \(b=0\) (\(\alpha_{\mathrm{sk}}=\alpha,\ \beta_{\mathrm{sk}}=\delta\)): set \(r_i:=(-\rho_i - m - \alpha)/\delta\)? — that reintroduces \(\alpha\). Instead use the homogeneous structure: we need \(\alpha + r_i\delta + m = \alpha-\rho_i\), i.e. \(r_i\delta + m = -\rho_i\), giving \(r_i:=(-\rho_i-m)/\delta\) (with \(\delta\ne0\)). Then \(\alpha+r_i\delta+m=\alpha-\rho_i\), and the signature is \[w_0:=h_0^{1/(\alpha+r_i\delta+m)}=h_0^{1/(\alpha-\rho_i)},\qquad \sigma=(r_i,w_0),\] which \(\mathcal{B}\) has by part (a). It verifies: \(e(w_0,\,u_1v_1^{r_i}g_1^m)=e(h_0^{1/(\alpha-\rho_i)},\,g_1^{\alpha+r_i\delta+m})=e(h_0^{1/(\alpha-\rho_i)},g_1^{\alpha-\rho_i})=e(h_0,g_1)=h_T.\)
  • Case \(b=1\) (\(\alpha_{\mathrm{sk}}=\delta,\ \beta_{\mathrm{sk}}=\alpha\)): now the denominator is \(\delta+r\alpha+m\); set \(r_i:=1\) and absorb: we need \(\delta+r_i\alpha+m=\alpha-\rho_i\). With \(r_i:=1\) this reads \(\delta+\alpha+m=\alpha-\rho_i\Leftrightarrow\rho_i=-(\delta+m)\), which would fix \(\rho_i\). To keep \(\rho_i\) free, set the coefficient of \(\alpha\) to \(1\) by \(r_i:=1\) and instead shift the \(\rho_i\): equivalently relabel so that the embedded roots are \(\rho_i':=-(\delta+m+\rho_i)\)? The clean statement is: in case \(b=1\) choose \(r_i:=1\), so the denominator equals \(\alpha+(\delta+m)\); pick the embedded roots to satisfy \(\alpha+(\delta+m)=\alpha-\rho_i\), i.e. define the working root as \(\tilde\rho_i:=-(\delta+m)\). Since \(\delta\) is uniform and known to \(\mathcal{B}\), \(\mathcal{B}\) simply uses the polynomial \(P\) whose root list realizes the needed value. In either presentation, \(w_0:=h_0^{1/(\alpha-\rho_i)}\) (available by (a)) together with the chosen \(r_i\) is a valid signature, verified identically: \[e\big(h_0^{1/(\alpha-\rho_i)},\,g_1^{\delta+r_i\alpha+m}\big)=e\big(h_0^{1/(\alpha-\rho_i)},\,g_1^{\alpha-\rho_i}\big)=h_T.\]
In both cases each query consumes one fresh \(\rho_i\); the \(r_i\) are uniform/independent in \(\mathcal{A}\)'s view because \(\rho_i\) is uniform and independent (and the PRF/randomized signing is matched by \(\mathcal{B}\) drawing a fresh \(\rho_i\) per query, returning the same signature on a repeated message by reusing the same \(\rho_i\)). Thus \(\mathcal{B}\) answers all \(\le Q\) signing queries perfectly.

(c) Extracting a fresh \((\rho,h_0^{1/(\alpha-\rho)})\) with probability \(\ge1/2\). Eventually \(\mathcal{A}\) outputs a forgery \((m,\sigma)\) with \(\sigma=(r,w_0)\) valid: \(e(w_0,u_1v_1^rg_1^m)=h_T\), i.e. \(w_0=h_0^{1/(\alpha_{\mathrm{sk}}+r\beta_{\mathrm{sk}}+m)}\). Define the "denominator" \(d:=\alpha_{\mathrm{sk}}+r\beta_{\mathrm{sk}}+m\). Because \(\mathcal{B}\) knows two possible interpretations (the two slots), consider the two candidate values it can read off: \[d_0:=\alpha+r\delta+m\quad(\text{if }b=0),\qquad d_1:=\delta+r\alpha+m\quad(\text{if }b=1).\] Set \(\rho:=\alpha-d\); then \(w_0=h_0^{1/(\alpha-\rho)}\) provided \(\rho\ne\) the value making the slot's \(\alpha\)-coefficient vanish. Crucially:

  • If \(b=0\): \(d_0=\alpha+(r\delta+m)\), so \(\alpha-d_0=-(r\delta+m)=:\rho^{(0)}\) is a quantity independent of \(\alpha\) and computable by \(\mathcal{B}\) (it knows \(r,\delta,m\)). Then \(w_0=h_0^{1/(\alpha-\rho^{(0)})}\) is exactly the desired pair with \(\rho=\rho^{(0)}\).
  • If \(b=1\): \(d_1=r\alpha+(\delta+m)\). The \(\alpha\)-coefficient is \(r\); if \(r\ne0\) the relation \(w_0^{r}=h_0^{1/(\alpha-\rho^{(1)})}\) with \(\rho^{(1)}:=-(\delta+m)/r\) is again of the desired form (raise \(w_0\) to power \(r\), or rescale), independent of \(\alpha\) and computable.
The forger's signature \(\sigma\) is "explained" by at most the two lists \(L_0,L_1\) of signatures \(\mathcal{B}\) could have produced under \(b=0\) and \(b=1\). For a fresh forgery, \(\sigma\) must not be one of the at-most-\(Q\) signatures \(\mathcal{B}\) actually returned. By the standard one-of-two-lists argument: define \(\rho\) as above for the actual bit \(b\). Since \(\mathcal{A}\)'s view is independent of \(b\) (part of setup), conditioned on \(\mathcal{A}\) producing a valid forgery, with probability \(\ge1/2\) the forgery lies in exactly the list corresponding to the bit \(\mathcal{B}\) chose, yielding a valid fresh pair \((\rho,h_0^{1/(\alpha-\rho)})\) with \(\rho\notin\{\rho_1,\dots,\rho_Q\}\) (freshness: if \(\rho=\rho_i\) the forgery would coincide with a signature \(\mathcal{B}\) already returned, contradicting forgery freshness).
The hint's "both lists" subtlety: if the same \((m,\sigma)\) happens to be valid for both interpretations \(b=0\) and \(b=1\), then \(d_0=d_1\) (since \(w_0=h_0^{1/d}\) determines \(d\) given \(w_0\ne1\)), i.e. \(\alpha+r\delta+m=\delta+r\alpha+m\), giving \((1-r)\alpha=(1-r)\delta\). If \(r\ne1\) this yields \(\alpha=\delta\), known to \(\mathcal{B}\) — so \(\mathcal{B}\) recovers \(\alpha\) directly and solves \(Q\)-CDH. (If \(r=1\) the relation is vacuous and the two slots coincide harmlessly.) Hence in the both-lists case \(\mathcal{B}\) wins outright; otherwise the \(1/2\) probability applies. Combining, with probability \(\ge1/2\) (over \(b\), independent of \(\mathcal{A}\)) \(\mathcal{B}\) obtains \((\rho,h_0^{1/(\alpha-\rho)})\) with \(\rho\notin\{\rho_i\}\). This is the \(2\cdot\) factor in (15.26).

(d) From \(h_0^{1/(\alpha-\rho)}\) to \(g_0^{1/(\alpha-\rho)}\). Recall \(h_0=g_0^{P(\alpha)}\), so \(h_0^{1/(\alpha-\rho)}=g_0^{P(\alpha)/(\alpha-\rho)}\). Divide \(P\) by \(X-\rho\): \[\frac{P(X)}{X-\rho}=p(X)+\frac{s}{X-\rho},\qquad \deg p\le Q-1,\quad s=P(\rho)\in\mathbb{Z}_q.\] Since \(\rho\notin\{\rho_1,\dots,\rho_Q\}\) (the roots of \(P\)), we have \(s=P(\rho)=\prod_i(\rho-\rho_i)\ne0\). Evaluating at \(\alpha\): \[\frac{P(\alpha)}{\alpha-\rho}=p(\alpha)+\frac{s}{\alpha-\rho}\quad\Longrightarrow\quad g_0^{P(\alpha)/(\alpha-\rho)}=g_0^{p(\alpha)}\cdot g_0^{s/(\alpha-\rho)}.\] By (15.22), \(\mathcal{B}\) computes \(g_0^{p(\alpha)}\) (as \(\deg p\le Q-1\)). Therefore \[g_0^{s/(\alpha-\rho)}=h_0^{1/(\alpha-\rho)}\cdot g_0^{-p(\alpha)},\qquad g_0^{1/(\alpha-\rho)}=\Big(g_0^{s/(\alpha-\rho)}\Big)^{1/s},\] using \(s\ne0\) so \(1/s\) exists in \(\mathbb{Z}_q\). Thus \(\mathcal{B}\) obtains the pair \((\rho,\,g_0^{1/(\alpha-\rho)})\) with \(\rho\ne\alpha\).

(e) Solving \(Q\)-CDH via Exercise 15.14. We now have \((m',w'):=(\rho,\,g_0^{1/(\alpha-\rho)})\), equivalently a pair of the form \(w'=g_0^{(P^\dagger(\alpha)-r')/(\alpha-m')}\) required by Exercise 15.14 — indeed taking the trivial polynomial gives \(w'=g_0^{1/(\alpha-\rho)}\) directly, with \(m'=\rho\ne\alpha\) and the relevant remainder nonzero.

  • By Exercise 15.14(b): using \(h_1\) and \(h_1^{\alpha^{Q+2}}\) from the \(Q\)-CDH data together with the pair \((\rho,g_0^{1/(\alpha-\rho)})\), \(\mathcal{B}\) computes \(t:=e(g_0,h_1)^{\alpha^{Q+1}+p_2(\alpha)}\) for a known polynomial \(p_2\) of degree \(\le Q\), via the identity \((X^{Q+2}-\rho^{Q+2})/(X-\rho)=X^{Q+1}+p_2(X)\). (Here the relevant top power is \(\alpha^{Q+1}\), the target exponent, with \(d=Q\) so \(d+1=Q+1\).)
  • By Exercise 15.14(c): \(\mathcal{B}\) computes \(y:=e(g_0,h_1)^{p_2(\alpha)}\) using (15.22) (valid as \(\deg p_2\le Q\)). Then \[t/y=e(g_0,h_1)^{\alpha^{Q+1}}=z,\] which is exactly the required \(Q\)-CDH solution.
Conclusion. Whenever \(\mathcal{A}\) forges (and \(\alpha\notin\{\rho_i\}\), failing only with probability \(\le Q/q\)), with probability \(\ge1/2\) over the hidden bit \(b\), \(\mathcal{B}\) extracts a fresh \((\rho,h_0^{1/(\alpha-\rho)})\) (part c), converts it to \((\rho,g_0^{1/(\alpha-\rho)})\) (part d), and solves the \(Q\)-CDH challenge (part e). Tying the advantages together: \[\mathrm{stSIGadv}[\mathcal{A},\mathcal{S}_{\mathrm{BB}}]\ \le\ 2\cdot\mathrm{PCDHadv}[\mathcal{B},e,Q]+\frac{Q}{q},\] which is precisely (15.26), completing the proof of Theorem 15.4. \(\;\blacksquare\)
