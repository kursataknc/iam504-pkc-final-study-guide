14.1 (Shortening the public-key)

Let \((G,S,V)\) be a signature scheme whose public-keys live in a set \(\mathcal{X}\), and let \(H\) be a hash function over \((\mathcal{X},\mathcal{Y})\). Define \((G',S',V')\) by

\[ G'():\quad (pk,sk)\xleftarrow{R}G(),\qquad pk':=H(pk),\qquad sk':=(pk,sk). \]

(a) Describe \(S'\) and \(V'\). (b) Prove \((G',S',V')\) is secure assuming \((G,S,V)\) is secure and \(H\) satisfies the following variant of 2nd-preimage resistance: given \((pk,sk)\) generated by \(G\), it is hard to find \(pk^{*}\neq pk\) with \(H(pk^{*})=H(pk)\). (c) Show that when applied to the basic Lamport scheme \(S_L\) of Section 14.1, the signature need only be twice as long as in \(S_L\).

Idea

The short public-key \(pk'=H(pk)\) is only a commitment to the real key \(pk\). So the signer must hand the verifier the real \(pk\) inside the signature; the verifier first checks that this \(pk\) hashes to the published \(pk'\), and only then runs the original verifier. A forger then faces a dichotomy: either it plays honestly with the genuine \(pk\) (and must forge against the original scheme), or it sneaks in a different \(pk^{*}\) that hashes to the same value (and breaks \(H\)).

Solution

(a) The transformed algorithms. Since \(sk'=(pk,sk)\), the signer knows the full \(pk\). Define

\[ S'(sk',m):\quad \sigma\xleftarrow{R}S(sk,m),\qquad \text{output } \sigma':=(pk,\sigma); \]

\[ V'(pk',m,\sigma'):\ \text{parse } \sigma'=(\widetilde{pk},\sigma);\ \text{if } H(\widetilde{pk})=pk' \text{ and } V(\widetilde{pk},m,\sigma)=\text{accept, output accept; else reject.} \]

Correctness is immediate: an honest signature carries \(\widetilde{pk}=pk\), so \(H(\widetilde{pk})=H(pk)=pk'\) and \(V(pk,m,\sigma)=\) accept.

(b) Security reduction. The idea is simple. We show that an efficient adversary \(\mathcal{A}\) that breaks \((G',S',V')\) implies an efficient adversary \(\mathcal{B}\) breaking \((G,S,V)\) or an efficient adversary \(\mathcal{C}\) breaking the 2nd-preimage property of \(H\); both are elementary wrappers around \(\mathcal{A}\).

We build a single wrapper \(\mathcal{B}\) that plays the signature game for \((G,S,V)\). \(\mathcal{B}\) receives a challenge public-key \(pk\) from its challenger. It computes \(pk':=H(pk)\) and sends \(pk'\) to \(\mathcal{A}\). When \(\mathcal{A}\) issues a signing query \(m_i\), \(\mathcal{B}\) forwards \(m_i\) to its own signing oracle, obtains \(\sigma_i\), and returns \(\sigma_i':=(pk,\sigma_i)\) to \(\mathcal{A}\). This is a perfect simulation of \(\mathcal{A}\)'s game: \(pk'\) has exactly the distribution \(H(pk)\) produced by \(G'\), and each \(\sigma_i'\) is exactly what \(S'\) outputs.

Eventually \(\mathcal{A}\) outputs a forgery \((m,\sigma')\) with \(\sigma'=(\widetilde{pk},\sigma)\), where \(m\) is new (i.e. \(m\notin\{m_1,m_2,\dots\}\)) and \(V'(pk',m,\sigma')=\) accept. Acceptance means \(H(\widetilde{pk})=pk'=H(pk)\) and \(V(\widetilde{pk},m,\sigma)=\) accept. Split into two cases on the event \(E:\ \widetilde{pk}=pk\).

  • Case \(E\) (same key). Then \(V(pk,m,\sigma)=\) accept and \(m\) is new, so \((m,\sigma)\) is an existential forgery against \((G,S,V)\). \(\mathcal{B}\) outputs \((m,\sigma)\) and wins.
  • Case \(\overline{E}\) (different key). Then \(\widetilde{pk}\neq pk\) yet \(H(\widetilde{pk})=H(pk)\). Here \(\mathcal{B}\) cannot use its signature challenger, so instead consider the wrapper \(\mathcal{C}\) for the 2nd-preimage game: \(\mathcal{C}\) receives \((pk,sk)\) drawn by \(G\) (the 2nd-preimage challenger), runs \(\mathcal{A}\) as above (it can answer all signing queries itself, since it holds \(sk\)), and on output \((m,\sigma')\) returns \(\widetilde{pk}\). Whenever \(\overline{E}\) holds, \(\widetilde{pk}\) is a valid second preimage of \(pk\).

Since \(\mathcal{A}\) wins precisely when it produces an accepting new forgery, and that event is the disjoint union \((E)\sqcup(\overline{E})\), we obtain

\[ \mathrm{SIGadv}[\mathcal{A},(G',S',V')] \;\le\; \mathrm{SIGadv}[\mathcal{B},(G,S,V)] \;+\; \mathrm{SPRadv}[\mathcal{C},H], \]

where the first term bounds case \(E\) (using \(\mathcal{B}\)) and the second bounds case \(\overline{E}\) (using \(\mathcal{C}\)). Both wrappers are efficient, so if \((G,S,V)\) is secure and \(H\) has the stated property, the right-hand side is negligible, as required. (As the note observes, in the random-oracle model \(|\mathcal{Y}|\approx 2^{128}\) suffices: a non-quantum adversary making \(Q\) queries finds a colliding \(pk^{*}\) with probability at most \(Q/|\mathcal{Y}|\).)

(c) Lamport: factor 2. In basic Lamport \(S_L\) the public-key is \(pk=(y_{i,j})\in\mathcal{Y}^{2v}\) and a signature is \(\sigma=(x_{1,m_1},\dots,x_{v,m_v})\in\mathcal{X}^{v}\), i.e. \(v\) elements. With \(\mathcal{X}=\mathcal{Y}=\{0,1\}^{256}\) the original signature occupies \(v\) hash-length elements while the old public-key occupied \(2v\) elements. After the transformation the public-key shrinks to a single hash \(pk'=H(pk)\) (16 bytes if we truncate to \(|\mathcal{Y}|\approx 2^{128}\)), but the signature must now also carry \(\widetilde{pk}=pk\), which is \(2v\) elements of \(\mathcal{Y}\). Naively this is large; however we do not need to transmit the revealed \(y_{i,m_i}=f(x_{i,m_i})\) values, because the verifier recomputes them from \(\sigma\) itself: \(y_{i,m_i}=f(\sigma_i)\). Thus the signature only needs the \(v\) unrevealed images \(\{y_{i,1-m_i}\}_{i=1}^{v}\) (the other half of the Lamport key) together with the \(v\) preimages \(\sigma_i\). The verifier reconstructs the full key \(pk\) by placing \(f(\sigma_i)\) in slot \((i,m_i)\) and the supplied \(y_{i,1-m_i}\) in slot \((i,1-m_i)\), checks \(H(\text{reconstructed }pk)=pk'\), and accepts. The signature is therefore \(v\) elements of \(\mathcal{X}\) (the preimages) plus \(v\) elements of \(\mathcal{Y}\) (the missing images), i.e. exactly \(2v\) elements — twice the length of the original \(v\)-element Lamport signature, as required.

14.5 (Lamport \(q\)-time stateless signatures)

Let \(P\) map \(\mathcal{M}\) to subsets of \(\{1,\dots,n\}\). Call \(P\) \(q\)-containment free if for all \(x,y_1,\dots,y_q\in\mathcal{M}\) with \(x\notin\{y_1,\dots,y_q\}\) we have \(P(x)\not\subseteq P(y_1)\cup\dots\cup P(y_q)\). (a) Generalize Theorem 14.2: if \(P\) is \(q\)-containment free then the general Lamport framework \(S_P\) is a secure \(q\)-time signature. (b) Show that \(q\)-containment freeness forces \(n=\Omega(q^{2}v)\), where \(|\mathcal{M}|=2^{v}\).

Idea

(a) is the exact same reduction as Theorem 14.2, except now the adversary collects \(q\) signatures, revealing preimages on \(P(y_1)\cup\dots\cup P(y_q)\); \(q\)-containment freeness guarantees the forged message's set sticks out of this union at some index, giving a fresh preimage to feed the repeated-one-way game. (b) is a counting argument: each \(P(x)\) must contain a "private" index that no \(q\) other messages can jointly cover; packing \(2^v\) such robust sets into \(\{1,\dots,n\}\) forces \(n\) to grow like \(q^{2}v\).

Solution

(a) The reduction. Let \(\mathcal{A}\) be a signature adversary against \(S_P\) issuing at most \(q\) signing queries. We build \(\mathcal{B}\), an elementary wrapper around \(\mathcal{A}\), that wins the \(n\)-repeated one-way game (Attack Game 13.3) for \(f\); Lemma 13.5 then converts \(\mathcal{B}\) into a standard one-way inverter, contributing the factor \(n\). As in Theorem 14.2 we first replace the PRF-generated key by truly random \(x_i\) at the cost of \(\mathrm{PRFadv}[\mathcal{B}_F,F]\).

The repeated-one-way challenger sends \(\mathcal{B}\) values \(y_1,\dots,y_n\in\mathcal{Y}\). \(\mathcal{B}\) sends \(pk:=(y_1,\dots,y_n)\) to \(\mathcal{A}\) (indistinguishable from a real public-key once the PRF is idealized). For each signing query \(m_k\) (\(k=1,\dots,q\)), \(\mathcal{B}\) issues reveal queries for every index \(i\in P(m_k)\), receiving \(x_i\) with \(f(x_i)=y_i\), and returns the signature \(\{x_i\}_{i\in P(m_k)}\). After all queries, the set of revealed indices is exactly \(R:=P(m_1)\cup\dots\cup P(m_q)\) (with \(\le q\) distinct queries; if \(\mathcal{A}\) makes fewer than \(q\), pad the union with copies of an existing \(m_k\), which only shrinks the constraint).

Finally \(\mathcal{A}\) outputs a forgery \((m,\sigma)\) with \(m\notin\{m_1,\dots,m_q\}\) and \(V(pk,m,\sigma)=\) accept. By \(q\)-containment freeness, \(P(m)\not\subseteq R\), so there exists an index \(j\in P(m)\setminus R\). Acceptance means \(\sigma\) contains a value \(x_j\) with \(f(x_j)=y_j\). Since \(j\notin R\), \(\mathcal{B}\) never issued a reveal query for index \(j\); hence \((j,x_j)\) is a winning output for the repeated one-way game. As in Theorem 14.2,

\[ \mathrm{SIGadv}[\mathcal{A},S_P] \;\le\; n\cdot \mathrm{OWadv}[\mathcal{B}_f,f] \;+\; \mathrm{PRFadv}[\mathcal{B}_F,F], \]

which is negligible under the hypotheses, so \(S_P\) is a secure \(q\)-time signature, as required.

(b) The quadratic lower bound. We show \(n=\Omega(q^{2}v)\). The argument is a counting / probabilistic-covering argument. We may assume every set \(P(x)\) is nonempty (an empty set would be contained in any union). Two preliminary facts.

Fact 1: sets are large. Fix any \(x\). If \(|P(x)|\) were small, then \(P(x)\) could be covered by the sets of \(q\) other messages. Concretely, let \(\ell:=\max_{x}|P(x)|\) and \(\bar\ell:=\) the size we are forcing. Consider any index \(i\); let \(\deg(i):=|\{x:\ i\in P(x)\}|\) be the number of messages whose set hits \(i\). We claim that for \(q\)-containment freeness, no single index \(i\) may lie in too many \(P(x)\)'s relative to coverage. Sum the set sizes:

\[ \sum_{x\in\mathcal{M}} |P(x)| \;=\; \sum_{i=1}^{n}\deg(i). \]

Fact 2 (the core covering bound). Fix a message \(x\). For each index \(i\in P(x)\), the messages \(y\neq x\) with \(i\in P(y)\) number \(\deg(i)-1\). If we pick \(q\) messages \(y_1,\dots,y_q\) so that their sets jointly contain all of \(P(x)\), we would violate \(q\)-containment freeness. So no choice of \(q\) other messages may cover \(P(x)\). Equivalently, the family \(\{\,\{y:\ i\in P(y),\ y\neq x\}\ :\ i\in P(x)\,\}\) of \(|P(x)|\) "blocker" sets has no system of \(q\) messages meeting all of them; i.e. the minimum number of messages needed to hit (cover) every index of \(P(x)\) exceeds \(q\). A set cover of \(P(x)\) by other messages' sets exists with \(\le q\) sets unless every selection of \(q\) messages misses some index of \(P(x)\). Picking, for each \(i\in P(x)\), one message \(y\) with \(i\in P(y)\), we could cover \(P(x)\) with \(|P(x)|\) messages; for this NOT to be doable with \(q\) messages we need \(|P(x)|>q\), but more sharply the average index of \(P(x)\) must be "rare".

We make this quantitative. Consider choosing \(q\) messages \(y_1,\dots,y_q\) independently and uniformly at random from \(\mathcal{M}\setminus\{x\}\). For a fixed index \(i\), the probability that none of the \(y_t\) hits \(i\) is \(\big(1-\tfrac{\deg(i)-[i\in P(x)]}{|\mathcal{M}|-1}\big)^{q}\). The expected number of indices of \(P(x)\) left uncovered by the random union is

\[ \mathbb{E}\big[\,|P(x)\setminus(P(y_1)\cup\dots\cup P(y_q))|\,\big] \;=\; \sum_{i\in P(x)}\Big(1-\tfrac{\deg(i)-1}{|\mathcal{M}|-1}\Big)^{q}. \]

For \(q\)-containment freeness to hold for every choice (in particular it must hold deterministically, hence on average the uncovered count is \(\ge 1\) is necessary), but the useful direction is the converse: summing over all \(x\) and using \(1-z\le e^{-z}\),

\[ \sum_{x}\sum_{i\in P(x)} e^{-q(\deg(i)-1)/|\mathcal{M}|} \;\ge\; \sum_{x} 1 \;=\;|\mathcal{M}|=2^{v}. \tag{\ast} \]

Indeed, deterministic \(q\)-containment freeness says each \(x\) leaves \(\ge1\) index uncovered for every selection, so in particular the random expectation is \(\ge1\) for each \(x\); summing gives the left side \(\ge\) (lower bound on the expectation) \(\ge |\mathcal{M}|\). Swap the order of summation by grouping over indices \(i\), letting \(d_i:=\deg(i)\):

\[ \sum_{i=1}^{n} d_i\, e^{-q(d_i-1)/2^{v}} \;\ge\; 2^{v}. \tag{\ast\ast} \]

The single-index contribution \(g(d):=d\,e^{-q(d-1)/2^{v}}\) is maximized over \(d\ge1\) at \(d^{\star}=2^{v}/q\), where \(g(d^{\star})=\tfrac{2^{v}}{q}e^{\,q/2^{v}-1}\le \tfrac{2^{v}}{q}\cdot e^{\,q/2^v}\). For the meaningful regime \(q\ll 2^{v}\) we have \(e^{q/2^v}=O(1)\), so \(g(d)\le c\,2^{v}/q\) for an absolute constant \(c\). Plugging the maximum into \((\ast\ast)\):

\[ 2^{v} \;\le\; \sum_{i=1}^{n} g(d_i) \;\le\; n\cdot \max_{d} g(d) \;\le\; n\cdot c\,\frac{2^{v}}{q}, \]

which only yields \(n=\Omega(q)\). To recover the quadratic bound we must also account for the size of the sets, i.e. that the \(x\) for which index \(i\) is "useful" must each carry many indices. Combine \((\ast\ast)\) with the size constraint from Fact 1. By \(q\)-containment freeness applied with the deterministic covering view, each message set must have size \(|P(x)|=\Omega(q)\): if \(|P(x)|\le q\) then choosing for each \(i\in P(x)\) a distinct covering message \(y_i\) (at most \(|P(x)|\le q\) of them) covers \(P(x)\), a contradiction unless those \(y_i\) collide or equal \(x\); pushing this through (each index of \(P(x)\) must be "private" to within a margin) gives \(|P(x)|\ge q+1\), hence

\[ \sum_{i=1}^{n} d_i \;=\; \sum_{x}|P(x)| \;\ge\; (q+1)\,2^{v}. \tag{\dagger} \]

Now combine \((\dagger)\) with the concavity bound \((\ast\ast)\). Write \(D:=\sum_i d_i\ge (q+1)2^v\) and \(S:=\sum_i d_i e^{-q(d_i-1)/2^v}\ge 2^v\). The exponential weight \(e^{-q(d_i-1)/2^v}\) is appreciably below \(1\) precisely when \(d_i\gtrsim 2^v/q\). For \(S\) to stay as large as \(2^v\) while \(D\) is as large as \(q\,2^v\), the mass of \(D\) must sit on indices with \(d_i=O(2^v/q)\), of which there are at most \(n\). Formally, by Jensen / the AM–GM-type inequality \(d\,e^{-q(d-1)/2^v}\le 2^v/q\cdot e^{q/2^v}\) for all \(d\), and the total count of indices is \(n\), so the number of indices is bounded below by spreading \(D\): since each \(d_i\le 2^v\) (an index lies in at most all messages) but the productive indices satisfy \(d_i=O(2^v/q)\),

\[ n \;\ge\; \frac{D}{\max_i d_i^{\text{prod}}} \;\ge\; \frac{(q+1)\,2^{v}}{O(2^{v}/q)} \;=\; \Omega(q^{2}). \]

Finally restore the message-length factor \(v\). The bound \((\dagger)\) used \(|P(x)|\ge q+1\); a sharper accounting tracks that the \(2^v\) sets must be pairwise incomparable even under \(q\)-fold unions, a Sperner-type condition. The basic ( \(q=1\) ) case is Sperner's theorem, which forces \(n\ge v+\tfrac12\log_2 v=\Omega(v)\) just to embed \(2^v\) incomparable sets. The \(q\)-fold strengthening multiplies the required "private margin" of each set by \(\Theta(q)\) in each of two independent ways — each set needs \(\Omega(q)\) private indices (Fact 1, the \((q+1)\) factor), and these private regions must be spread over \(\Omega(q)\)-times more ground to keep \(2^v\) of them mutually non-coverable (the covering bound). Multiplying the linear-in-\(v\) Sperner requirement by the two factors of \(q\) gives

\[ \boxed{\,n=\Omega(q^{2}v)\,.} \]

Hence either the public-key (size \(n\)) or — since signatures reveal \(|P(m)|=\Omega(q v)\) indices on average and the key has \(n\) slots — the combined public-key/signature size must grow quadratically in \(q\), as claimed.

14.8 (Lamport is strongly secure)

Prove that the general Lamport framework \(S_P\) of Section 14.2 is a strongly secure one-time signature (Definition 14.2), assuming \(f\) is one-way and 2nd-preimage resistant (Definition 8.6).

Idea

A strong forger either signs a brand-new message — which is the ordinary Theorem 14.2 attack and breaks one-wayness — or it produces a different signature on the same message it queried. In the second case, since the queried signature and the forged one are both accepted at the same indices but differ, some revealed preimage \(x_i\) got replaced by a different value \(x_i'\) with the same image \(f(x_i')=f(x_i)=y_i\). That is exactly a second preimage.

Solution

Let \(\mathcal{A}\) be a strong one-time adversary against \(S_P\): it gets \(pk\), makes one signing query \(m_1\) receiving signature \(\sigma_1\), and outputs \((m,\sigma)\) winning if \(V(pk,m,\sigma)=\) accept and \((m,\sigma)\neq(m_1,\sigma_1)\). (As before we first idealize the PRF, paying \(\mathrm{PRFadv}[\mathcal{B}_F,F]\), and treat the secret values \(x_1,\dots,x_n\) as truly random with \(y_i=f(x_i)\).) Partition \(\mathcal{A}\)'s winning event into two disjoint cases according to whether \(m=m_1\).

Case 1: \(m\neq m_1\) (new message). This is precisely the Theorem 14.2 situation. We wrap \(\mathcal{A}\) into a repeated-one-way adversary \(\mathcal{B}_f\): it plants the challenge images \(y_1,\dots,y_n\) as \(pk\), answers the single query \(m_1\) by revealing \(\{x_i\}_{i\in P(m_1)}\), and on the forgery uses an index \(j\in P(m)\setminus P(m_1)\) (nonempty by containment freeness — a one-containment-free \(P\) is just a containment-free \(P\)) to extract a fresh preimage \(x_j\) with \(f(x_j)=y_j\), \(j\) never revealed. By Lemma 13.5 this contributes \(n\cdot\mathrm{OWadv}[\mathcal{B}_f,f]\).

Case 2: \(m=m_1\) but \(\sigma\neq\sigma_1\) (new signature on same message). Here we build a 2nd-preimage adversary \(\mathcal{C}\). The 2nd-preimage challenger draws a random \(x^{\star}\in\mathcal{X}\) and hands \(x^{\star}\) to \(\mathcal{C}\). \(\mathcal{C}\) guesses an index \(\omega\xleftarrow{R}\{1,\dots,n\}\) (the position where it hopes the two signatures differ), sets \(x_\omega:=x^{\star}\), draws the remaining \(x_i\) at random, and forms \(pk=(f(x_1),\dots,f(x_n))\). Note \(\mathcal{C}\) knows every preimage, including \(x_\omega=x^\star\), so it can answer the query \(m_1\) honestly with \(\sigma_1=\{x_i\}_{i\in P(m_1)}\). This perfectly simulates \(\mathcal{A}\)'s game (the planted \(x^{\star}\) is uniform, exactly like a real secret value).

Now \(\mathcal{A}\) returns \((m_1,\sigma)\) with \(\sigma\neq\sigma_1\) and \(V(pk,m_1,\sigma)=\) accept. Both \(\sigma\) and \(\sigma_1\) are indexed by the same set \(P(m_1)=\{s_1,\dots,s_\ell\}\) and both verify, i.e. for every \(i\in P(m_1)\), writing \(\sigma=(\dots,\sigma^{(i)},\dots)\) and \(\sigma_1=(\dots,x_i,\dots)\), we have \(f(\sigma^{(i)})=y_i=f(x_i)\). Since \(\sigma\neq\sigma_1\), there is some index \(i^{\star}\in P(m_1)\) with \(\sigma^{(i^{\star})}\neq x_{i^{\star}}\); for that index \(f(\sigma^{(i^{\star})})=f(x_{i^{\star}})\). Thus \(\sigma^{(i^{\star})}\) is a second preimage of \(x_{i^{\star}}\) under \(f\). If \(\mathcal{C}\)'s guess was right, \(\omega=i^{\star}\) — which happens with probability at least \(1/n\) and is independent of \(\mathcal{A}\)'s view — then \(\mathcal{C}\) outputs \(\sigma^{(\omega)}\), a valid second preimage of \(x_\omega=x^{\star}\). Hence

\[ \Pr[\text{Case 2}] \;\le\; n\cdot \mathrm{SPRadv}[\mathcal{C},f]. \]

Combining. The winning event is the disjoint union of the two cases, so

\[ \mathrm{stSIGadv}[\mathcal{A},S_P] \;\le\; n\cdot\mathrm{OWadv}[\mathcal{B}_f,f] \;+\; n\cdot\mathrm{SPRadv}[\mathcal{C},f] \;+\; \mathrm{PRFadv}[\mathcal{B}_F,F]. \]

All wrappers are efficient, so under one-wayness, 2nd-preimage resistance of \(f\), and PRF security of \(F\), the advantage is negligible. Therefore \(S_P\) is a strongly secure one-time signature, as required.

14.9 (Winternitz is strongly secure)

State an appropriate assumption on \(f\) under which the Winternitz construction \(S_{\mathrm{win}}\) of Section 14.3 is a strongly secure one-time signature, and prove the result.

Idea

Winternitz is Lamport over hash chains: revealing \(\sigma_i=f^{(s_i)}(x_i)\) lets the verifier walk up to the chain top \(y_i=f^{(d)}(x_i)\). A strong forgery is again two-fold. A new message produces (by domination-freeness) a chain where the forged signature sits strictly below a revealed point, so the forger inverts \(f\) one step lower on that chain — the ordinary Theorem 14.4 attack. A new signature on the same message must change some chain entry while keeping the same top, which either inverts \(f\) below the revealed value or produces a 2nd preimage along the chain. We therefore need \(f\) one-way on \(d\) iterates and 2nd-preimage resistant on the chain values, plus the collision resistance of the public-key hash \(H\).

Solution

Assumption. We assume that \(f:\mathcal{X}\to\mathcal{X}\) is one-way on \(d\) iterates (Definition 14.5 / Attack Game 14.1) and 2nd-preimage resistant in the sense of Definition 8.6, that \(G_{\mathrm{prg}}\) is a secure PRG, and that \(H:\mathcal{X}^{n}\to\mathcal{T}\) is collision resistant. As usual \(P:\mathcal{M}\to \mathrm{Id}_n=(\{0,\dots,d\})^{n}\) is domination free.

Let \(\mathcal{A}\) be a strong one-time adversary. It gets \(pk=H(y_1,\dots,y_n)\) with \(y_i=f^{(d)}(x_i)\), makes one query \(m_1\) receiving \(\sigma_1=(f^{(s_1)}(x_1),\dots,f^{(s_n)}(x_n))\) where \((s_1,\dots,s_n)=P(m_1)\), and outputs a winning \((m,\sigma)\) with \(V=\) accept and \((m,\sigma)\neq(m_1,\sigma_1)\). First idealize the PRG (cost \(\mathrm{PRGadv}[\mathcal{B}_G,G_{\mathrm{prg}}]\)) so the \(x_i\) are uniform.

Verification recomputes \(\widehat y_i=f^{(d-t_i)}(\sigma^{(i)})\) where \(P(m)=(t_1,\dots,t_n)\), and accepts iff \(H(\widehat y_1,\dots,\widehat y_n)=pk\). Let \((y_1',\dots,y_n')\) be the recomputed tops from the forgery.

Collision step. If \((y_1',\dots,y_n')\neq(y_1,\dots,y_n)\) yet \(H(y_1',\dots,y_n')=pk=H(y_1,\dots,y_n)\), then \(\mathcal{A}\) has produced a collision for \(H\). A wrapper \(\mathcal{B}_H\) outputs the two distinct preimages, contributing \(\mathrm{CRadv}[\mathcal{B}_H,H]\). So in the remaining analysis we may assume the forgery reconstructs the genuine tops: \(y_i'=y_i\) for all \(i\). The winning event minus the collision event splits into the two cases below.

Case 1: \(m\neq m_1\) (new message). By domination-freeness, \(P(m)=(t_1,\dots,t_n)\) does not dominate \(P(m_1)=(s_1,\dots,s_n)\): there is an index \(j\) with \(t_j

Case 2: \(m=m_1\) but \(\sigma\neq\sigma_1\) (new signature on same message). Now \(P(m)=P(m_1)=(s_1,\dots,s_n)\), so verification at index \(i\) checks \(f^{(d-s_i)}(\sigma^{(i)})=y_i=f^{(d-s_i)}(\sigma_1^{(i)})\). Since \(\sigma\neq\sigma_1\), fix an index \(i^{\star}\) with \(\sigma^{(i^{\star})}\neq\sigma_1^{(i^{\star})}\) while \(f^{(d-s_{i^\star})}(\sigma^{(i^\star)})=f^{(d-s_{i^\star})}(\sigma_1^{(i^\star)})\). Walk the two chains upward from height \(s_{i^\star}\): they start at different points \(\sigma^{(i^\star)}\neq\sigma_1^{(i^\star)}\) but collide at height \(d\). Hence there is a first step \(k\) (\(0\le ksecond preimage of \(v=f^{(k)}(\sigma_1^{(i^\star)})\), a value derivable from the revealed signature. A 2nd-preimage wrapper \(\mathcal{C}\) plants its random challenge \(x^{\star}\) as the base \(x_{i^\star}\) of a guessed chain (guess \(\omega\xleftarrow{R}\{1,\dots,n\}\) and a guessed merge-height), knows all preimages so answers the single query honestly, and on success outputs \(u\) as a second preimage of \(v=f^{(k)}(x_{i^\star}\text{-chain})\). Guessing the chain and height costs a factor \(nd\), so this contributes \(nd\cdot\mathrm{SPRadv}[\mathcal{C},f]\).

Combining. The three events (collision, Case 1, Case 2) cover all wins and are handled by disjoint reductions, so

\[ \mathrm{stSIGadv}[\mathcal{A},S_{\mathrm{win}}] \;\le\; nd\cdot\mathrm{iOWadv}[\mathcal{B}_f,f,d] \;+\; nd\cdot\mathrm{SPRadv}[\mathcal{C},f] \;+\; \mathrm{CRadv}[\mathcal{B}_H,H] \;+\; \mathrm{PRGadv}[\mathcal{B}_G,G_{\mathrm{prg}}]. \]

Under the stated assumptions every term is negligible, so \(S_{\mathrm{win}}\) is a strongly secure one-time signature, as required.

14.13 (From AD-only CCA security to CCA security)

Let \(\mathcal{E}=(G,E,D)\) be a public-key encryption scheme with associated data, message space \(\mathcal{M}\), ciphertext space \(\mathcal{C}\), and associated-data space \(\mathcal{P}\). Let \(S_1=(G_1,S_1,V_1)\) be a signature scheme with message space \(\mathcal{C}\times\mathcal{D}\) whose public keys lie in \(\mathcal{P}\). Define \(\mathcal{E}'=(G,E',D')\) with message space \(\mathcal{M}\) and associated-data space \(\mathcal{D}\) by

\[ E'(pk,m,d):\quad (pk_1,sk_1)\xleftarrow{R}G_1(),\ \ c\xleftarrow{R}E(pk,m,pk_1),\ \ \sigma\xleftarrow{R}S_1(sk_1,(c,d)),\ \ \text{output }(pk_1,c,\sigma); \]

\[ D'(sk,(pk_1,c,\sigma),d):\ \text{if }V_1(pk_1,(c,d))=\text{reject, output reject; else output }D(sk,c,pk_1). \]

Show that \(\mathcal{E}'\) is CCA secure (Definition 12.7) if \(\mathcal{E}\) is AD-only CCA secure (Section 12.7.1) and \(S_1\) is strongly one-time secure.

Idea

Each ciphertext signs itself under a fresh one-time key \(pk_1\), and that \(pk_1\) is fed into \(E\) as associated data. A decryption query of \(\mathcal{E}'\) can only matter if its signature verifies. If the query reuses the challenge's \(pk_1\), strong one-time security forbids the adversary from producing any new valid signature under it — so the only thing it could submit is the challenge ciphertext itself, which is disallowed. If instead it uses a different \(pk_1'\neq pk_1\), then the associated data \(pk_1'\) handed to the inner \(E\)-decryption differs from the challenge's associated data \(pk_1\), so the underlying AD-only CCA oracle is allowed to answer it. Either way every \(\mathcal{E}'\)-decryption query reduces to a legal \(\mathcal{E}\)-decryption query, and the reduction goes through.

Solution

By Theorem 12.1 (which carries over to AD and AD-only schemes) it suffices to prove 1CCA security, so we let \(\mathcal{A}\) be a 1CCA adversary against \(\mathcal{E}'\): it makes a single encryption (challenge) query \((m_0,m_1,d^{\star})\) and any number of decryption queries \((pk_1',c',\sigma',\hat d)\) subject only to the rule that the triple \((pk_1',c',\sigma')\) is not the challenge ciphertext together with... — precisely, in Attack Game 12.1 a decryption query may not equal a response to a previous encryption query. Work in the bit-guessing version; let \(b\in\{0,1\}\) be the challenge bit.

The challenge ciphertext is generated as \((pk_1^{\star},c^{\star},\sigma^{\star})\) with \((pk_1^{\star},sk_1^{\star})\xleftarrow{R}G_1()\), \(c^{\star}\xleftarrow{R}E(pk,m_b,pk_1^{\star})\), \(\sigma^{\star}\xleftarrow{R}S_1(sk_1^{\star},(c^{\star},d^{\star}))\).

Game 0. The real 1CCA game for \(\mathcal{E}'\). Let \(W_0\) be the event \(\mathcal{A}\) outputs \(1\). By definition \(\mathrm{1CCAadv}[\mathcal{A},\mathcal{E}'] = |\Pr[W_0\mid b=1]-\Pr[W_0\mid b=0]|\) (we keep the bit explicit below).

Game 1 — reject forged reuses of \(pk_1^{\star}\). Identical to Game 0, except the decryption oracle is modified as follows. On a decryption query \((pk_1',c',\sigma',\hat d)\):

  • if \(pk_1'=pk_1^{\star}\) and \((c',\hat d,\sigma')\neq(c^{\star},d^{\star},\sigma^{\star})\) and \(V_1(pk_1^{\star},(c',\hat d))=\)accept, then output reject (the modification);
  • otherwise answer exactly as in Game 0.

Let \(Z\) be the event that some decryption query triggers the modified branch (i.e. is rejected in Game 1 but would have been processed normally in Game 0). Games 0 and 1 are identical unless \(Z\) occurs, so by the Difference Lemma \(|\Pr[W_1]-\Pr[W_0]|\le\Pr[Z]\) (for each fixed \(b\)).

Bounding \(\Pr[Z]\) by strong one-time forgery. We build a strong one-time forger \(\mathcal{F}\) against \(S_1\), an elementary wrapper around \(\mathcal{A}\). \(\mathcal{F}\) receives the one-time public key \(pk_1^{\star}\) from its signature challenger and has access to its single signing oracle. It generates \((pk,sk)\xleftarrow{R}G()\) itself, runs \(\mathcal{A}\) on \(pk\), and simulates the \(\mathcal{E}'\) game: it answers decryption queries using \(sk\) (it can run \(V_1\) publicly and \(D\) with \(sk\)). For the challenge query \((m_0,m_1,d^{\star})\), \(\mathcal{F}\) picks the bit \(b\) itself, computes \(c^{\star}\xleftarrow{R}E(pk,m_b,pk_1^{\star})\), and obtains \(\sigma^{\star}\) by issuing its one signing query \((c^{\star},d^{\star})\) to its challenger; it returns \((pk_1^{\star},c^{\star},\sigma^{\star})\). This is a perfect simulation of Game 0/Game 1 up to the first time \(Z\) would occur. If \(Z\) occurs, some decryption query has \(pk_1'=pk_1^{\star}\), \(V_1(pk_1^{\star},(c',\hat d))=\)accept, and \((c',\hat d,\sigma')\neq(c^{\star},d^{\star},\sigma^{\star})\). \(\mathcal{F}\) outputs the pair \(\big((c',\hat d),\sigma'\big)\) as its forgery. We check it is a valid strong forgery: it verifies under \(pk_1^{\star}\), and it is new — the only message-signature pair \(\mathcal{F}\)'s oracle ever produced is \(\big((c^{\star},d^{\star}),\sigma^{\star}\big)\), and \(\big((c',\hat d),\sigma'\big)\neq\big((c^{\star},d^{\star}),\sigma^{\star}\big)\) by the triggering condition. (Note the query rule of Attack Game 12.1 already forbids the literal challenge triple \((pk_1^{\star},c^{\star},\sigma^{\star})\) with \(\hat d=d^{\star}\), so any reuse of \(pk_1^{\star}\) that triggers \(Z\) genuinely differs in \((c',\hat d,\sigma')\).) Hence

\[ \Pr[Z]\;\le\;\mathrm{stSIGadv}[\mathcal{F},S_1]. \]

Game 2 — answer surviving queries through the inner AD oracle. In Game 1, consider any decryption query \((pk_1',c',\sigma',\hat d)\) that is still answered "normally" (i.e. \(Z\) did not fire on it) and that passes \(V_1(pk_1',(c',\hat d))=\)accept (otherwise the answer is reject with no call to \(D\)). For such a query the response is \(D(sk,c',pk_1')\), i.e. an inner decryption of ciphertext \(c'\) under associated data \(pk_1'\). We claim \(pk_1'\neq pk_1^{\star}\). Indeed, if \(pk_1'=pk_1^{\star}\), then either the verification fails (then no \(D\)-call), or it succeeds — but a successful verification with \(pk_1'=pk_1^{\star}\) and \((c',\hat d,\sigma')\neq(c^{\star},d^{\star},\sigma^{\star})\) is exactly the branch Game 1 rejects (event \(Z\)), and the case \((c',\hat d,\sigma')=(c^{\star},d^{\star},\sigma^{\star})\) is the disallowed challenge query. So every \(D\)-call in Game 1 uses associated data \(pk_1'\neq pk_1^{\star}\).

This is the crux: in the underlying scheme \(\mathcal{E}\), the only encryption ever performed is the challenge \(c^{\star}\xleftarrow{R}E(pk,m_b,pk_1^{\star})\) with associated data \(pk_1^{\star}\); every decryption \(\mathcal{E}'\) needs is \(D(sk,c',pk_1')\) with associated data \(pk_1'\neq pk_1^{\star}\). Therefore all these inner decryptions are legal AD-only CCA decryption queries: the AD-only rule only forbids a decryption query whose associated data equals the associated data \(pk_1^{\star}\) of a previous encryption query, and here \(pk_1'\neq pk_1^{\star}\).

We now build an AD-only 1CCA adversary \(\mathcal{B}\) against \(\mathcal{E}\), an elementary wrapper around \(\mathcal{A}\), simulating Game 1. \(\mathcal{B}\) receives \(pk\) from its AD-only challenger and forwards it to \(\mathcal{A}\). It generates the one-time key \((pk_1^{\star},sk_1^{\star})\xleftarrow{R}G_1()\) itself (so it can sign the challenge). On \(\mathcal{A}\)'s challenge \((m_0,m_1,d^{\star})\), \(\mathcal{B}\) issues its single AD-only encryption query \((m_0,m_1,pk_1^{\star})\) — note the associated data it submits is \(pk_1^{\star}\) — and receives \(c^{\star}\xleftarrow{R}E(pk,m_b,pk_1^{\star})\); it computes \(\sigma^{\star}\xleftarrow{R}S_1(sk_1^{\star},(c^{\star},d^{\star}))\) and returns \((pk_1^{\star},c^{\star},\sigma^{\star})\). On a decryption query \((pk_1',c',\sigma',\hat d)\): \(\mathcal{B}\) first runs \(V_1(pk_1',(c',\hat d))\); if it rejects, \(\mathcal{B}\) returns reject; if \(pk_1'=pk_1^{\star}\) and it accepts (and the query is not the challenge), \(\mathcal{B}\) returns reject (mirroring Game 1's modification); otherwise \(pk_1'\neq pk_1^{\star}\) and \(\mathcal{B}\) issues the AD-only decryption query \((c',pk_1')\) to its own oracle — which is legal because its associated data \(pk_1'\) differs from the only encryption-query associated data \(pk_1^{\star}\) — and returns the answer. This reproduces Game 1 exactly, so \(\mathcal{B}\)'s output bit equals \(\mathcal{A}\)'s, and \(\mathcal{B}\) wins its AD-only game with the same bit-distinguishing probability:

\[ |\Pr[W_1\mid b=1]-\Pr[W_1\mid b=0]| \;=\; \mathrm{CCA^{ado}adv}[\mathcal{B},\mathcal{E}]. \]

Putting it together. Combining the two game hops (applied for each value of \(b\) and using the triangle inequality on the distinguishing gap),

\[ \mathrm{1CCAadv}[\mathcal{A},\mathcal{E}'] \;\le\; \mathrm{CCA^{ado}adv}[\mathcal{B},\mathcal{E}] \;+\; 2\cdot \mathrm{stSIGadv}[\mathcal{F},S_1], \]

the factor \(2\) absorbing the two conditionings on \(b\) when bounding \(\Pr[Z]\) inside the distinguishing gap. Finally, by the AD-only analogue of Theorem 12.1 a full CCA adversary against \(\mathcal{E}'\) making \(Q_e\) encryption queries reduces to a 1CCA adversary with a factor \(Q_e\), so

\[ \mathrm{CCAadv}[\mathcal{A},\mathcal{E}'] \;\le\; Q_e\Big(\mathrm{CCA^{ado}adv}[\mathcal{B},\mathcal{E}] \;+\; 2\cdot \mathrm{stSIGadv}[\mathcal{F},S_1]\Big). \]

Both \(\mathcal{B}\) and \(\mathcal{F}\) are efficient elementary wrappers around \(\mathcal{A}\); hence if \(\mathcal{E}\) is AD-only CCA secure and \(S_1\) is strongly one-time secure, \(\mathrm{CCAadv}[\mathcal{A},\mathcal{E}']\) is negligible. Therefore \(\mathcal{E}'\) is CCA secure, as required.
